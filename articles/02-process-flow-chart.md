# 工程フローチャート（Process Flow Chart）

## 1. 概要

### 工程フローチャートとは

工程フローチャート（Process Flow Chart）とは、製品の製造工程の全体の流れを**視覚的に表現した図**です。原材料・部品の受入から完成品の出荷に至るまで、すべての工程ステップを時系列に並べ、加工・検査・運搬・保管などの作業種別を標準化された記号で表します。

> **名称について**: AIAG APQPマニュアルでは「Process Flow Chart」、IATF 16949（条項 8.3.5.2）では「Manufacturing Process Flow Charts」が正式名称です。実務では「Process Flow Diagram (PFD)」「工程フローチャート」とも呼ばれますが、本記事ではAPQPの正式名称に準拠します。

工程設計における**最初に作成すべき成果物**であり、後続のPFMEA・コントロールプラン・作業標準書など、すべての工程設計文書の基礎となります。

### 目的

| 目的 | 説明 |
|------|------|
| 工程の可視化 | 製造プロセス全体を誰でも一目で把握できるようにする |
| 抜け漏れの防止 | 全工程を洗い出すことで、必要な作業や検査の見落としを防ぐ |
| PFMEA・CPへの展開 | 各工程ステップに対してリスク分析や管理計画を策定する土台を作る |
| コミュニケーション | 生産技術・製造・品質・購買など部門間の共通認識を形成する |
| 工程改善の基礎 | 現状工程を図示することで、ムダ・ボトルネックを特定しやすくする |
| 監査対応 | IATF 16949やISO 9001の監査で、工程管理の体系を説明する根拠となる |

### 工程設計における位置づけ

```
製品図面・仕様書
    │
    ▼
┌─────────────┐
│ 工程フローチャート  │ ← 全ての工程設計文書の起点
└─────────────┘
    │
    ├──→ PFMEA（各工程ステップのリスク分析）
    ├──→ 設備仕様書（各工程で使用する設備の定義）
    ├──→ 工場レイアウト図（工程の流れに基づく配置計画）
    └──→ 工程設計書（全体の総括文書）
```

工程フローチャートの**工程番号**は、PFMEA・コントロールプラン・QC工程表・作業標準書と一致させます。これにより、すべての文書がトレーサブル（追跡可能）な状態になります。

## 2. 工程図記号（JIS Z 8206）

工程フローチャートでは、**JIS Z 8206（工程図記号）** で規定された標準記号を使用します。

### 基本記号

| 記号 | 名称 | 英語 | 定義 |
|:----:|------|------|------|
| ○ | 加工 | Operation | 原材料・部品・製品の形状、性質、状態に変化を与える作業 |
| ◇ | 検査 | Inspection | 品物の特性値を測定し、基準と比較して合否を判定する作業 |
| ⇒ | 運搬 | Transport | 品物がある位置から別の位置に移動すること |
| ▽ | 貯蔵 | Storage | 計画的に品物を蓄えている状態（倉庫保管など） |
| D | 停滞 | Delay | 計画にない一時的な停止・待ち状態 |

### 複合記号

基本記号を組み合わせて、同時に行われる作業を表現します。

| 記号 | 名称 | 意味 | 例 |
|:----:|------|------|-----|
| ○の中に◇ | 加工＋数量検査 | 加工しながら数量を確認 | カウンター付きプレス機での加工 |
| ◇の中に○ | 検査が主の加工 | 検査を主目的としながら加工を伴う | 選別しながらの仕分け作業 |
| □ | 数量検査 | 品物の数量を検査する | 員数確認 |
| ◇の中に□ | 品質検査＋数量検査 | 品質と数量を同時に検査 | 外観検査しながらの員数確認 |

### 記号の使い分けのポイント

- **加工（○）**: 製品に付加価値を与える作業すべてに使用。切削、溶接、組立、塗装、熱処理なども含む
- **検査（◇）**: 測定・試験・目視確認など判定を伴う作業。「量産中に全数行うか / 抜取で行うか」も併記することが多い
- **運搬（⇒）**: 工程間の移動。コンベア搬送、フォークリフト移動、手運びなど。距離や手段を添えて書くとより有用
- **貯蔵（▽）**: 意図的な保管。原材料倉庫、中間品の棚保管、完成品倉庫など
- **停滞（D）**: 意図しない待ち。ロット待ち、工程間の滞留、乾燥待ちなど。工程改善で削減対象となる

## 3. 記載項目

### ヘッダー情報

| 項目 | 内容 |
|------|------|
| 品名 / 製品名 | 対象製品の名称 |
| 品番 / 図番 | 製品の部品番号や図面番号 |
| 作成日 | 初版の作成日 |
| 改訂日 / 改訂番号 | 最新改訂の日付と版数 |
| 作成者 | 作成した担当者名 |
| 承認者 | 承認した責任者名 |
| 適用工場 / ライン | 対象となる生産拠点・ライン名 |

### 各工程ステップの記載項目

| 項目 | 説明 | 必須/任意 |
|------|------|:---------:|
| 工程番号 | 一意の識別番号（PFMEA・CPと一致させる） | 必須 |
| 工程図記号 | JIS Z 8206に基づく記号 | 必須 |
| 工程名 | 工程の名称（例：プレス成形、バリ取り） | 必須 |
| 作業内容 | 工程で行う作業の概要説明 | 必須 |
| 使用設備 | 設備名と管理番号 | 推奨 |
| 使用治工具 | 治具・金型などの名称 | 推奨 |
| 製品特性 | その工程で決まる製品の品質特性（寸法、外観など） | 推奨 |
| 工程特性 | 品質に影響する工程パラメータ（温度、圧力など） | 推奨 |
| 特殊特性 | 安全性・法規に関わる重要特性の識別マーク | 必須 |
| 材料投入 | その工程で投入する原材料・部品 | 該当時 |
| アウトプット | その工程の出力（中間品、廃棄物、スクラップ含む） | 推奨 |

### 特殊特性の表記

特殊特性（Special Characteristics）は、安全性・法規制・機能に重大な影響を及ぼす特性です。工程フローチャート上で明確に識別します。

| 記号 | 分類 | 意味 | 例 |
|:----:|------|------|-----|
| CC / ◆ | Critical Characteristic | 安全性・法規制に関わる特性 | ブレーキ部品の締付けトルク |
| SC / ◇ | Significant Characteristic | 製品機能に重大な影響を及ぼす特性 | 嵌合部の寸法公差 |

> **注意**: 特殊特性の記号は顧客（OEM）ごとに異なる場合があります。顧客固有要求事項（CSR: Customer Specific Requirements）を必ず確認してください。

## 4. 作成手順

### Step 1: インプット情報の収集

工程フローチャートを作成する前に、以下のインプットを揃えます。

- 製品図面・仕様書（設計記録）
- E-BOM（設計部品表）
- 特殊特性リスト
- 類似製品の既存工程フローチャート（あれば）
- 顧客固有要求事項

### Step 2: 工程の洗い出し

受入から出荷まで、すべての工程を時系列で洗い出します。

**洗い出しのチェックポイント**:
- 原材料・部品の受入検査は含まれているか
- 工程間の運搬・搬送は考慮されているか
- 中間品の保管・一時置きは識別されているか
- 検査工程（工程内検査、最終検査）は漏れていないか
- 梱包・出荷検査は含まれているか
- 手直し・リワーク工程は考慮されているか
- 不適合品の処理フロー（隔離・選別・廃棄）は含まれているか

### Step 3: 工程番号の付与

各工程に一意の番号を付与します。

**番号付けのルール例**:

| 方式 | 例 | メリット |
|------|-----|---------|
| 連番方式 | 10, 20, 30, 40... | シンプル。10飛ばしにすると後から挿入しやすい |
| 階層方式 | 1.0, 1.1, 1.2, 2.0, 2.1... | 大工程と小工程の関係が明確 |
| 区分方式 | R-10, P-10, A-10, I-10... | 工程種別（Receiving, Press, Assembly, Inspection）が一目で分かる |

> **推奨**: 10飛ばしの連番方式（10, 20, 30...）が最も広く使われています。後から工程を追加する場合に、15, 25 のように間に挿入できるためです。

### Step 4: 工程図記号と詳細情報の記入

各工程に対して、記号・設備・特性などの情報を記入します。

### Step 5: フローの接続と分岐の確認

- 工程間の流れを矢印で接続
- 検査工程の判定後の分岐（合格→次工程、不合格→手直し or 廃棄）を明記
- 部品合流点（サブアセンブリの投入など）を明記

### Step 6: レビューと承認

以下の関係者でレビューを実施します。

| 参加者 | レビュー観点 |
|--------|-------------|
| 生産技術 | 工程の技術的な成立性、設備能力 |
| 製造 | 現場での実行可能性、作業性 |
| 品質保証 | 検査工程の妥当性、特殊特性の網羅性 |
| 生産管理 | 物流・在庫の観点 |
| 安全衛生 | 作業安全の観点 |

## 6. 後続文書への展開

工程フローチャートは、後続の成果物に対して以下のように展開されます。

### PFMEA への展開

工程フローチャートの各ステップが、PFMEAの分析対象となります。

```
工程フローチャート                    PFMEA
──────────                    ──────
工程No.50 1次プレス成形   →   工程ステップ: 1次プレス成形
  ◆ 絞り深さ              →     故障モード: 絞り深さ不足
                                 影響: 後工程で組付不可 → 顧客クレーム
                                 原因: 金型摩耗 / 材料ロット変動
                                 現行予防管理: 金型寿命管理
                                 現行検出管理: 初物検査で絞り深さ測定
```

### コントロールプランへの展開

PFMEAの結果を反映して、各工程の管理方法を規定します。

```
工程フローチャート                    コントロールプラン
──────────                    ─────────────
工程No.50 1次プレス成形   →   工程No.50 1次プレス成形
  ◆ 絞り深さ              →     製品特性: 絞り深さ 15.0±0.3mm
                                 管理方法: ノギスで測定
                                 頻度: 初物・中間・終物 各1個
                                 対応計画: NG時は生産停止→金型確認
```

### 工程番号の一致

すべての文書で工程番号を一致させることが重要です。

| 文書 | 工程No.50 の記載内容 |
|------|---------------------|
| 工程フローチャート | ○ 1次プレス成形（400tプレス機 / 絞り金型） |
| PFMEA | 故障モード・影響・原因・管理方法の分析 |
| コントロールプラン | 管理特性・仕様・測定方法・頻度・対応計画 |
| QC工程表 | 管理項目・管理基準・管理方法・責任者 |
| 作業標準書 | 作業者が実行する具体的な手順と注意点 |

## 7. 類似文書との違い

### 工程フローチャート vs QC工程表

| 観点 | 工程フローチャート | QC工程表 |
|------|-------------|---------|
| 主な目的 | 工程全体の流れの可視化 | 各工程の品質管理方法の規定 |
| 表現形式 | フローチャート（図） | 表形式 |
| 詳細度 | 全体俯瞰（マクロ） | 管理項目の詳細（ミクロ） |
| 工程記号 | 使用する | 使用する |
| 管理基準の記載 | 通常なし | あり（具体的な数値基準） |
| 作成タイミング | 最初に作成 | 工程フローチャートの後に作成 |

### 工程フローチャート vs バリューストリームマップ（VSM）

| 観点 | 工程フローチャート | バリューストリームマップ |
|------|-------------|----------------------|
| 主な目的 | 工程の流れと管理の定義 | ムダの特定と改善（リーン生産） |
| 対象範囲 | 1製品の製造工程 | 受注〜出荷の情報・物の流れ全体 |
| 記載内容 | 加工・検査・運搬・保管 | リードタイム・在庫量・情報フロー |
| 活用場面 | 新製品の工程設計 | 既存工程の改善活動 |
| 規格との関係 | APQP / IATF 16949 で要求 | トヨタ生産方式 / リーン手法 |

## 8. よくある課題と対策

### 課題1: 工程の粒度が不適切

**問題**: 工程を細かく書きすぎて膨大になる、または粗すぎてPFMEAに展開できない。

**対策**:
- **大工程フローチャート**（全体俯瞰用）と**詳細工程フローチャート**（PFMEA展開用）の2段階で作成する
- 目安として、1つの工程ステップが「1つの設備 / 1つの作業場所」に対応する粒度が適切
- PFMEAで故障モードを分析できる単位まで分解されていることを確認する

### 課題2: 工程番号がPFMEA・コントロールプランと不一致

**問題**: 工程フローチャートを改訂した際に、PFMEA・コントロールプランの工程番号が更新されず、トレーサビリティが崩れる。

**対策**:
- 工程フローチャートの改訂時は、関連文書（PFMEA、CP、QC工程表、作業標準書）の同時改訂を必須とする運用ルールを定める
- 文書管理システム（DMS）やBOPシステムで工程番号を一元管理する

### 課題3: 「現状」と「あるべき姿」の混同

**問題**: 新製品の工程設計時に、類似品の既存フローをそのままコピーしてしまい、改善の機会を逃す。

**対策**:
- まず「あるべき工程」を白紙から検討し、その後に既存設備や制約条件を考慮して現実解に落とし込む
- 類似品の工程フローチャートは参考にするが、そのままコピーしない

### 課題4: 不適合品の処理フローが不明確

**問題**: 検査NG時の分岐（手直し・選別・廃棄）が工程フローチャートに明記されていない。

**対策**:
- すべての検査工程に対して、合格・不合格の分岐を明記する
- 手直し品が再投入される場合は、再投入先の工程を矢印で示す
- スクラップ品の処理フローも含める

### 課題5: 運搬・保管の軽視

**問題**: 加工・検査の工程のみ記載し、運搬・保管を省略してしまう。

**対策**:
- 運搬と保管も品質リスクの発生源である（搬送キズ、錆、異品混入など）
- すべての工程間移動と保管ポイントを図に含める
- 特に異品種混入リスクがある搬送・保管ポイントは明確に識別する

---

## 9. ベストプラクティス

1. **工程フローチャートはチームで作成する**: 生産技術だけでなく、製造・品質・生産管理の関係者を交えて作成する
2. **現場を歩いて確認する**: 机上で作成した後、実際の製造現場（または類似品の現場）を歩いて整合性を確認する
3. **PFMEAと同時並行で見直す**: PFMEAの分析過程で工程の追加・変更が必要になることが多い。両文書は反復的に更新する
4. **改訂管理を徹底する**: 工程変更時は必ず工程フローチャートを改訂し、関連文書も連動して更新する
5. **簡潔さを保つ**: 1枚の図で全体が俯瞰できることが理想。詳細は作業標準書に委ねる
6. **電子化を検討する**: Excel・Visio・専用ツールで作成し、変更管理を容易にする
