# QC工程表（QC Process Chart）

## 1. 概要

### QC工程表とは

QC工程表（QC Process Chart / QC工程図）とは、製造工程における品質管理の方法・基準を**工程ごとに一覧化した管理文書**です。原材料の受入から完成品の出荷まで、「いつ」「どこで」「だれが」「なにを」「どのように」管理するかを明確に定めます。

QC工程表は**日本の製造業で独自に発展した品質管理文書**であり、JISやIATF 16949などの国際規格で直接要求されるものではありません。しかし、日本の多くの製造業で品質保証の中核文書として広く使用されており、実質的にはコントロールプランと類似した役割を果たしています。

### 目的

| 目的 | 説明 |
|------|------|
| 品質管理体系の明文化 | 各工程で「何を、どう管理するか」を文書化し、品質保証の仕組みを可視化する |
| 管理の標準化 | 担当者が変わっても同じ品質管理が継続できるようにする |
| 異常時の対応明確化 | 管理基準から逸脱した際の処置手順を事前に定める |
| 教育訓練の基礎資料 | 新人や異動者に品質管理のポイントを体系的に伝える |
| 監査・顧客対応 | 品質管理体制を顧客や監査員に説明する根拠資料とする |
| 工程改善の起点 | 管理項目・基準を見直すことで継続的改善につなげる |

### 工程設計における位置づけ

```
工程フローチャート
    │
    ▼
PFMEA（リスク分析）
    │
    ▼
コントロールプラン ←→ QC工程表
    │                    │
    ▼                    ▼
作業指示書           現場での品質管理活動
```

QC工程表はコントロールプランと並行して、または代替として使用されます。自動車産業（IATF 16949適用）ではコントロールプランが必須ですが、日本の多くの製造業ではQC工程表を主たる管理文書として運用しています。

## 2. 記載項目

### ヘッダー情報

| 項目 | 内容 |
|------|------|
| 品名 / 製品名 | 対象製品の名称 |
| 品番 / 図番 | 製品の部品番号や図面番号 |
| 工程名称 | 対象となる工程全体の名称 |
| 作成日 / 改訂日 | 初版日と最新改訂日 |
| 改訂番号 | 版数管理 |
| 作成者 / 承認者 | 作成・承認の責任者名 |
| 適用工場 / ライン | 対象となる生産拠点・ライン名 |

### 本文の記載項目

| # | 項目 | 説明 | 例 |
|---|------|------|-----|
| 1 | 工程番号 | 工程フローチャートと一致する番号 | 10, 20, 30... |
| 2 | 工程名 | 工程の名称 | プレス成形、熱処理、外観検査 |
| 3 | 工程図記号 | JIS Z 8206に基づく記号 | ○（加工）、◇（検査） |
| 4 | 設備・装置 | 使用する設備の名称と管理番号 | 400tプレス機 (EQ-001) |
| 5 | 管理特性（原因系） | 品質に影響を与える工程パラメータ | 金型温度、プレス圧力、送り速度 |
| 6 | 品質特性（結果系） | 管理特性の結果として現れる品質 | 寸法、外観、硬度、重量 |
| 7 | 管理基準 | 合否判定の基準値・公差 | 15.0±0.3mm、Ra1.6以下 |
| 8 | 管理方法 | 測定・監視の手法 | ノギス測定、目視検査、管理図 |
| 9 | 測定機器 | 使用する測定器具の名称 | マイクロメータ、三次元測定機 |
| 10 | 測定頻度 | サンプリングの頻度と数量 | 毎ロット初物3個、2時間毎1個 |
| 11 | 担当者 / 責任者 | 管理の実施責任者 | 作業者、班長、品質検査員 |
| 12 | 記録方法 | 測定結果の記録方法・帳票 | 検査成績書、管理図、チェックシート |
| 13 | 異常時の処置 | 管理基準逸脱時の対応手順 | 生産停止→班長報告→選別→是正 |

### 管理特性（原因系）と品質特性（結果系）

QC工程表の特徴的な考え方として、**原因系と結果系を分けて管理する**点があります。

```
原因系（管理特性）              結果系（品質特性）
───────────                ───────────
プレス圧力: 200±10 kN    →   絞り深さ: 15.0±0.3 mm
金型温度: 180±5 ℃       →   成形品外観: キズ・ヒケなし
冷却時間: 30±3 sec       →   寸法精度: ±0.05 mm
```

- **原因系を管理する**ことで、結果系の不具合を**未然に防止**できる
- **結果系を検査する**ことで、原因系の管理が適切か**検証**できる
- 品質問題発生時には、原因系のどのパラメータが変動したかを追跡できる

## 3. 作成手順

### Step 1: インプットの準備

- 工程フローチャート（全工程の流れ）
- 製品図面・仕様書（品質特性の把握）
- PFMEA（リスク分析結果）※ある場合
- 過去の不良実績・クレーム情報
- 類似製品のQC工程表（あれば）

### Step 2: 工程の転記

工程フローチャートから全工程を転記します。工程番号・工程名・工程図記号を一致させます。

### Step 3: 管理項目の設定

各工程に対して、管理すべき特性を洗い出します。

**洗い出しの観点**:
- その工程で**決定される**品質特性は何か（結果系）
- その品質特性に**影響を与える**工程パラメータは何か（原因系）
- 特殊特性（安全・法規に関わる特性）はどれか
- 過去に不良が発生した項目はあるか

### Step 4: 管理基準の設定

各管理項目に対して、具体的な数値基準を設定します。

| 基準の種類 | 説明 | 例 |
|-----------|------|-----|
| 規格値 | 製品図面・仕様書で定められた値 | φ10.0 +0.02/-0.00 mm |
| 工程管理値 | 規格値より厳しい社内管理値 | φ10.0 +0.015/-0.000 mm |
| 設備パラメータ基準 | 設備の設定値と許容範囲 | 温度: 180±5 ℃ |

### Step 5: 管理方法・頻度の決定

| 項目 | 決定のポイント |
|------|---------------|
| 測定方法 | 特性に適した測定器を選定（精度・分解能を考慮） |
| 頻度 | リスクの大きさに応じて設定（特殊特性は高頻度） |
| 記録方法 | 管理図・チェックシート・検査成績書など |

**頻度の目安**:

| リスクレベル | 頻度の例 |
|-------------|---------|
| 特殊特性（CC） | 全数検査、または高頻度の抜取検査 |
| 重要特性（SC） | ロット初物・中間・終物、または定時抜取 |
| 一般特性 | ロット初物のみ、または日次確認 |

### Step 6: 異常時の処置の設定

管理基準を逸脱した場合の対応手順を明確にします。

**基本的な処置フロー**:
1. **生産停止**: 異常を検知したら即座に生産を停止
2. **報告**: 上長（班長・工長）へ報告
3. **隔離**: 疑わしい製品を隔離・識別
4. **選別**: 前回良品確認以降の製品を選別検査
5. **原因調査**: 原因を特定し是正処置を実施
6. **生産再開**: 是正完了後、初物確認をして生産再開
7. **記録**: 一連の対応を記録

### Step 7: レビューと承認

生産技術・製造・品質保証の関係者でレビューし、承認を得ます。

## 5. コントロールプランとの違い

QC工程表とコントロールプランは類似した目的を持つ文書ですが、いくつかの重要な違いがあります。

| 観点 | QC工程表 | コントロールプラン |
|------|---------|------------------|
| 出自 | 日本の品質管理実務から発展 | 米国自動車産業（AIAG）が標準化 |
| 規格要求 | JIS・ISO等で直接要求されない | IATF 16949で必須要求 |
| フォーマット | 各社で自由に設計 | AIAGが標準フォーマットを規定 |
| 段階区分 | 通常は量産用の1種類 | 試作CP・量産試作CP・量産CPの3段階 |
| PFMEA連携 | 必須ではない（独立して作成可） | PFMEAの結果反映が前提 |
| 原因系/結果系 | 明確に区分して記載 | 製品特性・工程特性として記載 |
| Reaction Plan | 「異常時の処置」として記載 | 専用カラムで詳細に規定 |
| 特殊特性 | 記載することが多い | 必須記載（分類コード付き） |
| 国際的な通用性 | 主に日本国内 | 国際的に通用する |

### 使い分けの指針

| ケース | 推奨文書 |
|--------|---------|
| IATF 16949認証取得企業 | コントロールプラン（必須） |
| 自動車OEM・Tier1への納入 | コントロールプラン（顧客要求） |
| 日本国内の一般製造業 | QC工程表（慣行として定着） |
| 両方が必要な場合 | コントロールプランを主、QC工程表を補助的に運用 |

### 両文書を併用する場合の注意点

- 管理項目・基準値に**矛盾がないこと**を確認する
- どちらが**正（マスター文書）** かを明確にする
- 改訂時に**両方を同時更新**する運用ルールを定める
- 可能であれば**一本化**を検討する（二重管理のリスク回避）

## 6. よくある課題と対策

### 課題1: 管理項目が多すぎて現場が運用できない

**問題**: すべての特性を管理項目に入れてしまい、現場の測定負荷が膨大になる。

**対策**:
- **重点管理**の考え方を取り入れる：特殊特性・過去不良項目を優先
- 工程が安定している項目は頻度を下げる（工程能力に応じた調整）
- 設備の自動監視で代替できる項目は自動化する

### 課題2: QC工程表が形骸化している

**問題**: 作成したまま更新されず、実際の現場管理と乖離している。

**対策**:
- 定期的なレビュー（最低年1回）をスケジュール化する
- 工程変更・設備変更時の改訂を文書管理規定に組み込む
- 内部監査でQC工程表と現場の整合性を確認する

### 課題3: 原因系と結果系の区分が曖昧

**問題**: 管理特性と品質特性が混同され、原因追究がしにくい。

**対策**:
- 特性要因図（フィッシュボーンダイアグラム）を活用して原因系を特定する
- 「この工程で変化を与えるパラメータは何か」（原因系）と「その結果どの品質が決まるか」（結果系）を分けて整理する

### 課題4: 異常時の処置が具体的でない

**問題**: 「班長に報告」としか書かれておらず、具体的な判断基準や処置手順がない。

**対策**:
- 「誰に」「何を」「どのように」報告するか、その後の判断基準と処置手順を具体的に記載する
- 異常時処置フロー図を別紙で作成し、QC工程表から参照する

## 7. ベストプラクティス

1. **工程フローチャートと工程番号を一致させる**: トレーサビリティの基本
2. **原因系の管理を重視する**: 結果の検査だけでなく、原因となるパラメータを管理することで不良の未然防止を図る
3. **PFMEAの結果を反映する**: PFMEAがある場合は、高リスク項目を確実にQC工程表に反映する
4. **現場で使える文書にする**: 管理項目を絞り込み、現場作業者が日常的に参照・運用できるレベルにする
5. **定期的に見直す**: 工程能力の向上に合わせて管理基準・頻度を最適化する
6. **電子化を検討する**: 測定データとの連携、改訂管理の効率化のためデジタル化を推進する
