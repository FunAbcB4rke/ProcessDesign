# 作業指示書（Process Instructions）

## 1. 概要

### 作業指示書とは

作業指示書（Process Instructions）とは、コントロールプランやQC工程表で定められた管理方法を、**現場作業者が実行できる具体的な手順**として文書化したものです。

APQP 3rd Edition（2024年）では **Phase 3（工程設計・開発）のアウトプット** として「Process Instructions」が明記されています。実務では「作業手順書（Work Instructions / WI）」「作業標準書（Standard Operating Procedure / SOP）」とも呼ばれます。

### 目的

| 目的 | 説明 |
|------|------|
| 品質の均一化 | 「誰がやっても同じ品質」を実現するための標準手順を提供する |
| コントロールプランの実行 | CPで規定した管理方法・検査方法を現場レベルの手順に落とし込む |
| 教育訓練の基礎 | 新人作業者・異動者に対する教育の基本資料とする |
| 安全の確保 | 作業者の安全に関するルールを明示する（IATF 16949で必須要求） |
| 異常時の対応 | 不良発生・設備異常時のエスカレーション手順を事前に定める |
| トレーサビリティ | 工程番号・管理項目をPFMEA・CPと一致させ、文書間の追跡可能性を確保する |

### 工程設計における位置づけ

```
工程フローチャート
    │
    ▼
PFMEA（リスク分析）
    │
    ▼
コントロールプラン / QC工程表
    │
    ├──→ 作業指示書（現場作業者向けの実行手順）
    │         │
    │         ├── 組立作業指示書
    │         ├── 検査作業指示書
    │         └── 段取り替え手順書
    │
    └──→ 検査基準書（検査員向けの判定基準）
```

作業指示書は、コントロールプランの「管理方法」「測定技法」「対応計画」を**作業者が実行可能なステップバイステップの手順**に翻訳した文書です。工程設計における最も下流の文書であり、現場との接点になります。

### IATF 16949の要求事項

IATF 16949:2016 条項 **8.5.1.2「標準作業 — 作業者指示書及び目視標準」** では、標準作業文書について以下を確実にすることを要求しています。

| 要求 | 内容 |
|------|------|
| a) 伝達と理解 | 作業の実行に責任を持つ従業員に伝達され、理解されていること |
| b) 判読性 | 判読可能（legible）であること |
| c) 言語 | 作業者が理解できる言語で提示されていること |
| d) アクセス性 | 指定された作業区域でアクセス可能であること |
| e) 安全 | 作業者の安全に関するルールを含むこと |

## 2. 関連文書との関係

### 文書の階層

製造現場で使用される文書には、類似した名称のものが複数あります。企業によって名称・使い分けは異なりますが、一般的な階層を示します。

```
品質マニュアル（組織全体の品質方針）
    │
    ▼
コントロールプラン / QC工程表（工程ごとの管理計画）
    │
    ▼
作業標準書（SOP）（工程の管理条件・品質基準を規定）
    │
    ▼
作業指示書 / 作業手順書（WI）（ステップバイステップの作業手順）
```

### 関連文書の比較

| 観点 | 作業標準書（SOP） | 作業指示書 / 作業手順書（WI） | 検査基準書 |
|------|-------------------|-------------------------------|-----------|
| 英語名 | Standard Operating Procedure | Process Instructions / Work Instructions | Inspection Standard |
| 主な内容 | 工程の管理条件・品質基準 | 個々の作業のステップバイステップ手順 | 検査項目・判定基準・限度見本 |
| 焦点 | 「何を達成するか」 | 「どのように行うか」 | 「何をどう判定するか」 |
| 対象者 | 工程管理者・作業者 | 作業者 | 検査員 |
| 更新頻度 | 工程変更時 | 工程変更・改善時 | 図面改訂・基準変更時 |

> **注意**: 企業によって名称の定義が異なります。「作業標準書」が手順を含む場合もあれば、「作業指示書」が管理条件を含む場合もあります。重要なのはコントロールプランの内容が漏れなく現場文書に展開されていることです。

## 3. 記載項目

### ヘッダー情報

| 項目 | 内容 |
|------|------|
| 文書番号 / 改訂番号 | 文書管理用の一意の識別番号と版数 |
| 品名 / 品番 | 対象製品の名称と部品番号 |
| 工程番号 / 工程名 | コントロールプラン・工程フローチャートと一致する番号 |
| 作成日 / 改訂日 | 初版日と最新改訂日 |
| 作成者 / 承認者 | 作成・承認の責任者名 |
| 適用ライン / 作業エリア | 対象となる製造ライン名や作業区域 |
| 関連文書番号 | コントロールプラン番号、PFMEA番号 |

### 本文（作業内容）

| # | 項目 | 説明 |
|---|------|------|
| 1 | 作業目的 | この工程で何を達成するか（機能・品質要件の概要） |
| 2 | 使用設備・治工具 | 設備名・管理番号、治具・工具のリスト |
| 3 | 使用材料・部品 | 投入する部品・材料の品番・規格 |
| 4 | 保護具（PPE） | 必要な保護具（安全メガネ、手袋、静電気対策等） |
| 5 | 作業手順 | 番号付きのステップバイステップ手順（写真・イラスト付き） |
| 6 | 工程パラメータ | 設備の設定値と許容範囲（トルク、温度、圧力等） |
| 7 | 品質チェックポイント | 管理すべき製品特性と判定基準（規格・公差） |
| 8 | 特殊特性の識別 | CC（◆）/ SC（◇）マークと管理上の注意点 |
| 9 | 検査方法・頻度 | 使用する測定器、サンプルサイズ、検査頻度 |
| 10 | 目視標準 | 良品/不良品の写真比較、限度見本の参照先 |
| 11 | 異常時の処置 | 不良発生・設備異常時の対応手順とエスカレーション先 |
| 12 | 標準作業時間 | サイクルタイム・タクトタイムの目標値 |
| 13 | 安全注意事項 | 危険源・禁止事項・緊急時の対応 |

### コントロールプランからの展開対応

```
コントロールプランのカラム           作業指示書の記載項目
─────────────────             ──────────────

工程番号 / 工程名          ──→  文書のタイトル / 対象工程の識別
設備・治工具              ──→  使用設備・治工具のリスト
製品特性                  ──→  品質チェックポイント（何を管理するか）
工程特性                  ──→  工程パラメータ（設定値と許容範囲）
特殊特性分類（CC/SC）      ──→  特殊特性マーク・特別注意事項として強調表示
仕様・公差                ──→  判定基準・合否基準
評価・測定技法            ──→  検査方法の手順・測定器の使い方
サンプルサイズ / 頻度      ──→  検査頻度（いつ・何個測定するか）
管理方法                  ──→  記録方法・管理図の記入手順
対応計画                  ──→  異常時の処置手順（具体的なステップ）
対応計画責任者            ──→  エスカレーション先（誰に報告するか）
```

## 4. 作成手順

### Step 1: インプットの準備

- コントロールプラン / QC工程表（管理項目・管理方法の定義）
- PFMEA（リスク分析結果、特殊特性の識別）
- 工程フローチャート（工程の全体像）
- 製品図面・仕様書（品質特性の詳細）
- 設備仕様書・治工具設計図（設備の操作方法）
- 類似製品の既存作業指示書（あれば）

### Step 2: 対象工程の選定

コントロールプランの全工程に対して作業指示書を作成しますが、以下の工程を優先します。

| 優先度 | 対象 | 理由 |
|--------|------|------|
| 最優先 | 特殊特性（CC/SC）を含む工程 | 安全・機能に重大な影響。管理方法を確実に実行する必要がある |
| 高 | AP = High の項目を含む工程 | PFMEAで高リスクと判定された工程 |
| 高 | 過去に不良が発生した工程 | 再発防止のために手順の標準化が必要 |
| 中 | 作業者のスキルに依存する工程 | 属人化を防ぎ、品質を安定させる |

### Step 3: 作業手順の分解

対象工程の作業を、作業者が迷わないレベルまで分解します。

**分解の粒度の目安**:
- 1ステップ = 1つの動作または判断
- 初めて作業する人が、この文書だけで作業を再現できるレベル
- 「適切に」「注意して」等の曖昧な表現は避け、具体的な数値・方法で記述

### Step 4: 写真・イラストの追加

各ステップに対応する写真やイラストを追加します。

| 推奨する視覚要素 | 用途 |
|-----------------|------|
| 作業写真 | 正しい作業状態・手の位置・工具の使い方 |
| OK/NG比較写真 | 良品と不良品の比較。目視判定の基準 |
| ポイント指示 | 写真上の矢印・丸印で注意箇所を明示 |
| 工程パラメータ表示 | 設備画面の設定値を示すキャプチャ |

### Step 5: 安全・異常時処置の記載

IATF 16949の要求に従い、安全注意事項を必ず含めます。

**安全注意事項の記載例**:
- 必要なPPE（保護具）の種類
- 危険源とその回避方法
- 禁止事項（やってはいけない操作）
- 緊急時の連絡先・避難経路

### Step 6: レビューと検証

1. **作成者レビュー**: 生産技術が作成した内容を品質保証がレビュー
2. **現場検証**: 実際に作業者に使わせ、手順通りに作業できるかを確認
3. **作業者フィードバック**: 分かりにくい箇所・改善提案を反映
4. **承認**: 関係者の承認を得て正式発行

## 5. 具体例

### インクジェットプリンタ PX-500: 工程No.60 キャリッジ組立 作業指示書

以下は、コントロールプランの工程No.60（キャリッジ組立）に対応する作業指示書の記載例です。

```
─────────────────────────────────────────────────────────────
作業指示書

文書番号: WI-PX500-060-R02          改訂日: 2026/01/15
品名: インクジェットプリンタ PX-500    品番: PX-500A Rev.B
工程番号: 60                          工程名: キャリッジ組立
適用ライン: プリンタ組立ライン #1
関連CP: CP-PX500-R01（特性No.5, 6）   関連PFMEA: FM-PX500-R01

作成: 生産技術 田中    承認: 製造課長 山田
─────────────────────────────────────────────────────────────

■ 作業目的
  印字ヘッドキャリッジをガイドレールに取付け、
  所定の位置精度（±0.05mm）に調整する。

■ 特殊特性
  ◆CC: ヘッドアライメント（位置ずれ量 ±0.05mm）
  ◇SC: キャリッジ摺動抵抗（摺動荷重 1.0±0.3 N）

■ 使用設備・治工具
  ・電動ドライバー EQ-060（トルク設定: 0.5 Nm）
  ・レーザー変位計 M-061
  ・キャリッジ組立治具 JG-060
  ・プッシュプルゲージ（2h毎の摺動抵抗確認用）

■ 使用材料・部品
  ・印字ヘッドユニット（部品番号: HD-500）
  ・キャリッジフレーム（部品番号: CF-500）
  ・ガイドレール（部品番号: GR-500）
  ・固定ネジ M3×8（4本）
  ・FFC（フレキシブルフラットケーブル）

■ 保護具（PPE）
  ・静電気対策リストストラップ【必須】
  ・静電気対策手袋【必須】
  ・保護メガネ

■ 安全注意事項
  ⚠ 印字ヘッドは静電気に弱い精密部品です。
    必ずリストストラップを装着してから取り扱ってください。
  ⚠ ヘッドを素手で触らないでください（ノズル面の汚損防止）。
  ⚠ ヘッドの落下に注意。専用トレーから直接治具へ移載すること。

■ 標準作業時間
  サイクルタイム: 90秒 / 台

─────────────────────────────────────────────────────────────
■ 作業手順
─────────────────────────────────────────────────────────────

  Step 1: 部品の準備・確認
  ─────────────────────
  1-1. 静電気対策リストストラップを装着し、接地を確認する。
  1-2. キャリッジフレーム (CF-500) をロットラベルで品番確認する。
  1-3. ガイドレール (GR-500) をエアブローで清掃する。
       【ポイント】異物付着は摺動不良の原因になる。

  Step 2: ガイドレールの取付
  ─────────────────────
  2-1. キャリッジフレームを組立治具 JG-060 にセットする。
  2-2. ガイドレールを治具の位置決めピンに合わせて載せる。
  2-3. 固定ネジ M3×8（4本）を電動ドライバー EQ-060 で締付ける。
       【工程パラメータ】締付けトルク: 0.5 Nm（電動ドライバー設定済み）
  2-4. 電動ドライバーのトルク到達表示（緑ランプ）を確認する。
       NG（赤ランプ）の場合 → 異常時処置へ。

  Step 3: 印字ヘッドの搭載
  ─────────────────────
  3-1. 専用トレーからヘッドユニット (HD-500) を取り出す。
       【注意】◆ 落下・衝撃厳禁。専用治具で移載すること。
  3-2. ヘッドをキャリッジのガイドに沿って挿入する。
  3-3. ヘッド固定レバーを「ロック」位置まで押し下げる。
       【チェック】クリック音を確認すること。
  3-4. FFCコネクタをヘッド側コネクタに挿入する。
       【チェック】◇ コネクタのロックタブが「カチッ」と嵌合すること。

  Step 4: アライメント測定 ◆CC
  ─────────────────────
  4-1. レーザー変位計 M-061 の電源を入れ、ゼロリセットする。
  4-2. 測定プログラム「PX500-ALIGN」を選択し、START を押す。
  4-3. 自動測定結果を確認する。
       【判定基準】位置ずれ量: ±0.05mm 以内 → 合格（緑表示）
  4-4. 合格 → Step 5 へ進む。
       不合格 → 異常時処置へ。

  Step 5: 摺動確認
  ─────────────────────
  5-1. キャリッジを手動で左右にスライドさせ、異音・引っ掛かりがないことを確認する。
  5-2. 【2時間毎の定期確認】プッシュプルゲージで摺動荷重を測定する。
       【判定基準】◇ 摺動荷重: 1.0±0.3 N 以内 → 合格
       検査結果を検査記録シート WI-060-REC に記入する。

  Step 6: 完了処理
  ─────────────────────
  6-1. レーザー変位計の測定結果が自動記録されていることを確認する。
  6-2. 組立治具からワークを取り外す。
  6-3. 次工程（工程70: 基板実装）の搬送トレーに載せる。

─────────────────────────────────────────────────────────────
■ 異常時の処置
─────────────────────────────────────────────────────────────

  【トルク到達NG（Step 2-4）】
  1. 作業を停止する。
  2. 組立班長に報告する。
  3. ネジの緩みを確認し、再締付けを行う。
  4. 再度トルク到達表示を確認する。
  5. 再NG → 電動ドライバーの校正状態を確認。設備保全に連絡。

  【アライメント不合格（Step 4-4）】◆CC
  1. 作業を停止する。
  2. 組立班長に報告する。
  3. ガイドレール固定ネジを緩め、レール位置を再調整する。
  4. 再締付け後、レーザー変位計で再測定する。
  5. 再測定合格 → 作業再開。
  6. 再測定NG → 前回良品確認以降のワークを隔離。
     生産技術に連絡し、治具・設備の点検を依頼する。

  【ヘッド破損・落下（Step 3）】◆CC
  1. 作業を停止する。
  2. 組立班長に報告する。
  3. 該当ヘッドを不良品置場に隔離する（赤札貼付）。
  4. 新しいヘッドで作業を再開する。
  5. 品質記録に不良内容を記入する。

─────────────────────────────────────────────────────────────
■ 改訂履歴
─────────────────────────────────────────────────────────────
  R01 (2025/10/01): 初版発行
  R02 (2026/01/15): ヘッド取出し専用治具の導入に伴い
                     Step 3 を改訂（PFMEA Step 6 最適化反映）
─────────────────────────────────────────────────────────────
```

| 責任者 | 組立班長 | 異常時の報告先として記載 |

## 6. よくある課題と対策

### 課題1: 作業指示書が読まれない・使われない

**問題**: せっかく作成しても、現場で参照されず形骸化している。

**対策**:
- **視覚的に分かりやすくする**: 文字ばかりの文書は敬遠される。写真・イラスト・色分けを多用する
- **作業区域に掲示する**: IATF 16949の要求通り、作業場所でアクセス可能にする（タブレット表示、ラミネート掲示等）
- **現場の言語で書く**: 専門用語を避け、作業者が日常的に使う言葉で記述する
- **1工程1文書の原則**: 文書が長すぎると使われない。1つの工程に対して1つの文書にまとめる

### 課題2: コントロールプランとの不整合

**問題**: CPの管理方法が変更されたが、作業指示書が更新されていない。

**対策**:
- CPの改訂トリガーと作業指示書の改訂を**連動させるルール**を定める
- 文書管理台帳でCP番号と作業指示書番号の紐付けを管理する
- CP改訂時のチェックリストに「関連する作業指示書の改訂」を含める
- 文書管理システム（DMS）で版数管理を徹底する

### 課題3: 手順が曖昧・属人的

**問題**: 「適切に締付ける」「注意して取り扱う」等の曖昧な記述が多く、作業者によって解釈が異なる。

**対策**:
- **数値で規定する**: 「適切に締付ける」→「トルク 0.5 Nm で締付ける」
- **具体的な動作で記述する**: 「注意して取り扱う」→「専用トレーから直接治具へ移載する。手で直接ノズル面に触れない」
- **判定基準を明示する**: 「問題ないことを確認する」→「位置ずれ量が±0.05mm以内であることを確認する」

### 課題4: 異常時処置が不十分

**問題**: 正常時の手順は書いてあるが、異常時の対応が「班長に報告」だけ。

**対策**:
- コントロールプランのReaction Planを**ステップバイステップの手順**として展開する
- 「誰に」「何を」「どのように」報告するかを具体的に記載する
- 不良品の識別方法（赤札等）と隔離場所を明示する
- 生産再開の条件（初物確認の数量・合格基準）を記載する

### 課題5: 改訂管理が追いつかない

**問題**: 工程改善・設備変更が頻繁に発生し、作業指示書が最新状態に保てない。

**対策**:
- 改訂トリガーを明確に定義する（設計変更、工程変更、不良対策、PFMEA更新時等）
- **電子化による版数管理**: 紙ベースからタブレット表示に移行し、最新版を常に配信する
- 変更点を**赤枠・マーカー**で強調し、作業者が差分を把握しやすくする
- 改訂時に作業者への教育（変更点の説明）を必ず実施する

## 7. ベストプラクティス

1. **ビジュアル重視で作成する**: 文字よりも写真・イラストで伝える。OK/NG比較写真、ポイント矢印を活用する
2. **1工程1文書にまとめる**: 長い文書は使われない。1つの工程に対して必要十分な情報を1文書にまとめる
3. **作業者と一緒に作成する**: 生産技術だけで作成せず、実際の作業者のフィードバックを反映する
4. **特殊特性を一貫して識別する**: PFMEA → CP → 作業指示書で同じ◆/◇マークを使い、一目で重要管理項目と分かるようにする
5. **異常時処置を充実させる**: 正常手順以上に、異常時の具体的な対応手順を詳しく書く
6. **Living Document として運用する**: 現場の改善提案を反映し、継続的に文書を改善する仕組みを設ける
7. **教育と連動させる**: 作業指示書の新規発行・改訂時には、必ず作業者への教育訓練を実施し、理解度を確認する
8. **電子化を推進する**: タブレット端末での閲覧により、最新版の自動配信・動画埋め込み・多言語対応が可能になる
