# 工程設計書（Process Design Document）

## 1. 概要

### 工程設計書とは

工程設計書（Process Design Document）とは、工程設計（Phase 3）で作成したすべての成果物を**総括・統合する上位文書**です。工程フローチャート・PFMEA・コントロールプラン・設備仕様書・治工具設計図・フロアプランレイアウト・梱包仕様書等の個別成果物を横断的に結びつけ、工程設計の全体像と設計根拠を一つの文書に集約します。

工程設計書は、工程設計の完了時に発行される**総括文書（Summary Document）** であり、Phase 4（製品・工程の妥当性確認）に進む前のデザインレビュー（DR）の基本資料として使用されます。

### 目的

| 目的 | 説明 |
|------|------|
| 工程設計の全体総括 | 個別成果物を横断的にまとめ、工程設計の全体像を一元的に把握する |
| 設計根拠の記録 | 工程・設備・治工具・梱包等の設計判断の根拠（なぜこの方式を選んだか）を記録する |
| デザインレビューの基礎資料 | Phase 3完了DR（工程設計DR）の審議資料として使用する |
| Phase 4への引継ぎ | 量産トライアル・MSA・工程能力調査の実施計画を明確にする |
| 変更管理の基準 | 量産後の工程変更時に元の設計意図・制約条件を参照する基準となる |
| 監査への対応 | IATF 16949の認証監査・顧客監査で工程設計のエビデンスを示す |

### 工程設計における位置づけ

```
Phase 3: 工程設計・開発

  ① 工程フローチャート ─┐
  ② PFMEA ─────────┤
  ③ コントロールプラン ──┤
  ④ 特性マトリクス ────┤
  ⑤ QC工程表 ─────────┤
  ⑥ 作業指示書 ────────┤    ┌──────────────────────┐
  ⑦ 設備仕様書 ────────┼──→│  工程設計書（総括文書） │ ◀ 本記事
  ⑧ 治工具設計図 ───────┤    │  全成果物を統合・総括  │
  ⑨ フロアプランレイアウト ┤    └──────────┬───────────┘
  ⑩ 梱包仕様書 ────────┘                │
                                         ▼
                              デザインレビュー（Phase 3 DR）
                                         │
                                         ▼
                              Phase 4: 製品・工程の妥当性確認
                                （MSA、工程能力調査、PPAP）
```

工程設計書は、工程設計の最終成果物として全ての個別文書を束ねる役割を持ちます。個別成果物の「サマリーと相互関係」を記述し、設計の全体整合性を保証します。

### IATF 16949の要求事項

| 条項 | 内容 |
|------|------|
| **8.3.4.3** 製造工程の設計からのアウトプット | 工程フローチャート、PFMEA、コントロールプラン、作業指示書、工程承認基準等を含む工程設計のアウトプットを文書化すること |
| **8.3.4.4** 製品承認プロセス | 顧客の要求する製品・製造工程承認手続きに適合していること |
| **8.3.5** 設計・開発からのアウトプット | 設計・開発のアウトプットが、監視・測定の要求事項を含み、受入基準を規定していること |

## 2. 工程設計書の構成

### 文書構成の全体像

工程設計書は以下のセクションで構成されます。

| セクション | 内容 | 主な参照文書 |
|-----------|------|------------|
| A. 表紙・改訂履歴 | 文書管理情報 | - |
| B. 工程設計の前提条件 | 製品仕様・生産条件・制約条件 | 製品図面、生産計画 |
| C. 工程概要 | 全工程の一覧と各工程の概要 | 工程フローチャート |
| D. リスク分析の総括 | PFMEA結果の要約と残留リスク | PFMEA |
| E. 品質管理計画の総括 | 特殊特性・管理方法の要約 | コントロールプラン、QC工程表 |
| F. 設備・治工具計画 | 設備・治工具の一覧と調達計画 | 設備仕様書、治工具設計図 |
| G. レイアウト計画 | 生産ラインの配置概要 | フロアプランレイアウト |
| H. 梱包・物流計画 | 梱包方法と物流要件 | 梱包仕様書 |
| I. 要員・教育計画 | 必要人員と教育訓練計画 | 作業指示書 |
| J. Phase 4 移行計画 | 量産トライアル・MSA・工程能力調査の実施計画 | - |
| K. 課題・リスク管理 | 未解決の課題とリスク対策 | - |
| L. 成果物一覧・文書マトリクス | 全成果物の文書番号と最新版数 | 全文書 |

## 3. 各セクションの記載項目

### A. 表紙・改訂履歴

| 項目 | 内容 |
|------|------|
| 文書番号 / 版数 | 工程設計書の識別番号と改訂版数 |
| 対象製品 | 製品名・品番・仕向地 |
| プロジェクト名 | 開発プロジェクトの名称 |
| 対象工場 / ライン | 生産する工場・ラインの識別 |
| 作成日 / 改訂日 | 初版日と最新改訂日 |
| 作成者 / 承認者 | 作成・レビュー・承認の責任者 |
| 配布先 | 工程設計書の配布先（関係部門） |
| 改訂履歴 | 改訂内容の記録 |

### B. 工程設計の前提条件

| 項目 | 内容 |
|------|------|
| 製品仕様の要約 | 主要な製品仕様・機能・性能要件の要約 |
| 特殊特性一覧 | CC（◆）/ SC（◇）の一覧とその管理目標 |
| 生産条件 | 計画生産量、タクトタイム、シフト体制、稼働日数 |
| 品質目標 | 工程内不良率目標、市場品質目標、Cpk目標値 |
| 投資予算 | 設備・治工具への投資予算 |
| 制約条件 | 建屋制約、既存設備の活用条件、納期制約 |
| 適用規格 | IATF 16949、顧客固有の要求事項、適用JIS |

### C. 工程概要

工程フローチャートの内容を要約し、各工程の概要を記述します。

| 項目 | 内容 |
|------|------|
| 工程フロー全体図 | 受入から出荷までの全工程のフロー図（簡略版） |
| 工程数 | 全工程数と工程分類（加工・組立・検査・搬送等） |
| 各工程の概要 | 工程番号・工程名・作業内容・主要設備を一覧で記述 |
| 工程間の搬送 | 工程間の搬送方式（コンベア・台車・手渡し等） |
| ボトルネック工程 | サイクルタイム上のボトルネック工程と対策 |

### D. リスク分析の総括

PFMEAの結果を要約し、経営層やデザインレビュー参加者が全体像を把握できるようにします。

| 項目 | 内容 |
|------|------|
| PFMEA実施範囲 | 対象とした工程の範囲 |
| 高リスク項目の要約 | AP = High の項目一覧と講じた対策 |
| 特殊特性に対するリスク | CC/SC特性に関連する故障モードと対策 |
| 残留リスク | 対策後もリスクが残る項目と今後の対応方針 |
| ポカヨケ一覧 | 治工具・設備に組み込んだポカヨケの一覧 |

### E. 品質管理計画の総括

| 項目 | 内容 |
|------|------|
| 管理特性の全体像 | コントロールプランで管理する特性の総数と分類 |
| 特殊特性の管理方法 | CC/SC特性ごとの管理方法の要約（全数検査、SPC等） |
| 検査方式 | 全数検査 / 抜取検査の工程別一覧 |
| 測定設備 | 使用する測定設備の一覧とMSA実施計画 |
| SPC適用工程 | SPC（管理図）を適用する工程と管理特性の一覧 |
| CP/QC工程表の整合 | コントロールプランとQC工程表の相互関係の確認 |

### F. 設備・治工具計画

| 項目 | 内容 |
|------|------|
| 設備一覧 | 全工程で使用する設備の一覧（設備名、管理番号、メーカー、仕様概要） |
| 治工具一覧 | 全工程で使用する治工具の一覧（治工具名、管理番号、種類） |
| 新規設備 / 既存流用 | 新規導入設備と既存設備の流用の区分 |
| 調達計画 | 設備・治工具の発注～納入～検収のスケジュール |
| 投資金額 | 設備・治工具の投資金額の内訳と合計 |
| FAT/SAT計画 | 設備の工場検収・現地検収の実施計画 |

### G. レイアウト計画

| 項目 | 内容 |
|------|------|
| レイアウト形式 | 採用したレイアウト形式（U字型等）とその選定理由 |
| レイアウト図 | フロアプランレイアウトの縮小版 |
| 占有面積 | 生産ラインの占有面積 |
| ユーティリティ | 電源・エアー・排気等のユーティリティ計画 |
| 安全設計 | 安全柵・非常停止・避難経路の概要 |

### H. 梱包・物流計画

| 項目 | 内容 |
|------|------|
| 梱包仕様の概要 | 一次・二次・三次梱包の要約 |
| 輸送試験結果 | 梱包設計の検証結果の要約 |
| 物流計画 | 出荷頻度・輸送手段・物流コストの概要 |

### I. 要員・教育計画

| 項目 | 内容 |
|------|------|
| 必要人員 | 工程別の必要作業者数とスキル要件 |
| 多能工化計画 | 作業者の多能工育成計画 |
| 教育訓練計画 | 作業指示書に基づく教育訓練のスケジュール |
| 力量認定基準 | 各工程の作業者に求める力量と認定方法 |

### J. Phase 4 移行計画

Phase 3（工程設計）からPhase 4（妥当性確認）への移行に必要な活動を計画します。

| 項目 | 内容 |
|------|------|
| 量産トライアル計画 | 量産条件での試作回数・数量・評価項目 |
| MSA実施計画 | 測定システム解析の対象設備・スケジュール |
| 工程能力調査計画 | Cp/Cpk調査の対象特性・サンプル数・目標値 |
| PPAP提出計画 | PPAP提出の対象要素・スケジュール |
| Phase 3 DR判定基準 | デザインレビューの合格基準と判定方法 |

### K. 課題・リスク管理

| 項目 | 内容 |
|------|------|
| 未解決課題 | 工程設計段階で解決に至らなかった技術課題 |
| リスク項目 | Phase 4以降で顕在化するリスクと予防措置 |
| 対応計画 | 各課題・リスクに対する担当者・期限・対応方針 |

### L. 成果物一覧・文書マトリクス

全ての個別成果物の文書番号・版数・発行日を一覧で管理し、文書間の相互参照を整理します。

| 項目 | 内容 |
|------|------|
| 成果物一覧表 | 全文書の番号・版数・発行日・作成者を一覧化 |
| 文書相互参照マトリクス | 文書間の参照関係を整理したマトリクス |
| 変更影響マトリクス | 1つの文書が変更された場合に影響を受ける文書の関係 |

## 4. 作成手順

### Step 1: 個別成果物の完成確認

すべての個別成果物が完成し、レビュー・承認が完了していることを確認します。

**完成確認チェックリスト**:

| # | 成果物 | 文書番号 | 版数 | 承認済み | 確認者 |
|---|--------|---------|------|---------|--------|
| 1 | 工程フローチャート | | | □ | |
| 2 | PFMEA | | | □ | |
| 3 | コントロールプラン | | | □ | |
| 4 | 特性マトリクス | | | □ | |
| 5 | QC工程表 | | | □ | |
| 6 | 作業指示書（全工程分） | | | □ | |
| 7 | 設備仕様書（全設備分） | | | □ | |
| 8 | 治工具設計図（全治工具分） | | | □ | |
| 9 | フロアプランレイアウト | | | □ | |
| 10 | 梱包仕様書 | | | □ | |

### Step 2: 文書間の整合性確認

個別成果物間の整合性を確認します。これは工程設計書を作成する上で最も重要なステップです。

**整合性確認マトリクス**:

| 確認項目 | 確認内容 |
|---------|---------|
| 工程フローチャート ↔ PFMEA | 全工程がPFMEAの分析対象に含まれているか |
| PFMEA ↔ コントロールプラン | PFMEAの高リスク項目がCPの管理対象に含まれているか |
| コントロールプラン ↔ 作業指示書 | CPの全管理項目が作業指示書に展開されているか |
| コントロールプラン ↔ QC工程表 | CPとQC工程表の管理項目が整合しているか |
| 設備仕様書 ↔ コントロールプラン | 設備の精度がCPの管理精度要件を満たしているか |
| 治工具設計図 ↔ PFMEA | PFMEAのポカヨケ要件が治工具に実装されているか |
| フロアプラン ↔ 工程フローチャート | 設備の配置順序が工程の順序と一致しているか |
| 梱包仕様書 ↔ PFMEA | 輸送中のリスクが梱包設計で対策されているか |

### Step 3: 各セクションの記述

セクションA～Lの内容を、個別成果物を参照しながら記述します。

**記述のポイント**:
- 個別成果物の内容をそのまま転記するのではなく、**要約・総括**として記述する
- 設計判断の**根拠（なぜこの方式を選んだか）** を記録する
- 数値（生産条件、品質目標、投資金額等）は正確に記載する
- 文書番号・版数を明記し、参照元を追跡可能にする

### Step 4: デザインレビューの準備

工程設計書を基にPhase 3完了DRを実施します。

| DR参加者 | 役割 |
|---------|------|
| プロジェクトリーダー | DR全体の進行、合否判定の総括 |
| 生産技術 | 工程設計の説明（設計者として） |
| 品質保証 | 品質管理計画の妥当性確認 |
| 製造 | 製造可能性・作業性の評価 |
| 購買 | 設備・治工具・梱包材の調達状況確認 |
| 設計（製品設計） | 製品仕様との整合確認 |
| 経営層 | 投資判断、リスク承認 |

**DR判定基準の例**:

| 判定 | 条件 |
|------|------|
| **合格（Go）** | 全成果物が承認済み、未解決の重大課題なし、Phase 4移行計画が確定 |
| **条件付き合格** | 軽微な未解決課題があるが、Phase 4開始までに解決見込み |
| **不合格（No-Go）** | 重大な未解決課題がある、設計変更が必要、再DRが必要 |

### Step 5: 発行と配布

DR合格後、工程設計書を正式に発行し、関係部門に配布します。

## 7. よくある課題と対策

### 課題1: 工程設計書が作成されない

**問題**: 個別成果物は作成されるが、それらを総括する工程設計書が作成されず、全体の整合性が確認されない。

**対策**:
- APQPのPhase 3完了DRに工程設計書の提出を必須とするプロセスを定める
- 工程設計書のテンプレートを標準化し、作成負荷を軽減する
- 工程設計書をプロジェクトマイルストーンの成果物として定義する

### 課題2: 個別成果物間の不整合

**問題**: 工程フローチャート・PFMEA・コントロールプラン・作業指示書間で工程番号や管理項目に不整合がある。

**対策**:
- 工程設計書作成時に「文書間の整合性確認マトリクス」（Step 2）を必ず実施する
- 工程番号・特性番号を全文書で統一する命名規則を定める
- 文書管理システム（DMS）で文書間のリンクを管理する

### 課題3: 設計根拠が記録されない

**問題**: 「なぜこの設備を選んだか」「なぜU字型レイアウトにしたか」等の設計判断の根拠が残らず、後で変更時に元の意図が不明になる。

**対策**:
- 工程設計書の各セクションに「選定理由」「設計根拠」の記述欄を設ける
- 比較検討した代替案とその評価結果も記録する
- DRの議事録を工程設計書に添付する

### 課題4: Phase 4移行計画が不明確

**問題**: 量産トライアル・MSA・工程能力調査の計画が曖昧で、Phase 4の活動がスムーズに開始できない。

**対策**:
- 工程設計書のセクションJ（Phase 4移行計画）で具体的なスケジュール・サンプル数・合格基準を明記する
- MSA・工程能力調査の対象特性を特殊特性一覧と紐付けて計画する
- PPAP提出のタイムラインを顧客と事前に合意する

### 課題5: 量産後に工程設計書が更新されない

**問題**: 量産開始後の工程変更時に工程設計書が改訂されず、現状と乖離する。

**対策**:
- 工程変更管理プロセスに工程設計書の改訂要否判断を含める
- 重大な工程変更（設備変更、工程追加・削除等）時には工程設計書を必ず改訂する
- 定期的な文書棚卸しで最新版の維持を確認する

## 8. ベストプラクティス

1. **個別成果物の完成後に作成する**: 工程設計書は総括文書であるため、個別成果物がすべて完成してから作成を開始する
2. **文書間の整合性確認を最重要視する**: 整合性確認マトリクスを使い、文書間の不整合を体系的に検出・解消する
3. **設計根拠を必ず記録する**: 設備選定理由、レイアウト選定理由、管理方法の決定理由等を文書に残す
4. **DRの基礎資料として活用する**: Phase 3 DRの議題構成を工程設計書のセクションに合わせ、レビューの効率を高める
5. **Phase 4への引継ぎを明確にする**: MSA・工程能力調査・PPAPの実施計画を具体的に記述し、Phase 4の活動をスムーズに開始する
6. **テンプレートを標準化する**: 製品ごとに一から作成するのではなく、標準テンプレートを用いて作成負荷を軽減する
7. **電子化して版数管理を徹底する**: 文書管理システムで全成果物の最新版を一元管理し、参照の利便性を高める
8. **未解決課題を正直に記録する**: 工程設計段階で解決できなかった課題を隠さず記録し、Phase 4での対応計画を明確にする
