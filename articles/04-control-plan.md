# コントロールプラン（Control Plan）

## 1. 概要

### コントロールプランとは

コントロールプラン（Control Plan / CP）とは、製品と工程のばらつきを最小化するために、各工程での管理方法・測定方法・サンプリング計画・異常時対応を体系的に記述した文書です。

IATF 16949（条項 8.5.1.1）で作成が要求されており、自動車産業における品質管理の中核文書です。PFMEAで特定されたリスクへの対策を具体的な管理活動として落とし込む役割を担います。

> **APQP 3rd Edition（2024年）** での定義: 「部品および工程の管理に必要な管理システムの文書化された記述」

### 目的

| 目的 | 説明 |
|------|------|
| ばらつきの最小化 | 工程パラメータと製品特性のばらつきを管理し、規格内に維持する |
| リスク対策の実行 | PFMEAで特定した高リスク項目への管理方法を具体化する |
| 管理体制の標準化 | 「何を、どの方法で、どの頻度で管理するか」を統一する |
| 異常時対応の明確化 | 規格逸脱時の対応計画（Reaction Plan）を事前に定める |
| 顧客要求への適合 | IATF 16949・顧客固有要求事項（CSR）への適合を証明する |
| PPAPの必須要素 | PPAP（生産部品承認プロセス）の18要素の1つとして提出する |

### 工程設計における位置づけ

```
工程フローチャート
    │
    ▼
PFMEA（リスク分析）──→ リスクの高い項目を特定
    │
    ▼
コントロールプラン ──→ リスク対策を具体的な管理活動に展開
    │
    ├──→ 作業指示書（現場作業者向けの手順）
    └──→ 検査基準書（検査員向けの判定基準）
```

コントロールプランは、PFMEAの「出口」であり、作業指示書の「入口」です。PFMEAで分析した膨大なリスク情報を、現場で実行可能な管理活動として凝縮した文書といえます。

## 2. 3段階のコントロールプラン

APQP体系では、製品開発の段階に応じて**3段階**のコントロールプランを作成します。

| 段階 | 名称 | 英語 | 適用時期 | 特徴 |
|------|------|------|---------|------|
| 1 | 試作CP | Prototype CP | 試作段階 | 試作品の寸法測定・材料試験・性能試験を中心に管理 |
| 2 | 量産試作CP | Pre-Launch CP | パイロット生産段階 | 量産設備を使用した試作。試作CPより管理項目・頻度を強化 |
| 3 | 量産CP | Production CP | 量産段階 | 正式量産の管理計画。工程能力に基づき管理頻度を最適化 |

### 各段階の違い

```
試作CP              量産試作CP            量産CP
──────              ────────              ──────
管理項目: 少         管理項目: 最多         管理項目: 最適化
頻度: 全数が多い     頻度: 高頻度           頻度: 工程能力に応じて設定
設備: 試作設備       設備: 量産設備         設備: 量産設備
目的: 設計の妥当性   目的: 工程の妥当性     目的: 量産時の安定管理
      確認                確認
```

### セーフローンチ（Safe Launch）

APQP 3rd Edition（2024年）では、量産初期に追加の監視措置を行う**セーフローンチ**の概念が追加されました。

| 項目 | 内容 |
|------|------|
| 適用時期 | 量産開始直後の一定期間（通常90日間程度） |
| 目的 | 量産初期の品質リスクを低減する |
| 主な活動 | 検査頻度の引き上げ、追加の検査項目、出荷前の強化確認 |
| 終了条件 | 工程が安定し、工程能力目標を達成した時点 |

セーフローンチの管理項目は量産CPに追記するか、別紙で管理します。終了条件を満たした後、通常の量産CPに移行します。

## 4. PFMEAからの展開方法

コントロールプランはPFMEAの結果を反映して作成します。展開の流れを示します。

### 展開の考え方

PFMEAで分析した全項目のうち、**管理が必要な項目**をコントロールプランに展開します。

```
PFMEA                              コントロールプラン
─────                              ─────────────

工程ステップ            ──→        工程番号 / 工程名
故障モードの影響        ──→        （管理の必要性の根拠）
故障の原因              ──→        工程特性（管理すべきパラメータ）
現行予防管理            ──→        管理方法
現行検出管理            ──→        評価・測定技法 / サンプルサイズ / 頻度
AP（Action Priority）   ──→        管理の厳格さの判断基準
推奨処置の結果          ──→        改善後の管理方法に反映
```

### 具体的な展開例

**PFMEAの分析結果**:

| 項目 | 内容 |
|------|------|
| 工程ステップ | 60: キャリッジ組立 |
| 機能/要求 | 印字ヘッドキャリッジを所定の位置精度（±0.05mm）に調整する |
| 故障モード | ヘッド位置ずれ |
| 故障の影響 | 印字品質不良（罫線ずれ）→ 顧客クレーム |
| 重大度 (S) | 8 |
| 故障の原因 | レール取付位置のばらつき |
| 現行予防管理 | 組立治具による位置決め |
| 発生度 (O) | 4 |
| 現行検出管理 | 印字テストでアライメント確認 |
| 検出度 (D) | 3 |
| AP | High |

**コントロールプランへの展開**:

| カラム | 記載内容 |
|--------|---------|
| 工程番号 | 60 |
| 工程名 | キャリッジ組立 |
| 設備 | 電動ドライバー (EQ-060) / レーザー変位計 (M-061) |
| 製品特性 | ヘッドアライメント（位置ずれ量） |
| 工程特性 | ネジ締付けトルク、レール取付位置 |
| 特殊特性分類 | CC（◆） |
| 仕様・公差 | ±0.05 mm |
| 測定技法 | レーザー変位計 |
| サンプルサイズ | 全数 |
| サンプル頻度 | 全数 |
| 管理方法 | 自動測定記録 |
| 対応計画 | ライン停止 → レール再調整 → 再測定合格確認後に再開 |
| 対応計画責任者 | 組立班長 |

### AP（Action Priority）と管理の厳格さ

| AP | 管理の厳格さ |
|----|-------------|
| High | 全数検査または高頻度の抜取検査。管理図による常時監視。ポカヨケの導入検討 |
| Medium | 定期的な抜取検査。チェックシートによる記録 |
| Low | 標準的な管理。ロット初物確認など |

## 6. Reaction Plan（対応計画）の書き方

対応計画はコントロールプランの中でも特に重要な項目です。規格逸脱が発生した際に、品質問題の拡大を防ぐための手順を具体的に記述します。

### 記載すべき内容

| 要素 | 質問 | 例 |
|------|------|-----|
| 即時処置 | 何をまず行うか？ | 生産停止、設備停止 |
| 報告 | 誰に報告するか？ | 製造班長、品質担当 |
| 封じ込め | 疑わしい製品をどうするか？ | 前回良品確認以降の製品を隔離・選別 |
| 原因調査 | どのように原因を特定するか？ | 設備パラメータ確認、金型点検 |
| 是正処置 | どのように再発を防ぐか？ | 金型交換、パラメータ再設定 |
| 再開条件 | どうなったら生産を再開するか？ | 初物3個が規格内、班長承認 |
| 出荷判定 | 影響範囲の製品をどう処理するか？ | 全数選別後、合格品のみ出荷 |
| 顧客通知 | 顧客への通知は必要か？ | 出荷済み品に影響がある場合は通知 |

### 良い対応計画の例

```
【良い例】
生産停止 → 製造班長に報告 → 前回良品確認時点以降の
製品を隔離 → 全数選別検査実施 → 金型摩耗確認・
必要に応じ交換 → 初物3個合格確認後に生産再開 →
選別結果を品質担当に報告
```

```
【悪い例】
班長に報告する
```

## 7. よくある課題と対策

### 課題1: PFMEAとコントロールプランの不整合

**問題**: PFMEAで特定した管理方法がコントロールプランに正しく反映されていない。

**対策**:
- PFMEAとコントロールプランを**同時に作成・レビュー**する
- PFMEA改訂時にコントロールプランも必ず連動して更新する
- 工程番号・特性番号でトレーサビリティを確保する

### 課題2: 対応計画が形式的

**問題**: Reaction Planが「班長に報告」だけで、具体的な処置手順がない。

**対策**:
- 前述の「記載すべき内容」の各要素を網羅する
- 実際に異常が発生した際のシミュレーションを行い、対応計画が機能するか確認する
- APQP 3rd Editionで追加された「対応計画責任者」カラムを活用する

### 課題3: 3段階のCPが適切に移行されない

**問題**: 量産試作CPのまま量産に入り、管理が過剰または不足になる。

**対策**:
- 各段階のCP移行条件を事前に定義する
- Phase 4（妥当性確認）の結果に基づいて量産CPに移行する
- 工程能力（Cpk）の結果に応じて、管理頻度を最適化する

### 課題4: 管理方法が測定のみに偏っている

**問題**: 検査（検出管理）ばかりで、予防管理（ポカヨケ、自動制御）が不足している。

**対策**:
- AIAG-VDA FMEAの考え方に従い、**予防管理を検出管理より優先**する
- ポカヨケ（Poka-Yoke）の導入を積極的に検討する
- SPC（統計的工程管理）による工程監視を取り入れる

### 課題5: 改訂管理が追いつかない

**問題**: 工程変更や設計変更が頻繁に発生し、CPの改訂が遅れる。

**対策**:
- CPの改訂トリガー（設計変更、工程変更、顧客クレーム、内部不良等）を明確に定義する
- 変更管理プロセスにCPの改訂を組み込む
- 文書管理システム（DMS）で版数管理を徹底する

## 8. APQP 3rd Edition（2024年）での主な変更点

| 項目 | 2nd Edition | 3rd Edition |
|------|-------------|-------------|
| 対応計画責任者 | カラムなし | **新規カラム追加** |
| セーフローンチ | 概念のみ | **正式に体系化** |
| CP段階 | 3段階 | 3段階 + セーフローンチ |
| PFMEA連携 | 推奨 | **AIAG-VDA FMEAのAP（Action Priority）との連携を強化** |
| デジタル化 | 言及なし | デジタルツールの活用を推奨 |
