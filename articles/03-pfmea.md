# PFMEA（工程FMEA / Process FMEA）

## 1. 概要

### PFMEAとは

PFMEA（Process Failure Mode and Effects Analysis / 工程FMEA）とは、製造工程における**潜在的な故障モード**を体系的に特定し、その影響・原因を分析して、リスクを事前に低減するための手法です。

AIAG-VDA FMEAハンドブック（2019年）では以下のように定義されています。

> 「PFMEAは、製造・組立工程の潜在的な故障モードを分析し、製品の品質・安全性・信頼性に対するリスクを特定し、適切な管理方法を設定するための体系的なアプローチである」

### 目的

| 目的 | 説明 |
|------|------|
| リスクの事前特定 | 量産前に工程上の潜在リスクを洗い出す |
| 不良の未然防止 | 予防管理と検出管理を計画し、不良品の流出を防ぐ |
| コントロールプランへの展開 | 分析結果を管理計画に反映し、効果的な工程管理を実現する |
| 優先順位付け | 限られたリソースを高リスク項目に集中させる |
| 知見の蓄積 | 過去の不良経験やノウハウをチームで共有・文書化する |
| 継続的改善 | 量産後も新たなリスク情報を反映し、工程を改善し続ける |

### 工程設計における位置づけ

```
工程フローチャート ──→ PFMEAの分析対象を定義（工程構造）
    │
    ▼
PFMEA ──→ 各工程の故障モード・影響・原因を分析し、管理方法を設定
    │
    ▼
コントロールプラン ──→ PFMEAの管理方法を工程管理計画として文書化
    │
    ▼
作業指示書 ──→ コントロールプランの内容を現場手順に展開
```

PFMEAは工程フローチャートの**各ステップに対してリスク分析を行う**文書であり、コントロールプランの**インプット**となります。

### DFMEAとPFMEAの違い

| 観点 | DFMEA（設計FMEA） | PFMEA（工程FMEA） |
|------|-------------------|-------------------|
| 分析対象 | 製品の設計 | 製造工程 |
| 故障モード | 製品が意図通りに機能しない | 工程が意図通りに製品を作れない |
| 典型的な例 | 部品の破損、腐食、疲労破壊 | 寸法不良、組付け忘れ、異品混入 |
| 作成時期 | Phase 2（製品設計・開発） | Phase 3（工程設計・開発） |
| 主管部門 | 設計部門 | 生産技術 / 品質保証 |
| インプット | 製品図面・仕様書 | 工程フローチャート |
| アウトプット先 | 製品仕様への反映 | コントロールプラン |

## 2. AIAG-VDA 7ステップアプローチ

2019年に発行された**AIAG-VDA FMEAハンドブック**では、従来のFMEA手法を統合・刷新し、**7ステップアプローチ**が導入されました。

### 全体フロー

```
┌───────────────────────────────────────────────────────┐
│ システム分析（Step 1〜3）                                │
│                                                        │
│  Step 1: 計画と準備 ──→ スコープの定義                  │
│  Step 2: 構造分析   ──→ 工程構造の可視化                │
│  Step 3: 機能分析   ──→ 各工程の機能と要求事項           │
│                                                        │
├───────────────────────────────────────────────────────┤
│ 故障分析とリスク低減（Step 4〜6）                         │
│                                                        │
│  Step 4: 故障分析   ──→ 故障モード・影響・原因の特定      │
│  Step 5: リスク分析 ──→ S・O・D評価、AP決定              │
│  Step 6: 最適化     ──→ リスク低減アクションの実施        │
│                                                        │
├───────────────────────────────────────────────────────┤
│ 結果の文書化（Step 7）                                   │
│                                                        │
│  Step 7: 結果の文書化 ──→ 報告書作成、関係者への伝達      │
│                                                        │
└───────────────────────────────────────────────────────┘
```

### Step 1: 計画と準備（Planning and Preparation）

FMEAプロジェクトのスコープと計画を定義します。

| 項目 | 内容 |
|------|------|
| 目的 | FMEA実施の目的を明確化 |
| スコープ | 分析対象の工程範囲を定義 |
| チーム構成 | 多部門（生産技術・製造・品質・保全等）のメンバーを選定 |
| スケジュール | 実施時期とマイルストーンを計画 |
| ベースFMEA | 類似製品・工程のFMEAを参考資料として収集 |
| 5T | Timing, Team, Task, Tool, Intent の確認 |

### Step 2: 構造分析（Structure Analysis）

製造工程の構造を**3階層**で整理します。

```
プロセス項目（上位）     プロセスステップ（中位）     プロセス作業要素（下位）
Process Item            Process Step                Process Work Element
───────────           ──────────              ───────────────
プレスライン             1次プレス成形                材料セット
                                                    金型閉じ
                                                    プレス加工
                                                    ワーク取出し
                        2次プレス成形                材料セット
                                                    穴抜き加工
                                                    トリミング
                        工程内検査                   寸法測定
                                                    外観確認
```

**構造分析のポイント**:
- 工程フローチャートをベースに、各工程をさらに作業要素レベルまで分解する
- **4M**（Man, Machine, Material, Method）+ Environment の観点で要素を洗い出す
- 構造ツリー図やブロックダイアグラムで可視化する

### Step 3: 機能分析（Function Analysis）

各工程ステップとその作業要素に対して、**機能（何をするか）**と**要求事項（どの基準を満たすべきか）**を定義します。

```
プロセス項目の機能         プロセスステップの機能       作業要素の機能
───────────           ──────────            ───────────
ブラケットを              ブランク材を所定の            金型にブランク材を
製品仕様通りに製造する    絞り深さに成形する            正しい位置にセットする
                         [要求: 15.0±0.3mm]           [要求: 位置決めピンに嵌合]
```

### Step 4: 故障分析（Failure Analysis）

各機能に対して、故障モード・影響・原因を**故障チェーン**として特定します。

```
故障の影響（上位）  ←──  故障モード（中位）  ←──  故障の原因（下位）
Failure Effect          Failure Mode             Failure Cause
───────────        ──────────            ───────────
後工程で組付不可         絞り深さ不足               金型摩耗
顧客ラインストップ                                  材料ロット変動
                                                    プレス圧力不足
                                                    材料セット位置ずれ

外観不良                 成形品にキズ               金型面のキズ
顧客クレーム                                        異物噛み込み
```

**故障モードの典型例（工程別）**:

| 工程種別 | 故障モードの例 |
|---------|---------------|
| 加工 | 寸法不良、形状不良、面粗さ不良、バリ残り |
| 組立 | 部品の組付け忘れ、異品組付け、締付けトルク不足/過大 |
| 溶接 | 溶接強度不足、溶接位置ずれ、スパッタ付着 |
| 塗装 | 膜厚不足/過大、色差、タレ、ハジキ、異物付着 |
| 熱処理 | 硬度不足/過大、変形、組織異常 |
| 検査 | 不良品の見逃し、良品の誤判定 |
| 搬送・保管 | 搬送キズ、錆、異品混入、紛失 |

### Step 5: リスク分析（Risk Analysis）

各故障モードに対して、**重大度（S）**、**発生度（O）**、**検出度（D）** を評価し、**AP（Action Priority）** を決定します。

#### 評価基準

**重大度（Severity / S）**: 故障の影響の深刻さ

| S | 影響の程度 | 説明 |
|---|-----------|------|
| 10 | 極めて重大（警告なし） | 安全性・法規制に関わる故障（作業者・使用者への危害）、警告なし |
| 9 | 極めて重大（警告あり） | 安全性・法規制に関わる故障、警告あり |
| 8 | 非常に重大 | 製品の主要機能が喪失（車両走行不能等） |
| 7 | 重大 | 製品の主要機能が低下 |
| 6 | 中程度 | 製品の一部機能が喪失 |
| 5 | 軽度 | 製品の一部機能が低下 |
| 4 | 非常に軽度 | 外観・異音など、大部分の顧客が気づく |
| 3 | 軽微 | 外観・異音など、一部の顧客が気づく |
| 2 | ごくわずか | ほとんどの顧客が気づかない |
| 1 | なし | 影響なし |

**発生度（Occurrence / O）**: 故障原因が発生する頻度

| O | 発生の予測 | 目安となる発生率 |
|---|-----------|----------------|
| 10 | 極めて高い | ≧ 100 / 1,000個 |
| 9 | 非常に高い | 50 / 1,000個 |
| 8 | 高い | 20 / 1,000個 |
| 7 | やや高い | 10 / 1,000個 |
| 6 | 中程度 | 2 / 1,000個 |
| 5 | やや低い | 0.5 / 1,000個 |
| 4 | 低い | 0.1 / 1,000個 |
| 3 | 非常に低い | 0.01 / 1,000個 |
| 2 | ほぼない | ≦ 0.001 / 1,000個 |
| 1 | 排除されている | ポカヨケにより発生が排除 |

**検出度（Detection / D）**: 現行の管理方法で故障を検出できる能力

| D | 検出の可能性 | 説明 |
|---|-------------|------|
| 10 | 検出不可能 | 検出手段がない、または検査していない |
| 9 | ほぼ不可能 | 故障モードの検出が困難 |
| 8 | 非常に低い | ランダムなサンプリング検査 |
| 7 | 低い | 目視検査（二重チェックなし） |
| 6 | やや低い | 目視検査（チェックシート使用） |
| 5 | 中程度 | ゲージ・治具による検査 |
| 4 | やや高い | SPC（管理図）による監視 |
| 3 | 高い | 自動検査（自動測定・センサー） |
| 2 | 非常に高い | ポカヨケ（検出型：不良品を検出して排除） |
| 1 | ほぼ確実 | ポカヨケ（予防型：不良品の発生自体を防止） |

#### AP（Action Priority）

AIAG-VDA FMEAハンドブックでは、従来のRPN（S×O×D）に代わり**AP（Action Priority）** が導入されました。S・O・Dの組み合わせに基づいて、3段階の優先度を決定します。

| AP | 意味 | 対応 |
|----|------|------|
| **High（H）** | 高優先度 | 予防・検出管理の改善が**必須**。アクションを特定し、実施すること |
| **Medium（M）** | 中優先度 | 予防・検出管理の改善を**推奨**。改善を検討すべき |
| **Low（L）** | 低優先度 | 予防・検出管理の改善は**任意**。現行管理を継続してもよい |

**APの決定方法**:

APはS・O・Dの値の組み合わせに基づく参照テーブル（AIAG-VDA FMEAハンドブック付録）から決定します。単純なRPN計算ではなく、**重大度（S）を最優先**に考慮する点が特徴です。

**RPNとAPの違い**:

| 観点 | RPN（旧方式） | AP（新方式） |
|------|-------------|-------------|
| 計算方法 | S × O × D の単純な積 | S・O・Dの組み合わせテーブル |
| 問題点 | 同じRPN値でも重大度が異なる場合がある（例: 2×5×10=100 と 10×5×2=100） | 重大度を最優先に考慮するため、安全リスクを見逃さない |
| 閾値 | 企業ごとに独自の閾値（例: RPN≧100） | H / M / L の3段階で標準化 |
| 推奨対応 | 閾値超えの項目に対策 | APレベルに応じた段階的対応 |

### Step 6: 最適化（Optimization）

AP = High の項目を中心に、リスク低減アクションを計画・実施します。

**アクションの優先順位**:

| 順位 | アプローチ | 説明 | 例 |
|------|-----------|------|-----|
| 1 | 工程設計の変更 | 故障原因そのものを排除する | 工程の追加・順序変更 |
| 2 | 予防管理の強化 | 故障の発生度（O）を下げる | ポカヨケ導入、設備精度向上 |
| 3 | 検出管理の強化 | 故障の検出度（D）を下げる | 自動検査導入、検査頻度の増加 |

> **重要**: 予防管理（O低減）は検出管理（D低減）より常に優先されます。「検査を増やす」よりも「不良を発生させない」工程設計を目指します。

### Step 7: 結果の文書化（Results Documentation）

分析結果を文書化し、関係者に伝達します。

- FMEAレポートの作成
- コントロールプランへの展開
- 残留リスクの明示
- 経営層への報告

## 4. 具体例

### 金属プレス部品のPFMEA（抜粋）

品名: ブラケット A　品番: BK-1234

#### 工程No.50: 1次プレス成形

```
工程ステップ: 1次プレス成形（400tプレス機 / 絞り金型）
機能: ブランク材を所定の絞り深さ（15.0±0.3mm）に成形する
```

| 故障モード | 故障の影響 | S | 故障の原因 | 現行予防管理 | O | 現行検出管理 | D | AP |
|-----------|-----------|---|-----------|-------------|---|-------------|---|-----|
| 絞り深さ不足 | 後工程で組付不可→顧客ラインストップ | 8 | 金型摩耗 | 金型ショット数管理（寿命管理表） | 4 | 初物検査で絞り深さ測定 | 5 | **H** |
| 絞り深さ不足 | 同上 | 8 | 材料ロット変動（板厚・引張強度） | 受入検査で板厚・材質確認 | 3 | 初物検査で絞り深さ測定 | 5 | **M** |
| 絞り深さ不足 | 同上 | 8 | プレス圧力設定ミス | 作業標準書に設定値明記 | 2 | 初物検査で絞り深さ測定 | 5 | **M** |
| 成形品にキズ | 外観不良→顧客クレーム | 6 | 金型面のキズ・打痕 | 金型定期点検（週1回） | 3 | 外観検査（目視） | 7 | **M** |
| 成形品にキズ | 同上 | 6 | 異物噛み込み | エアブロー装置で金型面を清掃 | 3 | 外観検査（目視） | 7 | **M** |
| 成形品の割れ | 機能喪失→安全問題の可能性 | 9 | 材料の延性不足（材料不良） | 材料メーカーとの仕様取り決め | 2 | 初物検査で外観確認 | 6 | **H** |

## 5. コントロールプランへの展開

PFMEAの分析結果は、コントロールプランに以下のように展開されます。

```
PFMEAカラム                     コントロールプランカラム
──────────                     ────────────────

プロセスステップ           ──→  工程番号 / 工程名
機能・要求事項             ──→  製品特性（仕様・公差）
故障の原因                 ──→  工程特性（管理パラメータ）
特殊特性                   ──→  特殊特性分類（CC / SC）
現行予防管理               ──→  管理方法
現行検出管理               ──→  評価・測定技法 / サンプルサイズ / 頻度
AP                         ──→  管理の厳格さ（頻度・方法の決定根拠）
```

### 展開のポイント

- **AP = High の項目**: 全数検査または高頻度の抜取検査。SPC監視、ポカヨケ導入を検討
- **AP = Medium の項目**: 定期的な抜取検査。管理方法の改善を検討
- **AP = Low の項目**: 標準的な管理（ロット初物確認など）
- **予防管理**はコントロールプランの「管理方法」に、**検出管理**は「評価・測定技法」に反映する

## 6. よくある課題と対策

### 課題1: 故障モードが表面的

**問題**: 「寸法不良」のような漠然とした記述で、具体的な故障の状態が不明確。

**対策**:
- 故障モードは「機能の否定形」で記述する（例: 絞り深さが浅い、穴位置がずれている）
- 「不良」「NG」のような曖昧な表現を避け、具体的な状態を記述する
- Step 3（機能分析）を丁寧に行い、各機能に対する故障モードを導出する

### 課題2: S・O・Dの評価がチーム内でばらつく

**問題**: 評価者によってスコアが大きく異なり、一貫性がない。

**対策**:
- AIAG-VDA FMEAハンドブックの**評価基準表を必ず参照**して評価する
- チーム全員で合意形成（コンセンサス）を行う
- 類似製品のFMEA実績を参考にする（ベースFMEA）
- 評価の根拠を記録に残す

### 課題3: PFMEAが形骸化している

**問題**: 初回作成後に更新されず、実態と乖離している。

**対策**:
- **Living Document（生きた文書）** として運用する意識を持つ
- 工程変更・設計変更・市場クレーム・内部不良発生時にPFMEAを更新する
- 定期レビュー（年1回以上）をスケジュール化する

### 課題4: 検出管理（検査）ばかりに頼っている

**問題**: 予防管理が不十分で、「検査で見つける」アプローチに偏っている。

**対策**:
- AIAG-VDA FMEAの原則: **予防管理は検出管理より常に優先**
- 発生度（O）を下げる管理（ポカヨケ、SPC、設備メンテナンス）を先に検討する
- 検出度（D）を下げる管理（自動検査、センサー）は補完的に使用する

### 課題5: 故障の原因分析が浅い

**問題**: 表面的な原因（「作業者のミス」）で止まり、真の原因にたどり着いていない。

**対策**:
- **なぜなぜ分析（5 Why）** を活用して根本原因まで掘り下げる
- 4M + Environment の観点で原因を網羅的に洗い出す
- 「作業者のミス」は原因ではなく結果。「なぜミスが起きるか」（標準不明確、治具不備等）を特定する

---

## 7. ベストプラクティス

1. **多部門チームで実施する**: 生産技術・製造・品質・保全・購買の代表者を含める
2. **工程フローチャートを基盤にする**: Step 2の構造分析は工程フローチャートから展開する
3. **ベースFMEAを活用する**: 類似製品・工程の既存FMEAを起点として効率化する
4. **予防管理を優先する**: 「検査で見つける」より「不良を作らない」工程を設計する
5. **APに基づいて行動する**: 全項目に対策するのではなく、H→M→Lの優先順位で効率的に改善する
6. **Living Documentとして運用する**: 量産後も継続的に更新し、知見を蓄積する
7. **コントロールプランと連動させる**: PFMEAの変更はコントロールプランに即座に反映する
