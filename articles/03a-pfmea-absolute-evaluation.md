# PFMEA 絶対評価法（Absolute Evaluation Method — 客観説TQMに基づくアプローチ）

## 1. 概要

### 本記事の位置づけ

本記事は [PFMEA（工程FMEA）](./03-pfmea.md) の**関連・補足記事**です。`03-pfmea.md` では自動車産業の国際標準である AIAG-VDA FMEAハンドブック（2019年）の7ステップアプローチ・10点評価・AP（Action Priority）方式を解説しています。

本記事では、客観説TQM研究所（鵜沼崇郎氏）が提唱する**絶対評価4点法**という代替アプローチを解説します。この手法は民生品向けに改良されたもので、AIAG-VDA方式の形骸化問題を解決することを目的としています。

> **注意**: 本記事のアプローチは IATF 16949 や自動車 OEM が要求する公式の FMEA 形式ではありません。IATF 16949 対応が必要な場合は [PFMEA（工程FMEA）](./03-pfmea.md) を使用してください。本手法は社内改善活動や非自動車分野での活用に適しています。

### なぜ代替アプローチが必要か

AIAG-VDA の10点相対評価方式には、実務上いくつかの問題が指摘されています。

| 問題点 | 内容 |
|--------|------|
| **形骸化しやすい** | 評価基準が10段階と細かく、チーム内でスコアの合意が難しいため、形式的な記入になりがち |
| **相対評価の曖昧さ** | 全故障モードの分析が完了するまでスコアの妥当性が確定しない。評価者によってスコアが大きく異なる |
| **RPN同値問題** | 同じ RPN 値でもリスクプロファイルが全く異なる（例: S=2×O=5×D=10=100 と S=10×O=5×D=2=100） |
| **製品の性質による不適合** | 一品料理（ロケット等）と量産民生品では、FMEA の目的・考え方が根本的に異なる |

**製品の性質による FMEA アプローチの違い**:

```
一品料理（ロケット・医療機器等）          量産民生品（家電・プリンタ等）
──────────────────────          ──────────────────────
失敗が許されない                         軽微な欠陥も売上に影響
予防に全力を尽くす                         製品の「あたりまえ品質」だけでは不十分
AIAG-VDA の厳格な手法が適合              形骸化せず実効力あるアプローチが必要
```

### 目的

| 目的 | 説明 |
|------|------|
| 形骸化の防止 | 4段階の明確な絶対基準により、チーム内の評価のばらつきを低減する |
| 独立した合否判定 | 1つの故障モードだけで合否が決まるため、全モードの分析完了を待つ必要がない |
| 故障モードの正確な定義 | 「故障」と「故障モード」を峻別し、分析の精度を高める |
| 工程構造の視点 | 製品の不良だけでなく、工程の5M要素の変化を故障モードとして捉える |
| 民生品への適用 | 量産民生品に適したシンプルで実効力のある FMEA を実現する |

### 工程設計における位置づけ

```
工程フローチャート ──→ PFMEAの分析対象を定義
    │
    ▼
PFMEA（リスク分析）── 本記事の手法: 絶対評価4点法（客観説TQM）
    │              ── 標準手法:    AIAG-VDA 7ステップ・10点法（03-pfmea.md）
    │
    ▼
コントロールプラン ──→ PFMEAの結果を工程管理計画として文書化
```

## 2. 故障モードと故障の区別

絶対評価法を正しく適用するうえで最も重要な前提知識が、「**故障モード（Failure Mode）**」と「**故障（Failure）**」の区別です。

### 故障モード（Failure Mode）＝ 構造破壊

故障モードとは、システム（製品・部品・工程要素）に起こる**物理的・化学的な構造の破壊・変化**を指します。

| 故障モードの例 | 分類 |
|--------------|------|
| 割れ（クラック） | 機械的破壊 |
| 摩耗 | 機械的破壊 |
| 変形（塑性変形） | 機械的破壊 |
| 腐食・錆 | 化学的劣化 |
| 材質劣化 | 化学的劣化 |
| 短絡（ショート） | 電気的破壊 |
| 緩み | 接合破壊 |
| 異物混入 | 汚染 |
| 濁り | 物性変化 |

### 故障（Failure）＝ 機能障害

故障とは、構造破壊の結果として生じる**機能の喪失・低下**（症状）を指します。

| 故障の例 | 対応する故障モード（例） |
|---------|----------------------|
| 回転しない | 軸受の摩耗、シャフトの折損 |
| 止まらない | ブレーキパッドの摩耗 |
| 印字できない | ノズルの目詰まり（異物付着） |
| 異音が発生する | ギヤの摩耗、軸受の損傷 |

### 因果関係

```
故障モード（構造破壊）  ──→  故障（機能障害）  ──→  影響（顧客への影響）
Failure Mode              Failure                 Effect

例: 軸受摩耗           ──→  モーター回転せず  ──→  製品が動作しない
例: ネジ緩み           ──→  部品が外れる      ──→  安全上の問題
例: ノズル目詰まり     ──→  インク不吐出      ──→  印字品質不良
```

### よくある誤り

FMEA で最もよく見られる誤りは、**故障を故障モードとして記載してしまう**ことです。

| 誤り（故障を記載）| 正しい（故障モードを記載） |
|-----------------|--------------------------|
| 回転しない | 軸受の摩耗 / シャフトの折損 |
| ネジが締まらない | ネジ山の摩耗 / 材質強度不足 |
| 印字がかすれる | ノズルへの異物付着（目詰まり） |
| トルクが出ない | モーター巻線の断線 / 絶縁劣化 |
| 速度が出ない | 回転速度が ○○±○○ r/min の範囲から逸脱 |

> **ポイント**: 「回転しない」は故障（症状）です。故障モードは「なぜ回転しないのか」の物理的原因であり、「軸受の摩耗」や「シャフトの折損」のように**構造の変化**として記述します。速度のように定量的に表せる場合は「○○±○○ r/min から逸脱」のように記述するのが理想です。

## 3. 工程FMEAにおける故障モードの2つの考え方

工程 FMEA における「故障モードとは何か」については、客観説 TQM において**2つの異なる考え方**が整理されています。

### 製品構造説（Product Structure Theory）

製品を製造・処理する工程において、**製品に発生し得る構造破壊**を故障モードとする考え方です。

- 設計 FMEA（DFMEA）の考え方を工程に適用したもの
- 日本の多くの企業で採用されている
- 「工程中に製品に何が起きるか」の視点

**例**: キャリッジ組立工程における製品構造説の故障モード
- ヘッドの位置ずれ
- コネクタの変形
- レールの傷

**製品構造説の課題**:
- 不良は予測しきれない原因で発生するため、列挙した故障モード以外の不良が発生することがある
- 発生しないかもしれない不良項目にも対策を講じてしまい、労力と費用を無駄にする恐れがある

### 工程構造説（Process Structure Theory）

製品の製造・処理工程を構成する**5M要素に起こり得る変化・破壊**を故障モードとする考え方です。客観説 TQM が推奨するアプローチです。

- 「工程の何が変化するか」の視点
- 工程が安定した状態を維持できているかどうかに着目する
- ねらい: 良好な工程を得たときに、その状態が変わらないよう安定・維持させること

**5M の各要素における故障モードの例**:

| 5M要素 | 工程構造説による故障モード例 |
|--------|--------------------------|
| Machine（設備） | モーター回転速度が規定値±○○ r/min から逸脱 |
| Man（作業者） | 締付けトルクが規定値 ○○±○○ Nm から逸脱（うっかりミス） |
| Material（材料） | 部品寸法が規格 ○○±○○ mm から逸脱（受入不良） |
| Method（方法） | チューブ挿入深さが規定値 ○○±○○ mm から逸脱 |
| Measurement（計測） | 測定器のゼロ点が校正値から逸脱 |

### 両者の比較

| 観点 | 製品構造説 | 工程構造説 |
|------|----------|----------|
| 故障モードの対象 | 製品に生じる構造破壊（不良項目） | 工程の5M要素に起こる変化・破壊 |
| 主なねらい | 製品不良の発生を予防する | 工程の安定状態を維持する |
| 視点 | 「製品に何が起きるか」 | 「工程の何が変化するか」 |
| 設計 FMEA との関係 | DFMEA に近い発想 | DFMEA とは異なる固有の発想 |
| 適用例 | 加工寸法不良の予防 | 設備パラメータ逸脱の防止 |

> **推奨**: 客観説 TQM では、工程構造説を中心に分析し、必要に応じて製品構造説を補完的に実施することを推奨しています。設計 FMEA と工程構造説を理解すれば、製品構造説の FMEA は容易に実施できます。

## 4. 絶対評価4点法（Absolute 4-Point Evaluation Method）

### AIAG-VDA 10点法との根本的な違い

| 観点 | AIAG-VDA 10点法（相対評価） | 絶対評価4点法 |
|------|--------------------------|-------------|
| 評価スケール | 1～10（各指標） | 1～4（各指標） |
| 評価方式 | **相対評価**（他の故障モードとの比較） | **絶対評価**（各故障モードを独立して評価） |
| スコア確定タイミング | 全故障モードの分析完了後 | 各故障モードごとに即時確定 |
| リスク指標 | RPN（S×O×D、範囲1～1,000）またはAP表引き | RI = ∛(a×b×c)（範囲1.0～4.0） |
| 評価の一貫性 | 評価者・時期によりブレやすい | 4段階の明確な基準で安定 |
| b=c の関係 | S・O・D は独立して評価 | b=c特則あり（後述） |
| 組み合わせ数 | 1,000通り（10の3乗） | 64通り（4の3乗） |
| 主な適用分野 | 自動車産業（IATF 16949 必須） | 民生品・社内改善全般 |

### 影響度（a）の評価基準

影響度（a）は、故障モードが引き起こす最悪の影響の深刻さを評価します。

| スコア | 評価 | 判断の目安 |
|--------|------|----------|
| **1** | 影響なし | 故障モードが発生しても、次工程・顧客・社会に実質的な影響を与えない |
| **2** | 許容範囲内 | 影響はあるが、許容できる範囲内。軽微な品質低下や違和感程度 |
| **3** | 許容範囲外 | 許容できる範囲を超える影響。製品の機能低下、顧客クレームの発生 |
| **4** | 致命的 | 安全上の危害、法規制違反、製品リコール、会社の重大な信用失墜 |

### 発生頻度（b）の評価基準

発生頻度（b）は、故障モードの原因に対してどれだけ予防管理が行われているかを評価します。

| スコア | 評価 | 判断の目安 |
|--------|------|----------|
| **1** | 完全な管理 | ポカヨケ等により原因の発生が実質的に排除されている |
| **2** | 十分な管理 | 適切な予防管理が施されており、発生頻度は十分低い |
| **3** | 管理不十分 | 予防管理はあるが不十分。改善の余地がある |
| **4** | 無管理 | 予防管理が存在しない。発生を防ぐ手段がない |

### 検知度（c）の評価基準

検知度（c）は、故障モードや影響が顧客・社会に大事を引き起こす前に検知できるかを評価します。

| スコア | 評価 | 判断の目安 |
|--------|------|----------|
| **1** | 完全な管理 | 必ず検知できる仕組みがある（ポカヨケ型検知、全数自動検査等） |
| **2** | 十分な管理 | 十分な検知管理がある。高い確率で検知できる |
| **3** | 管理不十分 | 検知管理はあるが不十分。見逃しの可能性がある |
| **4** | 無管理 | 検知手段が存在しない |

### b=c 特則（b=1 のとき c=1）

絶対評価4点法の特徴的なルールが**b=c 特則**です。

**ルール**: 発生頻度 b=1（原因が完全に管理・排除されている）の場合、検知度 c は自動的に1とみなします（c=1）。

**理由**: 原因が発生しないのであれば、検知する必要もないからです。発生しないものを検知するための設備・コストは不要です。

```
b=1（原因がポカヨケ等で完全に排除されている）
    │
    ▼
c は自動的に 1（検知管理の必要なし）
    │
    ▼
RI = ∛(a × 1 × 1) = ∛a
```

**実例**: 新幹線は約60年間、車軸の故障モード（金属疲労による破断）を設計と管理により実質的に排除しています（b=1）。そのため走行中の車軸破断検知装置への依存度は最小限で済みます。これが b=c 特則の考え方です。

> **注意**: b=c 特則は b=1 の場合**のみ**適用します。b=2 の場合は c を独立して評価する必要があります。b=2 で c=1 とすることは誤りです。

## 5. RI（Risk Index）の算出と判定

### 計算式

```
RI = ∛(a × b × c)   （a・b・c の積の3乗根）
```

> **なぜ3乗根か**: a・b・c の単純な積は最大 4×4×4=64 になります。3乗根を取ることで評価スケール（1～4）と同じ範囲（1.0～4.0）に正規化され、直感的な解釈が可能になります。AIAG-VDA の RPN（最大1,000）と異なり、解釈が容易です。

### 判定基準

| RI 値 | 判定 | 対応 |
|-------|------|------|
| **RI ≦ 2.0** | **合格** | 現行の対策で十分。管理を継続する |
| **2.0 ＜ RI ≦ 2.3** | **保留** | 改善を検討する。経過観察・レビューを実施する |
| **RI ＞ 2.3** | **不合格** | 改善アクションが必要。a・b・c のいずれかを低減する対策を講じる |

### 特例規定（a=1 のとき）

**ルール**: 影響度 a=1（影響なし）の場合、b=4・c=4 であっても（RI = ∛16 ≒ 2.52）、判定は**不合格ではなく「保留」**とします。

**理由**: 故障モードが発生しても外部（次工程・顧客・社会）に実質的な影響がないため、高リスクとして扱う必要がないからです。

### RI 値の早見表

RI = ∛(a×b×c) の主要な組み合わせを示します。

| a＼b,c | b=1,c=1 | b=1,c=2 | b=2,c=2 | b=2,c=3 | b=3,c=3 | b=3,c=4 | b=4,c=4 |
|--------|---------|---------|---------|---------|---------|---------|---------|
| **a=1** | 1.00 合格 | 1.26 合格 | 1.59 合格 | 1.82 合格 | 2.08 保留 | 2.29 保留 | 2.52 保留※ |
| **a=2** | 1.26 合格 | 1.59 合格 | 2.00 合格 | 2.29 保留 | 2.62 不合格 | 2.88 不合格 | 3.17 不合格 |
| **a=3** | 1.44 合格 | 1.82 合格 | 2.29 保留 | 2.62 不合格 | 3.00 不合格 | 3.30 不合格 | 3.63 不合格 |
| **a=4** | 1.59 合格 | 2.00 合格 | 2.52 不合格 | 2.88 不合格 | 3.30 不合格 | 3.63 不合格 | 4.00 不合格 |

※ a=1 特例により「保留」（通常なら不合格）

> **b=c 特則の効果**: b=1 の列では c=1（特則適用）のため、RI = ∛a となります。a=4 でも RI=1.59（合格）です。

### 改善の方向性

不合格・保留の場合、以下の観点から対策を検討します。

| 改善の観点 | アプローチ | 例 |
|----------|----------|-----|
| 影響度（a）を下げる | 故障モードが発生しても影響が出ない設計 | フェイルセーフ設計、二重系構造 |
| 発生頻度（b）を下げる | 原因の発生を防ぐ予防管理 | ポカヨケ、設備精度向上、SPC |
| 検知度（c）を下げる | 故障モードを確実に検知する | 全数自動検査、センサー検知 |

> **優先順位**: a の低減（影響抑制）＞ b の低減（発生防止）＞ c の低減（検知強化）の順が望ましい。「検査で見つける」より「起こさない」設計を優先します。

## 6. 具体例

### 例1: PX-500 キャリッジ組立（工程No.60）— 工程構造説による分析

工程No.60（キャリッジ組立）を工程構造説で分析します。[03-pfmea.md の同工程（AIAG-VDA方式）](./03-pfmea.md) との対比で絶対評価法の特徴を示します。

```
工程No.60: キャリッジ組立
使用設備: 電動ドライバー EQ-060 / レーザー変位計 M-061
機能: 印字ヘッドキャリッジをレールに取付け、位置精度（±0.05mm）に調整する
```

**工程構造説による故障モードの特定（5M分析）**:

| 5M要素 | 故障モード（工程構造説） | 対応する AIAG-VDA の故障モード（製品構造説） |
|--------|----------------------|----------------------------------------|
| Machine | ネジ締付けトルクが規定値 0.5±0.05 Nm から逸脱 | ヘッド位置ずれ |
| Machine | レーザー変位計のゼロ点が校正値から逸脱 | 測定誤差による誤判定 |
| Man | 段取り時の速度設定が規定値から逸脱 | 組立不良 |
| Material | ガイドレール（GR-500）の寸法が規格から逸脱 | レール平行度不良 |
| Method | ネジの締付け順序が手順書から逸脱 | ヘッド取付け精度不足 |

**絶対評価4点法による PFMEA（工程No.60 抜粋）**:

```
品名: インクジェットプリンタ PX-500  品番: PX-500A
FMEA番号: FM-PX500-AE-R01  対象工程: 工程No.60 キャリッジ組立
```

| 故障モード | 5M | 原因 | a | b | c | RI | 判定 |
|----------|-----|------|---|---|---|------|------|
| ネジ締付けトルクが 0.5±0.05 Nm から逸脱 | Machine | 電動ドライバーの摩耗・劣化 | 3 | 2 | 2 | 2.29 | 保留 |
| レーザー変位計のゼロ点逸脱 | Machine | 未校正・経時変化 | 2 | 2 | 2 | 2.00 | 合格 |
| 段取り速度設定が規定値から逸脱 | Man | 作業者のうっかり操作 | 4 | 3 | 2 | 2.88 | 不合格 |
| レール寸法が規格から逸脱 | Material | 受入検査不十分 | 3 | 2 | 2 | 2.29 | 保留 |
| 締付け順序の逸脱 | Method | 手順書の不明確 | 3 | 2 | 3 | 2.62 | 不合格 |

**改善アクションと改善後評価**:

| 故障モード | 対策 | a | b | c | RI | 判定 |
|----------|------|---|---|---|------|------|
| トルク逸脱 | トルク監視装置＋異常時自動停止（ポカヨケ）導入 | 3 | 1 | 1（b=c特則） | 1.44 | 合格 |
| 段取り速度逸脱 | 速度変更シフトをノックピンで固定（ポカヨケ） | 4 | 1 | 1（b=c特則） | 1.59 | 合格 |
| 締付け順序逸脱 | 作業指示書の改訂＋治具で締付け順序を物理的に規制 | 3 | 2 | 2 | 2.29 | 保留 |

**AIAG-VDA方式（03-pfmea.md）との同工程の比較**:

| 観点 | AIAG-VDA 方式（03-pfmea.md） | 絶対評価4点法（本記事） |
|------|----------------------------|----------------------|
| 故障モードの視点 | 製品構造説：「ヘッド位置ずれ」 | 工程構造説：「トルクが規定値から逸脱」 |
| 評価スコア | S=8, O=4, D=3, AP=High | a=3, b=2, c=2, RI=2.29（保留） |
| 改善後スコア | S=8, O=3, D=2, AP=Low | a=3, b=1, c=1, RI=1.44（合格） |
| 合否の即時性 | 全モード分析後に相対評価 | 各モードごとに即時判定 |

### 例3: 薬瓶ねじ蓋本締め工程（客観説TQM 原典例）

客観説TQM で紹介されている代表的な事例です。PX-500 以外の工程にも本手法が適用できることを示します。

```
工程: かぜ薬の錠剤が入った瓶のねじ蓋を本締めする組立工程
設備: 自動本締め機
前工程: 仮締め済みの瓶が搬送されてくる
```

| 故障モード（工程構造説） | 5M | 原因 | a | b | c | RI | 判定 |
|----------------------|----|----|---|---|---|------|------|
| 本締め機の回転速度が規定速度から逸脱 | Machine | 段取り作業者のうっかり操作 | 4 | 3 | 3 | 3.30 | 不合格 |

**最悪の影響（a=4 の根拠）**: 回転速度が規定より遅い → 蓋が本締めされず緩い → 輸送中に蓋が外れる → 内容物（かぜ薬）が変質・汚染 → 消費者への健康被害 → 製品リコール・会社の信用失墜

**改善アクション**: 速度変更シフトレバーをノックピンで固定し、物理的に速度変更を不可能にする（ポカヨケ）

| 改善後 | a | b | c | RI | 判定 |
|--------|---|---|---|------|------|
| ノックピン固定後 | 4 | 1（完全に排除） | 1（b=c特則） | 1.59 | 合格 |

## 7. AIAG-VDAアプローチとの比較

### 評価方法の総合比較

| 比較項目 | AIAG-VDA（03-pfmea.md） | 絶対評価4点法（本記事） |
|---------|------------------------|----------------------|
| 規格・根拠 | AIAG-VDA FMEA Handbook (2019) | 客観説TQM（鵜沼崇郎） |
| 評価スケール | 各指標 1～10 | 各指標 1～4 |
| 評価方式 | 相対評価 | 絶対評価 |
| リスク指標 | AP（H/M/L、表引き）または RPN | RI = ∛(a×b×c) |
| スコア確定タイミング | 全故障モード分析完了後 | 各故障モードごとに即時確定 |
| 重大度の扱い | S=9～10 は AP=High 確定 | a=1 特例規定あり |
| b・c の関係 | S・O・D は完全に独立 | b=1 → c=1（b=c特則） |
| 評価組み合わせ数 | 1,000通り | 64通り |
| 形骸化のリスク | 比較的高い | 比較的低い |
| IATF 16949 適合 | 必須要件を満たす | 公式要件ではない |
| 民生品への適合 | 過剰なケースがある | 設計思想が民生品向き |
| 学習コスト | 高い（7ステップ・複雑な評価表） | 低い（シンプルな4段階） |

### 適用場面の使い分け

| 状況 | 推奨アプローチ |
|------|--------------|
| IATF 16949 認証・自動車 OEM への PPAP 提出 | **AIAG-VDA 方式**（必須） |
| 自動車部品サプライヤーの内部品質活動 | **AIAG-VDA 方式**（顧客提出用と整合させる） |
| 民生品メーカーの工程リスク分析（非自動車） | **絶対評価4点法**（シンプルで実効的） |
| 社内の工程改善・リスクの優先順位付け | **絶対評価4点法**（形骸化を防ぎやすい） |
| PFMEA を初めて導入する組織 | **絶対評価4点法** → 将来 AIAG-VDA へ移行 |
| 医療機器・食品・化学品などの工程分析 | **絶対評価4点法**（製品の性質に合わせて評価基準を調整） |

### 併用の推奨

IATF 16949 対応が必要な企業においても、以下のように両手法を併用することができます。

```
社内分析（絶対評価4点法）          顧客提出用（AIAG-VDA方式）
──────────────────          ──────────────────
・チーム内の素早いリスク合意         ・IATF 16949 要求への対応
・形骸化しにくい評価基準             ・顧客の標準フォーマットに準拠
・工程構造説による5M分析             ・7ステップアプローチに従った分析
         │                                    ↑
         └──→ 分析結果をAIAG-VDA形式に変換 ──┘
```

## 8. よくある課題と対策

### 課題1: 故障モードと故障を混同する

**問題**: 「回転しない」「印字できない」等の機能障害（故障）を故障モードとして記載してしまい、分析が表面的になる。

**対策**:
- 本記事のセクション2の区別（故障モード＝構造破壊、故障＝機能障害）を FMEA 実施前にチームで共有する
- 記載した故障モードに対して「なぜそれが起きるのか（物理的メカニズムは何か）」を問い、構造破壊の記述になるまで掘り下げる
- 定量的に表現できるパラメータは「○○が規定値 ○○±○○ から逸脱」の形式で記述する

### 課題2: 工程構造説の故障モードが抽象的になる

**問題**: 「パラメータが逸脱する」「設備が異常になる」等、曖昧な記述になる。

**対策**:
- 必ず具体的なパラメータ名・単位・規定値を含めて記述する（例: 「ネジ締付けトルクが 0.5±0.05 Nm から逸脱」）
- 5M の各要素について、管理しているパラメータのリストを事前に作成しておく
- 設備仕様書・作業指示書・コントロールプランの管理パラメータを故障モードの候補として活用する

### 課題3: 4点法でも評価にばらつきが出る

**問題**: 4段階でも評価者によってスコアが異なり、合意が難しい。

**対策**:
- 各スコアに対する自社・工程固有の判断例を事前に作成しておく（評価ガイドライン）
- a=4（致命的）の基準を最初に明確にする（安全上の危害・リコールに相当するものを a=4 とする）
- 評価の根拠をコメント欄に記録し、レビュー時の議論の基礎にする

### 課題4: AIAG-VDA 要求がある顧客への対応

**問題**: 顧客が AIAG-VDA 形式の PFMEA 提出を要求しているが、社内は絶対評価法を使いたい。

**対策**:
- 絶対評価4点法を社内の分析・合意形成ツールとして使用し、その結果を AIAG-VDA 形式に変換して提出する
- 変換のマッピング例: RI≦2.0（合格）→ AP=Low、2.0＜RI≦2.3（保留）→ AP=Medium、RI＞2.3（不合格）→ AP=High
- 最終的には AIAG-VDA の要求カラム（S/O/D/AP）を満たす文書として整備する

### 課題5: b=c 特則の過剰適用

**問題**: b=2 や b=3 の場合にも「発生頻度が低いから」という理由で c=1 としてしまう。

**対策**:
- b=c 特則は**b=1（原因が完全に排除されている）場合のみ**適用する
- b=1 の根拠として、ポカヨケ・フールプルーフ等の具体的な防止機構を FMEA に記録する
- b=2 以上の場合は必ず c を独立して評価する

## 9. ベストプラクティス

1. **故障モードと故障の区別をチームで共有してから始める**: FMEA 開始前に全員がセクション2の概念を理解していることを確認する
2. **工程構造説と製品構造説を使い分ける**: まず工程構造説（5M分析）で工程の安定性を評価し、補完的に製品構造説を実施する
3. **5M を網羅的にチェックする**: Machine・Man・Material・Method・Measurement の各要素について、管理しているパラメータをすべてリストアップしてから故障モードを特定する
4. **b=c 特則を厳格に適用する**: b=1 の根拠（ポカヨケの具体的内容）を必ず記録し、b≧2 の場合は c を独立して評価する
5. **評価の根拠を記録する**: a・b・c の各スコアに対して判断の根拠・コメントを残し、後日のレビューに備える
6. **改善前後を必ず記録する**: 対策前の RI と対策後の RI を記録し、改善効果を定量的に示す
7. **AIAG-VDA が必要な場合は両手法を使い分ける**: 社内分析に絶対評価法、顧客提出用に AIAG-VDA 形式という二段階の運用を定着させる
8. **定期的に見直す**: 工程変更・クレーム・内部不良発生時に FMEA を更新し、Living Document として運用する
