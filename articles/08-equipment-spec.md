# 設備仕様書（Equipment Specification）

## 1. 概要

### 設備仕様書とは

設備仕様書（Equipment Specification）とは、製造工程で使用する**生産設備・検査設備・試験装置**に対して、生産技術部門が要求する性能・精度・機能・安全要件を文書化したものです。

設備の設計・製作・調達・検収のすべての段階で基準となる文書であり、**ユーザー要求仕様書（URS: User Requirements Specification）** とも呼ばれます。設備メーカー（サプライヤー）に対して発行し、設備の仕様合意・契約の根拠となります。

### 目的

| 目的 | 説明 |
|------|------|
| 要求の明確化 | 設備に求める性能・精度・機能を定量的に定義し、認識齟齬を防ぐ |
| 設備選定の基準 | 複数の設備メーカーを比較評価する共通基準を提供する |
| 設備検収の基準 | 納入された設備が要求仕様を満たしているかを検証する根拠とする |
| トレーサビリティ | 工程フローチャートの各ステップで必要な設備を文書で追跡可能にする |
| 保全計画の基礎 | 設備の保全周期・点検項目の根拠となる設計情報を記録する |
| 変更管理 | 設備変更時に元の仕様との差分を明確にし、変更の影響を評価する |

### 工程設計における位置づけ

```
工程フローチャート（各工程の作業内容を定義）
    │
    ▼
設備仕様書（各工程で必要な設備の要求仕様を定義）
    │
    ├──→ 設備メーカーへの発注・仕様合意
    │
    ├──→ 治工具設計図（設備に合わせた治具・工具を設計）
    │
    ├──→ フロアプランレイアウト（設備を配置する平面計画）
    │
    └──→ 設備検収・据付・試運転
            │
            ▼
        コントロールプラン / 作業指示書（設備の管理方法を規定）
```

設備仕様書は工程フローチャートで定めた各工程の**技術的要求を設備レベルに落とし込む**文書です。PFMEA・コントロールプランで設定した品質管理要件（精度・能力等）を満たす設備を定義する役割を担います。

### IATF 16949の要求事項

IATF 16949:2016では、設備に関連する主な要求条項は以下のとおりです。

| 条項 | 内容 |
|------|------|
| **7.1.3** インフラストラクチャ | 工程の実施に必要な設備・施設・インフラを決定し、提供・維持すること |
| **8.5.1** 製造・サービス提供の管理 | 適切な設備・実施設備を使用した管理された条件下で行うこと |
| **8.5.1.5** 生産ツールのマネジメント | 工具・ゲージ・設備の識別・管理・保管・保全の仕組みを確立すること |
| **8.5.4** 保存 | 製品と構成部品の特性を維持する適切な設備・環境を提供すること |

## 2. 設備の分類

工程設計で扱う設備は目的に応じて分類されます。

| 分類 | 内容 | 例 |
|------|------|-----|
| **生産設備** | 製品の加工・組立・製造を行う設備 | 電動ドライバー、圧入機、溶接機、プレス機、射出成形機 |
| **検査・試験設備** | 製品の品質特性を測定・判定する設備 | 三次元測定機、画像検査装置、リーク試験機、電気検査装置 |
| **搬送・保管設備** | 製品の搬送・位置決め・保管を行う設備 | コンベア、AGV、棚、トレー |
| **ユーティリティ設備** | 生産環境を維持する設備 | 空調、圧縮空気供給、電源、クリーンブース |
| **保全用設備** | 設備の維持管理に使用する設備 | 校正器、診断装置 |

## 3. 記載項目

### ヘッダー情報

| 項目 | 内容 |
|------|------|
| 設備仕様書番号 | 文書の識別番号と版数 |
| 設備名称 | 設備の正式名称 |
| 設備管理番号 | 社内の設備識別番号（EQ-XXX等） |
| 対象工程 | 工程フローチャートの工程番号・工程名 |
| 作成日 / 改訂日 | 初版日と最新改訂日 |
| 作成者 / 承認者 | 担当者と承認責任者 |
| 設備メーカー | 納入先の設備製作会社名 |

### 本文（要求仕様）

#### A. 基本要件

| 項目 | 説明 |
|------|------|
| 目的・用途 | 設備で実施する工程と加工・処理内容の概要 |
| 対象製品 | 処理する製品の品番・形状・材質・重量 |
| 生産能力 | 要求タクトタイム、稼働率目標 |
| 設置環境 | 設置場所・床荷重・電源・エアー圧力・温湿度条件 |
| 設備寸法・重量 | 最大寸法と重量（搬入経路・基礎設計の根拠） |
| 安全規格 | 適合すべき安全規格（CE、JIS、OSHA等） |

#### B. 性能・精度要件

| 項目 | 説明 |
|------|------|
| 加工精度 / 測定精度 | 必要な精度（公差・分解能）と繰返し精度（Cpk≥1.67等） |
| 処理能力 | 1サイクルの処理時間・スループット |
| 可用性 | 要求稼働率（OEE目標等） |
| 測定分解能 | 測定設備の場合、規格の10分の1以上の分解能（MSAガイドライン） |

#### C. 機能要件

| 項目 | 説明 |
|------|------|
| 主要機能 | 設備が実現すべき機能の詳細仕様 |
| ポカヨケ機能 | 誤品・誤工程を防止するためのエラープルーフ機能 |
| データ収集機能 | 品質データ・工程パラメータの記録・出力機能（トレーサビリティ） |
| アラーム機能 | 異常検知時の警報・停止機能 |
| インターロック | 安全確保のための連動停止・ロック機能 |
| 段取り替え | 品種切替時の段取り替え手順・時間 |
| ヒューマンインターフェース | 操作パネル、表示言語、使いやすさ |

#### D. 品質管理要件

| 項目 | 説明 |
|------|------|
| 工程能力 | 設備で達成すべきCpk目標値（通常Cpk≥1.67以上） |
| 特殊特性への対応 | CC（◆）/SC（◇）の管理に必要な設備機能 |
| 記録・トレーサビリティ | 測定・加工結果の記録方式と保管方法 |
| SPC対応 | 管理図・SPCシステムとの連携機能 |

#### E. 保全・信頼性要件

| 項目 | 説明 |
|------|------|
| MTBF（平均故障間隔） | 目標とするMTBF値 |
| MTTR（平均修復時間） | 故障時の目標復旧時間 |
| 予防保全計画 | 定期点検・部品交換の周期と内容 |
| スペアパーツ | 必須スペアパーツのリストと推奨在庫数 |
| 保守性 | メンテナンス作業性（工具アクセス、交換容易性） |

#### F. 受入検査（FAT/SAT）要件

| 項目 | 説明 |
|------|------|
| FAT（工場検収試験） | 設備メーカー工場での検収試験の実施条件・合格基準 |
| SAT（現地検収試験） | 製造現場での据付後の検収試験の実施条件・合格基準 |
| 検収項目 | 仕様書の全要件に対する検証方法と合格基準 |
| 試運転期間 | 量産前の試運転期間と性能実証方法 |

## 4. 作成手順

### Step 1: 要求事項の整理

工程フローチャートとPFMEAをインプットに、設備に対する要求事項を整理します。

| インプット | 抽出する情報 |
|-----------|------------|
| 工程フローチャート | 各工程の作業内容、使用する設備の概要 |
| PFMEA | 故障モードと対策（ポカヨケ・エラープルーフ要件） |
| コントロールプラン | 管理精度、測定方法、サンプリング頻度 |
| 特殊特性リスト | CC/SC特性に対応した設備機能要件 |
| 製品図面 | 加工対象の形状・材質・寸法 |
| 生産計画 | 生産量・タクトタイム・稼働率目標 |

### Step 2: ベンチマークと技術調査

類似設備の実績や市場調査を行い、実現可能な仕様を確認します。

- 社内の既存類似設備の性能実績を確認
- 設備メーカーへの技術相談・カタログ調査
- 業界標準・規格（JIS、ISO、安全規格）の確認

### Step 3: 仕様の起案

収集した情報をもとに設備仕様書の草案を作成します。

**精度要件の設定原則**:
```
製品の品質特性の公差        設備の精度要件
──────────────     ─────────────
公差の10分の1以下   →   設備の繰返し精度（3σ）
公差の4分の1以下    →   設備の測定分解能（一般的なMSAガイドライン）
```

**タクトタイム計算例**:
```
タクトタイム = 利用可能時間 ÷ 要求生産量
            = 7.5h × 3600s ÷ 500台
            = 54秒/台

設備サイクルタイム要求 = タクトタイム × 稼働率目標
                      = 54秒 × 85%
                      ≒ 46秒以内
```

### Step 4: 関係者レビュー

| 参加者 | レビュー観点 |
|--------|------------|
| 生産技術 | 技術的な実現可能性・コスト |
| 品質保証 | 品質管理要件（精度・トレーサビリティ） |
| 製造 | 操作性・保全のしやすさ |
| 安全衛生 | 安全規格への適合 |
| 購買 | 予算・調達リードタイム |
| 設備メーカー | 技術的実現性の確認（RFQ前の技術協議） |

### Step 5: 設備メーカーへの提示と合意

設備仕様書を設備メーカーに提示し、技術的な実現可否・コストを確認したうえで仕様を合意します。

- **RFQ（見積依頼書）** と一緒に設備仕様書を配布
- 設備メーカーからの**技術提案書・偏差リスト（Deviation List）** を確認
- 仕様に対する例外・変更事項を文書で合意

### Step 6: 受入検査（FAT・SAT）

設備納入時に仕様書の要件を検証します。

```
FAT（Factory Acceptance Test）
  └── 設備メーカー工場で実施
  └── 検証項目: 精度測定、機能確認、サイクルタイム確認
  └── 合格後に搬出許可

SAT（Site Acceptance Test）
  └── 製造現場に据付後に実施
  └── 検証項目: 据付精度、量産品での性能実証、インターフェース確認
  └── 合格後に量産承認
```

## 5. 具体例

### インクジェットプリンタ PX-500 組立工程の設備仕様書

工程No.60（キャリッジ組立）で使用する**レーザー変位計**の設備仕様書を例示します。

```
─────────────────────────────────────────────────────────────
設備仕様書

設備仕様書番号: ES-PX500-060-01-R01    改訂日: 2025/12/01
設備名称: ヘッドアライメント計測装置
設備管理番号: M-061
対象工程: 工程No.60 キャリッジ組立
関連CP: CP-PX500-R01（特性No.5 ヘッドアライメント）
関連PFMEA: FM-PX500-R01（工程No.60）

作成: 生産技術 田中    承認: 生産技術部長 佐藤
設備メーカー: ○○精機株式会社
─────────────────────────────────────────────────────────────

■ 目的・用途

  レールへの取付後、ヘッドアライメント（位置ずれ量）を
  全数自動測定し、合否を自動判定する。

■ 対象製品
  製品名: インクジェットプリンタ PX-500
  品番: PX-500A
  測定対象: 印字ヘッドキャリッジ（ヘッドアライメント）

■ 管理特性
  ◆CC: ヘッドアライメント（位置ずれ量）
       仕様: ±0.05mm

─────────────────────────────────────────────────────────────
■ A. 基本要件
─────────────────────────────────────────────────────────────

  生産能力
    要求サイクルタイム: 15秒以内 / 台（工程全体54秒の一部）
    対応タクトタイム: 54秒 / 台（計画生産量: 500台 / 日 / シフト）

  設置環境
    設置場所: 組立ライン工程No.60 測定ステーション
    電源: AC200V 50Hz 3相 15A以下
    エアー: 使用なし
    環境条件: 温度 20±5℃、湿度 30〜70%RH（結露なし）

  安全規格
    CE マーキング適合
    JIS B 9700（機械安全一般原則）準拠
    レーザー安全クラス: Class 1（IEC 60825-1準拠）

─────────────────────────────────────────────────────────────
■ B. 性能・精度要件
─────────────────────────────────────────────────────────────

  測定精度
    測定範囲: ±1.0mm
    分解能: 0.001mm（1μm）以下
    繰返し精度（3σ）: ±0.005mm 以内
    直線性誤差: ±0.003mm 以内
    温度特性: ±0.002mm / 10℃ 以内

  【精度要件の根拠】
    品質特性の仕様: ±0.05mm
    設備精度要件  : 繰返し精度 ±0.005mm（仕様の1/10）
    → Cpk ≥ 1.67 を達成するための必要精度

  測定サイクル
    測定ポイント数: 3点（左端・中央・右端）
    1台あたり測定時間: 10秒以内

  可用性
    目標稼働率: 95%以上

─────────────────────────────────────────────────────────────
■ C. 機能要件
─────────────────────────────────────────────────────────────

  自動測定機能
    ワーク搬入後、測定プログラム「PX500-ALIGN」を自動実行する
    測定値を自動で合否判定し、結果を表示する
    合格（緑ランプ）/ 不合格（赤ランプ・ブザー）

  ポカヨケ機能（◆CC 特性のため必須）
    不合格品はライン上の停止機構で自動停止させ、次工程への流出を防止する
    合格品のみ次工程へ搬出可能とする

  データ収集・トレーサビリティ
    全数の測定値・合否判定結果・測定日時・製品シリアル番号を記録する
    生産管理システム（MES）へのデータ自動転送機能を設ける
    データ保存期間: 最低10年間

  ゼロ点補正・校正機能
    校正用基準ゲージによるゼロ点補正機能（量産開始前の実施）
    校正周期: 4時間ごと（自動リマインド機能）
    校正記録の自動保存機能

  アラーム・インターロック
    測定値が管理限界（±0.04mm）を超えた場合に早期警告アラームを発する
    （仕様 ±0.05mm の手前で段階的な管理）

─────────────────────────────────────────────────────────────
■ D. 保全・信頼性要件
─────────────────────────────────────────────────────────────

  信頼性目標
    MTBF（平均故障間隔）: 2,000時間以上
    MTTR（平均修復時間）: 2時間以内

  予防保全
    日常点検: レンズ清掃、測定値確認（作業者実施）
    月次点検: 精度確認測定、固定部ネジ増し締め（保全実施）
    年次点検: センサー校正、消耗部品交換（メーカー実施）

  スペアパーツ
    受光センサーユニット: 1式（納入時同梱）
    発光ユニット: 1式（納入時同梱）
    制御ボード: 1式（メーカー在庫確認）

─────────────────────────────────────────────────────────────
■ E. 受入検査（FAT / SAT）要件
─────────────────────────────────────────────────────────────

  FAT（工場検収試験）
    場所: 設備メーカー工場
    実施者: 生産技術 田中 + 品質保証
    合格基準:
      ① 精度確認: 基準ゲージ（±0.050mm, ±0.030mm, ±0.010mm）
                  による測定値が仕様を満足すること
      ② サイクルタイム: 3点測定が10秒以内であること
      ③ ポカヨケ動作: 意図的な不合格品（±0.060mm相当）を
                      投入した際に停止機構が作動すること
      ④ データ出力: 測定値・判定結果が正しく出力されること

  SAT（現地検収試験）
    場所: 製造現場 工程No.60
    実施者: 生産技術 + 品質保証 + 製造
    合格基準:
      ① FATと同等の精度確認
      ② 実機（PX-500A）50台の連続測定でサイクルタイム達成
      ③ 生産管理システム（MES）とのデータ連携確認
      ④ 作業者への操作教育完了

─────────────────────────────────────────────────────────────
■ 改訂履歴
─────────────────────────────────────────────────────────────
  R01 (2025/12/01): 初版発行
─────────────────────────────────────────────────────────────
```

## 6. URS（ユーザー要求仕様書）との関係

設備仕様書は**URS（User Requirements Specification）** とも呼ばれ、設備メーカーが作成する**DS（Design Specification / 設計仕様書）** と対をなします。

```
URS（ユーザー要求仕様書）             DS（設計仕様書）
─────────────────                ─────────────────
作成者: ユーザー（生産技術）           作成者: 設備メーカー

「何が必要か」を定義する             「どのように実現するか」を定義する
  ・測定精度 ±0.005mm              ・レーザー変位方式 350nm波長
  ・サイクルタイム 10秒以内          ・3軸リニアモーター駆動
  ・ポカヨケ機能（CC特性）            ・PLC制御 + 機械式ストッパー

           ↓ 要求
URS ────────────────→ DS
           ← 合意・承認 ──
```

**整合確認の重要性**: DSがURSの全要件を満たしているかを**要件トレーサビリティマトリクス（RTM）** で確認します。

## 7. よくある課題と対策

### 課題1: 精度要件が曖昧

**問題**: 「高精度な設備」「十分な能力を持つ設備」等、定量的な基準がない。

**対策**:
- 製品の品質特性と公差から逆算して設備の精度要件を導出する
- PFMEA・コントロールプランと連動させ、管理特性ごとに数値を設定する
- 一般原則として「設備の繰返し精度 ≤ 製品公差の1/10」を目安とする
- MSA（測定システム解析）のP/T比（精度/公差比）で検証できる形にしておく

### 課題2: 設備仕様書が形式的で実態と乖離

**問題**: 設備メーカーに要求を出す前に仕様書を作成せず、設備納入後に後から作成するため、実態と乖離した文書になる。

**対策**:
- 工程設計の初期段階（PFMEA作成と同時期）に設備仕様書を作成する
- 設備仕様書を設備選定・発注の前提条件とする
- FAT・SATの合格基準を事前に設備仕様書に明記する

### 課題3: 受入検査（FAT/SAT）の合格基準が不明確

**問題**: 設備納入時に何をもって合格とするかが定義されていないため、検収が形式的になる。

**対策**:
- 仕様書に検証方法・測定方法・サンプル数・合格基準を具体的に記載する
- FAT・SAT記録シートを事前に作成し、全要件の検証結果を記録する
- 不合格項目は設備メーカーへのパンチリスト（是正要求リスト）として管理する

### 課題4: 改訂管理が不十分

**問題**: 設備改造や部品交換時に設備仕様書が更新されず、最新状態が不明になる。

**対策**:
- 設備変更管理プロセスに設備仕様書の改訂を組み込む
- 変更時にはCP・作業指示書とともに設備仕様書も必ず更新する
- IATF 16949の変更管理要求（条項8.5.6）に準拠する

### 課題5: 設備仕様書とコントロールプランの精度要件が不整合

**問題**: コントロールプランで定めた測定精度要件を、設備仕様書が満たしていない。

**対策**:
- コントロールプランの「評価・測定技法」カラムと設備仕様書の精度要件を照合するレビューを実施する
- MSA計画と設備仕様書を連動させ、測定システムの妥当性を事前検討する

## 8. ベストプラクティス

1. **工程フローチャートと連動して作成する**: 各工程の設備仕様書を一覧で管理し、設備管理番号でトレーサビリティを確保する
2. **PFMEA・CPと精度要件を整合させる**: 特殊特性（CC/SC）に対応した精度要件・ポカヨケ機能を仕様書に明記する
3. **ポカヨケ機能を積極的に要求する**: 「人のミスを設備で防ぐ」設計を仕様段階から盛り込む
4. **FAT・SAT合格基準を事前に詳細定義する**: 受入検査の曖昧さをなくし、設備メーカーとの認識齟齬を防ぐ
5. **設備メーカーとの技術協議を早期に実施する**: 設計段階で実現可能性を確認し、仕様の実現性リスクを低減する
6. **デジタルツイン・シミュレーションを活用する**: 設備導入前にデジタル環境で動作を検証し、問題を事前解消する
7. **設備能力（OEE）の目標値を明示する**: 稼働率・品質率・パフォーマンス率のOEE目標を仕様書に含める
