# 識別・トレーサビリティ設計（Identification & Traceability Design）

## 1. 概要

### 識別・トレーサビリティとは

識別・トレーサビリティ（Identification & Traceability）とは、製造工程における製品・部品・材料を一意に識別し、その製造履歴（どの材料が、どの工程を、いつ、どの設備で、誰が加工したか）を記録・追跡できる状態を保つ仕組みです。

IATF 16949:2016 条項 8.5.2「識別及びトレーサビリティ」で作成・維持が要求される管理システムであり、工程設計（Phase 3）の段階で識別方式・記録項目・データフローを設計します。

> **補足**: 「識別（Identification）」は製品や部品を区別・特定すること、「トレーサビリティ（Traceability）」はその製造履歴を遡及・追跡できることを指します。両者は一体的に設計します。

### 目的

| 目的 | 説明 |
|------|------|
| 品質問題の原因特定 | 不具合発生時に影響ロット・工程・材料を迅速に特定し、是正処置の範囲を限定する |
| リコール範囲の限定 | 不具合の影響範囲を正確に把握し、回収コストと顧客リスクを最小化する |
| 工程内混入防止 | 未検査品・不合格品・異種ロットの混入を識別によって防止する |
| 規制要求への対応 | 自動車・医療・食品分野の法規制（製造物責任・安全規制）への適合を証明する |
| 顧客要求への対応 | 顧客固有要求事項（CSR）のトレーサビリティ条件を満たす |
| 工程改善のデータ活用 | 製造履歴データを分析し、ばらつき低減・工程能力向上に活用する |

### 工程設計における位置づけ

```
製品図面・仕様書
（部品番号・ロット管理要件）
    │
    ▼
工程フローチャート
（工程数・工程順序の確定）
    │
    ▼
┌───────────────────────────────┐
│  識別・トレーサビリティ設計   │ ← 本記事
│  ・識別単位（ロット/個体）    │
│  ・識別媒体・記号体系         │
│  ・記録項目・データフロー     │
└───────────────────────────────┘
    │
    ├──→ コントロールプラン（識別・記録の管理項目に反映）
    ├──→ 作業指示書（識別作業・記録作業の手順に反映）
    ├──→ 設備仕様書（ラベラ・バーコードリーダ等の設備要件に反映）
    └──→ 工程設計書（識別・トレーサビリティ設計の概要を記載）
```

## 2. トレーサビリティの種類

### 追跡方向による分類

| 種類 | 英語 | 方向 | 用途 |
|------|------|------|------|
| **フォワードトレーサビリティ** | Forward Traceability | 材料・部品 → 製品 | 「この材料はどの製品に使われたか」を追跡。リコール時の対象製品特定に使用 |
| **バックワードトレーサビリティ** | Backward Traceability | 製品 → 材料・部品 | 「この製品はどの材料・部品から作られたか」を遡及。品質問題の原因調査に使用 |
| **双方向トレーサビリティ** | Bidirectional Traceability | 双方向 | 上記両方を実現。IATF 16949・航空宇宙・医療分野で要求されることが多い |

### 管理単位による分類

| 単位 | 説明 | 適用例 |
|------|------|--------|
| **ロット管理** | 同一条件で製造した一群（ロット）を識別単位とする。ロット内の個体は区別しない | 射出成形品、プレス品、電子部品（同一ロット） |
| **個体管理（シリアル管理）** | 製品1個ごとに固有識別子（シリアル番号）を付与する。最も高いトレーサビリティを実現 | 自動車エンジン、医療機器、高付加価値組立品 |
| **バッチ管理** | 同一の製造バッチ（日付・シフト・設備単位等）を識別単位とする | 化学品、塗料、接着剤、食品 |

> **補足**: 管理単位は「トレーサビリティの必要精度」と「管理コスト」のバランスで決定します。ロット管理は運用コストが低い反面、問題発生時の影響範囲がシリアル管理より広くなります。

### 工程内トレーサビリティ

製品が複数工程を経る過程での識別状態を管理します。

| 状態区分 | 説明 | 識別方法の例 |
|----------|------|-------------|
| 受入前（未検査） | 受入検査前の状態。合否不明 | 黄色タグ / 専用エリア保管 |
| 受入合格 | 受入検査合格品 | 緑色タグ / 合格ラベル貼付 |
| 仕掛品 | 工程途中の半製品 | 工程識別タグ・トラベラカード |
| 不合格品 | 検査不合格品。選別・廃棄待ち | 赤色タグ / 隔離エリア保管 |
| 特採品 | 特別採用で使用を認めた品 | 特採承認書を添付 |
| 完成品 | 全工程完了・出荷検査合格品 | 出荷ラベル貼付 |

## 3. トレーサビリティシステムの設計

### 設計の進め方

```
Step 1: 要求事項の確認
（顧客CSR・法規制・IATF 16949要求事項の棚卸し）
    │
    ▼
Step 2: 識別単位の決定
（ロット / 個体 / バッチの選定）
    │
    ▼
Step 3: 識別媒体・記号体系の設計
（ラベル・バーコード・QRコード・RFID等の選定）
    │
    ▼
Step 4: 記録項目の設計
（各工程で記録するデータ項目の定義）
    │
    ▼
Step 5: データフローの設計
（記録・保管・参照のシステム構成の設計）
    │
    ▼
Step 6: 記録保管期間の設定
（法規制・顧客要求・製品寿命に応じた保管期間の定義）
```

### 識別単位の決定基準

| 判断要素 | ロット管理 | 個体管理（シリアル） |
|----------|-----------|---------------------|
| 顧客要求 | ロット識別の要求 | シリアル識別の要求 |
| 製品安全リスク | 低〜中 | 高（安全部品・CC特性品） |
| 製品単価 | 比較的低い | 高い |
| 製造数量 | 大量生産 | 少量〜中量 |
| 管理コスト | 低い | 高い |
| リコール時の影響範囲 | ロット全体 | 特定個体のみ |

> **重要**: 特殊特性（CC / SC）を持つ部品・工程は、ロット管理では影響範囲が広すぎる場合があります。安全部品は個体管理を検討してください。

### 記録項目の設計

各工程で記録すべき標準的なデータ項目を以下に示します。

| カテゴリ | 記録項目 | 説明 |
|----------|----------|------|
| **識別情報** | 部品番号・品名 | 対象部品の識別 |
| | ロット番号 / シリアル番号 | 製造単位の識別 |
| | 製造数量 | ロットのサイズ |
| **材料情報** | 原材料ロット番号 | 使用した原材料の追跡 |
| | 部品ロット番号 | 使用した購入部品の追跡 |
| | 仕入先情報 | 材料・部品の調達元 |
| **工程情報** | 工程番号・工程名 | 実施工程の識別 |
| | 製造日時 | 製造タイムスタンプ |
| | 設備番号 | 使用設備の識別 |
| | 作業者ID | 担当作業者の識別 |
| | 工程条件（パラメータ） | 温度・圧力・速度等の設定値と実績値 |
| **検査情報** | 検査結果（合否） | 検査の判定結果 |
| | 測定値 | 実測データ |
| | 検査日時・検査者ID | 検査のタイムスタンプと担当者 |

### 記録保管期間の設定基準

| 根拠 | 保管期間の考え方 |
|------|-----------------|
| 顧客要求 | 顧客が指定する保管年数に従う（例：15年、廃番後10年） |
| 製品寿命 | 製品の使用期間中は遡及できることが望ましい |
| IATF 16949 条項 7.5.3.2 | 文書化した情報の保管期間は組織が決定する |
| 製造物責任法（PL法） | 日本では製造物の引渡しから10年以内の請求を想定して保管 |
| 自動車業界慣行 | 最終納入から15年以上が一般的（顧客CSRで確認） |

## 4. 識別媒体と識別手段

### 識別媒体の種類と特徴

| 識別媒体 | 特徴 | 適用場面 |
|----------|------|----------|
| **紙ラベル（バーコード印刷）** | 安価・汎用的。高温・油・水に弱い場合あり | 一般的な工程内識別、梱包ラベル |
| **金属プレート / 刻印** | 耐久性が高い。恒久識別が必要な部品に有効 | エンジン、変速機等の機能部品 |
| **バーコード（1次元）** | リーダが安価。情報量はQRコードより少ない | 部品番号・ロット番号の識別 |
| **QRコード（2次元）** | 高情報量。小面積に多くのデータを格納できる | 複合情報（部品番号+ロット+日付等）の一括識別 |
| **RFID タグ** | 非接触読取・複数同時読取が可能。コストが高い | パレット・治具・金型の管理、自動倉庫システム |
| **データマトリクス（DM）** | 部品への直接刻印（Direct Part Marking）に対応 | 機能部品への永久識別（自動車部品の個体管理） |

### バーコード体系の設計

工場内で統一されたバーコード体系を設計することで、読取エラーや識別ミスを防止します。

```
バーコード（例：QRコード）に含める情報の構成例

┌────────────────────────────────────────┐
│ 部品番号   : PX-500A                   │
│ ロット番号 : LOT250115-001             │
│           （YYYYMMDD-通し番号）         │
│ 製造日     : 2025-01-15                │
│ 製造ライン : LINE-1                    │
└────────────────────────────────────────┘
```

> **補足**: 自動車産業ではAIAG B-10（バーコード規格）、AIAG B-4（ラベル規格）が参照されることが多いです。顧客からラベルフォーマットが指定される場合はCSRを確認してください。

### 識別の「抹消・引継ぎ」ルール

識別情報は、製品の状態変化（工程完了・合格・不合格・分割・合体）に合わせて適切に更新します。

| 状態変化 | 識別の取り扱い |
|----------|---------------|
| 工程完了・合格 | 工程完了スタンプを押す / 次工程ラベルに貼り替える |
| ロット分割 | 分割後の各ロットに新ロット番号を付与し、元ロット番号を親として記録する |
| 梱包（複数個を1梱包に） | 梱包ラベルに含まれる個体またはロットの番号を全て記録する |
| 不合格・隔離 | 不合格タグに交換または不合格印を押す。合格ラベルは除去する |

### トレーサビリティデータフロー

```
受入段階
  受入ラベル（紙）
      │ 受入検査合格後
      ▼
フレーム加工（工程20）
  シリアル番号刻印（フレームに永久刻印）
  ＋トラベラカード（紙）でロット番号を追跡
      │
      ▼ 組立・搭載工程（40〜60）
  トラベラカードに工程ごとの記録を追記
  各工程完了後にスキャン → MES に電子記録
      │
      ▼ 調整・検査工程（70〜90）
  全記録を MES（製造実行システム）に電子登録
  シリアル番号をキーに測定値・判定結果を紐付け
      │
      ▼ 出荷（工程100）
  QRコード出荷ラベル（部品番号＋シリアル番号＋出荷日）
  MES から出荷実績を出力し、記録保管
```

### フォワード・バックワードトレーサビリティの運用例

**【バックワードトレーサビリティの例：品質問題発生時】**

```
顧客から印字不良の報告（シリアル番号: SN250115-0042）
    │
    ▼
MES でシリアル番号 SN250115-0042 を検索
    │
    ▼
工程70（校正）の設備番号・校正パラメータ実績を確認
  → 設備 CAL-02 の校正値が基準外であることを発見
    │
    ▼
影響ロットの特定（同日・同設備で校正した個体を抽出）
  → 対象: SN250115-0038 〜 SN250115-0055 の18台
    │
    ▼
該当18台に対し追加確認・必要に応じてリコール処置
```

**【フォワードトレーサビリティの例：部品不具合対応】**

```
ヘッド仕入先から特定ロット（LOT250110-007）の不具合連絡
    │
    ▼
MES で LOT250110-007 の使用実績を検索
    │
    ▼
該当ヘッドを使用した製品のシリアル番号リストを抽出
  → SN250112-0001 〜 SN250114-0080 のうち 23台
    │
    ▼
在庫品: 即時隔離
出荷済み: 顧客への連絡・回収処置
```

## 6. よくある課題と対策

### 課題1: トレーサビリティ記録の漏れ・抜け

| 項目 | 内容 |
|------|------|
| **課題詳細** | 作業者が記録作業を忘れる、または省略する。紙記録の場合は記入漏れが発生しやすい |
| **対策** | 記録をポカヨケ化する。「記録が完了しないと次工程に進めない」仕組み（MES・生産管理システムとの連動）を設計する。記録作業を工程設計の標準時間に含め、プレッシャーによる省略を防ぐ |

### 課題2: ラベルの紛失・剥がれ

| 項目 | 内容 |
|------|------|
| **課題詳細** | 製造環境（油・熱・水）によってラベルが剥がれ、識別不能になる |
| **対策** | 製造環境に適した材質・接着剤のラベルを選定する。高温・油環境ではアルミ製ラベルや直接刻印（Direct Part Marking）を採用する。ラベル仕様を設備仕様書・梱包仕様書に明記する |

### 課題3: 読取エラー・バーコード損傷

| 項目 | 内容 |
|------|------|
| **課題詳細** | バーコードやQRコードの汚れ・傷により、スキャナーで読み取れなくなる |
| **対策** | 印字品質基準（ANSI/AIM グレード規格）を設備仕様書に明記する。バーコード検証機（ベリファイア）による印字品質チェックを受入検査または印字工程後に組み込む。QRコードはエラー訂正機能があるため、一部損傷しても読取可能 |

### 課題4: 紙記録からのデータ転記ミス

| 項目 | 内容 |
|------|------|
| **課題詳細** | 紙の記録を後からPCに手入力する際に転記ミスが発生する |
| **対策** | 紙記録と電子記録の二重管理を廃止し、現場でのリアルタイム電子入力（タブレット・PC端末）またはスキャン入力に切り替える。移行が困難な場合は、入力後のベリファイ（読み上げ確認）を手順に組み込む |

### 課題5: ロット分割時の識別管理

| 項目 | 内容 |
|------|------|
| **課題詳細** | ロットを複数に分割した際、分割後のロットと元ロットの関係記録が残らず、遡及が困難になる |
| **対策** | ロット分割のルール（分割後ロット番号の付番規則、元ロット番号との紐付け記録）を作業指示書に明記する。分割記録は製造管理システムで自動的に親子関係として記録する |

### 課題6: 記録保管期間の管理

| 項目 | 内容 |
|------|------|
| **課題詳細** | 記録の保管期間を設定せずに運用した結果、品質問題発生時に必要な記録が廃棄済みになっている |
| **対策** | 顧客CSR・法規制・製品寿命に基づき保管期間を文書で定める。電子記録システムに廃棄スケジュールを設定する。記録保管の要求事項をQC工程表・コントロールプランに明記する |

## 7. ベストプラクティス

1. **トレーサビリティ設計は工程設計の早期段階で着手する**: ラベラ・バーコードリーダ等の設備要件が確定するため、設備仕様書作成前にトレーサビリティ方式を決定する

2. **「記録の自動化」を優先する**: 作業者の手入力に頼る記録は抜け・誤りが生じやすい。スキャン・センサー・MESとの連動により自動記録を実現する

3. **最小限の記録項目から始める**: 記録項目が多すぎると運用負荷が増加し、記録の精度が下がる。まず必須項目（誰が・何を・いつ・どの設備で）を確実に記録し、必要に応じて拡張する

4. **顧客CSRをトレーサビリティ設計の起点とする**: 各顧客が要求するラベルフォーマット・データ項目・保管期間を事前に確認し、設計に反映する

5. **バックワードトレーサビリティを定期的に訓練する**: 品質問題発生時に実際に使える状態を維持するため、定期的に架空の問題シナリオで遡及演習（Traceability Drill）を実施する

6. **電子記録と物理識別の二重管理を設計する**: システム障害時にも最低限の識別ができるよう、製品自体の物理識別（刻印・ラベル）とMES等の電子記録を補完的に設計する

7. **不合格品の識別・隔離を最優先で設計する**: 工程内での不合格品混入は重大なリスク。不合格品の識別（赤タグ・不合格エリア）と合格品の識別（緑タグ・合格エリア）を明確に区別する
