# 工具要求仕様書（Tool Requirement Specification）

## 1. 概要

### 工具要求仕様書とは

工具要求仕様書（Tool Requirement Specification）とは、製造工程で必要となる治工具（Jig・Fixture・Gauge・Special Tool）に対して、**工程設計技術課が生産技術課へ発行する要求仕様文書**です。

「何を達成すべきか（What）」を記述する文書であり、「どのように設計・製作するか（How）」は記述しません。治工具の設計方針・構造・材質の決定は、この文書を受け取った生産技術課が担う責務です。

### 目的

| 目的 | 説明 |
|------|------|
| 要求の明文化 | 工程設計技術課が持つ品質・機能・性能の要求を、設計者が解釈できる形で文書化する |
| 部門間の合意形成 | 要求側（工程設計技術課）と設計側（生産技術課）の認識を統一し、手戻りを防ぐ |
| 受入基準の確立 | 治工具完成時の検収判断基準を事前に合意しておく |
| トレーサビリティ | PFMEAの故障モード・コントロールプランの管理特性と、治工具の機能要求を紐付ける |
| 変更管理 | 後工程での仕様変更が発生した場合、要求の起点を明確にする |

### 工程設計における位置づけ

```
PFMEA / コントロールプラン（故障モード・管理特性・ポカヨケ要求）
    │
    ▼
工具要求仕様書（Tool Requirement Specification）◀ 本記事
  【発行】工程設計技術課
  【内容】何を達成する治工具が必要か（機能・性能・制約・受入条件）
    │
    ▼
治工具設計仕様書（Jig & Fixture Design Specification）
  【作成】生産技術課
  【内容】要求を満たすための設計方針・構造・材質・精度根拠
    │
    ▼
治工具設計図（Jig & Fixture Design Drawing）
  【作成】生産技術課
  【内容】製作可能な詳細図面（形状・寸法・公差・部品表）
    │
    ▼
治工具製作
  【実施】治具メーカー（外注）
    │
    ▼
検収・受入（工具要求仕様書の受入条件に照らして合否判定）
```

### 組織間の役割分担

| 組織 | 役割 | 主な成果物 |
|------|------|-----------|
| 工程設計技術課 | 工程設計の観点から「何が必要か」を定義する | 工具要求仕様書 |
| 生産技術課 | 要求を受けて「どう実現するか」を設計する | 治工具設計仕様書、治工具設計図 |
| 治具メーカー | 設計図に基づいて治工具を製作する | 製作品（治工具本体） |

> **重要**: 工具要求仕様書は工程設計技術課の「発注意図」を表す文書です。設計の解を押しつけるのではなく、要求（要件）を正確に伝えることが求められます。設計解を記述しすぎると生産技術課の最適設計を妨げます。

## 2. 文書体系と3文書の関係

### 3文書の記述範囲の違い

| 観点 | 工具要求仕様書 | 治工具設計仕様書 | 治工具設計図 |
|------|-------------|--------------|------------|
| 発行者 | 工程設計技術課 | 生産技術課 | 生産技術課 |
| 主な問い | **What / Why** | **How（方針）** | **How（詳細）** |
| 構造・材質 | 記載しない（制約がある場合のみ例外） | 設計方針として記述 | 詳細指定 |
| 寸法・公差 | 要求精度のみ（例：繰返し精度±0.05mm） | 精度根拠・公差配分 | 具体的な公差値 |
| ポカヨケ | 「誤方向防止が必要」の要求レベル | 採用する防止方式 | 具体的な形状・構造 |
| 受入条件 | 機能確認・性能測定の合否基準 | 検証方法の設計 | 検査項目・検査寸法 |

### ドキュメントフロー

```
工程フローチャート ─→ 工程No.と作業内容
PFMEA             ─→ 故障モード・ポカヨケ要求・管理特性
コントロールプラン ─→ 管理特性・測定方法・頻度
設備仕様書         ─→ 設備側インターフェース（取付座・エア配管等）
製品図面           ─→ ワーク形状・寸法・基準面
          └──────────────────────────────────┐
                                            ▼
                                   工具要求仕様書（本文書）
                                            │
                                            ▼
                                   治工具設計仕様書
                                   （生産技術課が作成）
                                            │
                                            ▼
                                   治工具設計図
                                   （生産技術課が作成）
```

## 3. 記載項目と粒度

### 3.1 ヘッダー情報

| 項目 | 粒度・記載内容 |
|------|-------------|
| 文書番号 | TR-[製品コード]-[工程No.]-[連番]（例：TR-PX500-090-01） |
| 版数 | Rev.00（初版）から管理 |
| 発行日 / 改訂日 | 発行日と最新改訂日 |
| 発行部署 | 工程設計技術課（担当者名・承認者名） |
| 宛先 | 生産技術課（担当者名） |
| 対象製品 | 製品名・品番・適用ライン |
| 対象工程 | 工程フローチャートの工程No.・工程名 |
| 対応設備 | 治工具を取り付ける設備の名称・設備管理番号 |
| 関連文書 | PFMEA番号、コントロールプラン番号、設備仕様書番号 |

**粒度の目安**: 追跡・改訂管理ができる最低限の識別情報を記載する。設計者が迷わず参照文書を特定できることが目標。

---

### 3.2 要求背景（なぜこの治工具が必要か）

この項目は記載が省略されやすいが、**設計者が要求の意図を理解するうえで最も重要**な情報です。

| 項目 | 粒度・記載内容 |
|------|-------------|
| 工程での課題・リスク | PFMEAの故障モードとの対応関係（FM番号を引用） |
| 管理特性との対応 | コントロールプランの管理特性番号・特性種別（CC/SC） |
| 現状の問題点 | 既存治工具や手作業での問題（精度不足・見落とし・ばらつき等） |
| 規格・基準の要求 | IATF 16949、顧客固有要求等で定められている場合はその条項 |

**粒度の目安**: 「なぜこの治工具を作るのか」を読んだだけで理解できるレベル。3～5文程度で記述する。

---

### 3.3 機能要求（Function Requirements）

治工具が果たすべき機能を列挙します。**「○○を△△できること」の形式**で記述します。

| 機能カテゴリ | 記載例 |
|------------|-------|
| 位置決め機能 | ワークを基準面に対して繰返し精度±0.05mm以内で位置決めできること |
| 固定機能 | 所定の作業荷重に対してワークがずれないよう固定できること |
| 検出機能 | 規定の漏れ量（○○mL/min相当）以上のリークを全数検出できること |
| ポカヨケ機能 | ワークの誤方向セットを物理的に防止できること（逆方向ではセット不可） |
| 自動判定機能 | 検査結果をGo/No-Go（合格/不合格）として自動出力できること |
| インターフェース機能 | 設備（EQ-○○○）の取付座に適合し、所定の操作で着脱できること |

**粒度の目安**: 機能の「達成基準」まで記述する。「どんな構造で実現するか」は記述しない。機能ごとに1文で記述できる粒度が適切。

---

### 3.4 性能要求（Performance Requirements）

各機能を「どの程度」達成する必要があるかを数値で定義します。

| 性能項目 | 粒度・記載内容 |
|---------|-------------|
| 位置決め精度 | 繰返し精度の数値（例：±0.02mm）と測定条件（測定回数等） |
| 検出感度 | 検出できる最小の欠陥量（例：リーク量 0.1mL/min以上を検出） |
| サイクルタイム | 治工具操作（セット～判定完了～取出し）に許容される時間（秒） |
| 耐久寿命 | 想定される使用回数・年数（例：100万サイクル以上） |
| 測定確度 | 測定値の許容誤差（例：設定値の±5%以内） |
| 誤検知率 | 良品を不合格と判定する確率の上限（例：0.1%未満） |

**粒度の目安**: 測定・検証が可能な数値で記述する。「高精度に」「十分な感度で」といった定性的な表現は不可。品質特性の公差・コントロールプランの管理値を起点に導出すること。

---

### 3.5 制約条件（Constraints）

設計に先立って確定している制約事項を記述します。設計の自由度を意図的に制限する場合のみ記載します。

| 制約カテゴリ | 粒度・記載内容 |
|------------|-------------|
| 設備インターフェース | 取付ボルト径・ピッチ、設備側Tスロット寸法、最大取付高さ等 |
| エネルギー・ユーティリティ | 使用可能なエア圧（例：0.5±0.05 MPa）、電圧・信号仕様 |
| ワーク接触面の制約 | 外観面への接触禁止、接触可能面の指定 |
| 設置スペース | フロアプランで確保されているスペース寸法（W×D×H） |
| 操作者の制約 | 片手操作が必要、操作力の上限（例：50N以下）等 |
| 保全・清掃の制約 | 工具なしで分解・清掃できること等 |
| コスト上限 | 予算上限（外注製作費の目標値） |
| 納期 | 量産トライ開始日に合わせた治工具完成期限 |

**粒度の目安**: 設計者が設計判断を行うために必要かつ十分な制約のみを記載する。不必要な制約の追加は設計最適化を妨げるため避ける。

---

### 3.6 品質要求（Quality Requirements）

治工具の品質・信頼性・安全性に関する要求を記述します。

| 項目 | 粒度・記載内容 |
|------|-------------|
| 特殊特性への対応 | CC/SCに対応する工程では、ポカヨケ設計の必須化を明記 |
| 精度維持期間 | 規定精度を維持すべき期間・使用回数 |
| 保全可能性 | 摩耗部品の交換が工具なし or 社内保全班のみで可能なこと |
| 安全性 | 作業者の挟まれ・接触リスクを排除する設計要求 |
| 識別管理 | 管理番号・校正期限ラベルの貼付可能スペースの確保 |

---

### 3.7 受入条件（Acceptance Criteria）

治工具の完成時に、工程設計技術課と生産技術課が合否を判定する基準を事前に合意します。これが記載されていないと、検収が属人的になります。

| 受入検証項目 | 粒度・記載内容 |
|------------|-------------|
| 寸法検証 | 位置決め精度の三次元測定結果が要求値以内であること |
| 機能確認 | ポカヨケ動作（逆向きNGの確認）、センサー検知動作確認 |
| 性能確認 | 規定サイクルの繰返し試験（例：100回着脱）後に精度維持 |
| 実機ワーク試験 | 実際のワークをセットして規定の検査が正しく実施できること |
| 自動判定確認 | 意図的な不良ワーク（マスターサンプル）で正しくNG判定されること |
| 量産トライ | 量産条件での連続稼働試験（台数・時間の要件を記載） |

**粒度の目安**: 合否の判断が客観的にできるよう、測定方法・使用する基準器・判定値をセットで記述する。

## 4. 粒度レベルの判断基準

### 記載過少（Under-specification）の典型例

| 問題のある記述 | 何が不足しているか |
|-------------|----------------|
| 「高精度に位置決めできること」 | 数値がなく、検証不可能 |
| 「インク漏れを検出できること」 | 検出感度・検出方式・判定基準が不明 |
| 「操作しやすい治具とすること」 | 定性的すぎて設計者が判断できない |
| 「PFMEA対応のポカヨケを設ける」 | どの故障モードへの対応か不明 |

### 記載過多（Over-specification）の典型例

| 問題のある記述 | 何が問題か |
|-------------|---------|
| 「ベースプレートはS50C材、厚さ15mmとする」 | 材質・寸法は設計者が最適化すべき事項 |
| 「トグルクランプを2個使用すること」 | クランプ方式の選定は生産技術課の担務 |
| 「ロケーターピン径はφ5mm」 | 設計解を記述しており、工具要求仕様書の範囲外 |
| 「○○社製エアシリンダーを使用する」 | 特定メーカー指定は調達制約に記載すべき例外事項 |

### 粒度レベルの目安

```
                 粒度レベル（低 ─────────────────── 高）

記述例           定性記述    要件記述    数値記述     設計指定
                 ↓           ↓           ↓            ↓
例             「高精度」  「CC特性対応」  「±0.05mm」  「SKD11 HRC60」

工具要求仕様書      ✗           ✓           ✓            ✗（原則）
治工具設計仕様書    ✗           ✓           ✓            ✓
治工具設計図       ✗           ✗           ✓            ✓
```

## 5. 具体例：PX-500 インクリーク検出治工具 工具要求仕様書

PX-500の**工程No.90（インク供給系組立）**後に実施するエアリーク検査に使用する治工具の工具要求仕様書を例示します。インクリークは重要特性（◆CC）であり、プリンタにとって最重大リスクのひとつです。

```
═══════════════════════════════════════════════════════════════════
工具要求仕様書
                                          文書番号: TR-PX500-090-01
                                          版数: Rev.00
                                          発行日: 2026/01/10
═══════════════════════════════════════════════════════════════════

【ヘッダー情報】

  発行部署: 工程設計技術課
  担当者:   山田 太郎（内線: 3421）
  承認者:   工程設計技術課長 田中 一郎
  宛先:     生産技術課 鈴木 次郎

  対象製品: インクジェットプリンタ PX-500（品番: PX-500A）
  対象工程: 工程No.90 インク供給系組立（インクリーク検査）
  適用ライン: プリンタ組立ライン #1
  対応設備: エアリーク試験機 EQ-090

  関連文書
    ・PFMEA:          FM-PX500-R01（工程No.90 故障モードNo.14）
    ・コントロールプラン: CP-PX500-R01（管理特性No.9 インクリーク）
    ・設備仕様書:      EQ-PX500-090-R01

═══════════════════════════════════════════════════════════════════
【1. 要求背景】
═══════════════════════════════════════════════════════════════════

  インク供給系組立（工程No.90）では、インクチューブのコネクタ接続部・
  インクカートリッジ装着部・プリントヘッド接続部（計8箇所）を組み付ける。

  PFMEA（FM番号14）において、コネクタ接続不良によるインク漏れを
  重大故障モードとして特定済みである（影響度: 製品全損・顧客クレーム、
  発生頻度: 工程由来で中程度）。インクリーク 0 mL/min は重要特性
  （◆CC）であり、全数検査が要求されている（CP-PX500-R01 参照）。

  現状、組立後の目視確認のみでは微小リーク（毛細管現象による浸潤型の
  漏れ）の検出が困難であり、組立完了後の通電検査（工程No.100）で
  インク汚染による基板ショートが発見されるケースが月平均2件発生している。

  本治工具により、工程No.90完了直後に全数エアリーク検査を実施し、
  微小リークを含む全てのリークを工程内で検出・排除することを目的とする。

═══════════════════════════════════════════════════════════════════
【2. 機能要求】
═══════════════════════════════════════════════════════════════════

  F-01: ワーク保持機能
        PX-500本体（インク供給系組立完了状態）を安定して保持し、
        エアリーク試験機（EQ-090）のエア接続操作・切離し操作の間、
        ワークが動かない状態を維持できること。

  F-02: インク流路の気密封止機能
        インク供給系の全インク流路入口・出口（計8箇所）を外部から
        気密封止し、試験用エアを流路内に閉じ込められること。
        各封止箇所の封止／開放操作が独立して行えること。

  F-03: エアリーク試験機への接続機能
        封止した流路をエアリーク試験機（EQ-090）の測定ポートへ
        接続できること。流路は設備の測定チャンネル数（最大4ch）に
        対応してグループ分けできること。

  F-04: リーク検出・判定機能
        エアリーク試験機と連携し、各測定チャンネルの圧力降下量を
        測定して、合格／不合格を自動判定・表示できること。

  F-05: ポカヨケ機能
        ・ワークの前後・左右の誤方向セットを物理的に防止できること
          （誤方向では治工具にセット不可であること）。
        ・封止未完了の状態でエア供給が開始されないインターロックを
          有すること。

  F-06: 判定結果の出力機能
        判定結果（合格/不合格/測定チャンネル番号）を上位システム
        （生産管理システム）に出力できること（信号仕様はEQ-090の
        仕様に準ずる）。

═══════════════════════════════════════════════════════════════════
【3. 性能要求】
═══════════════════════════════════════════════════════════════════

  P-01: 検出感度
        インク流路内のリーク量 0.1 mL/min（空気換算）以上を検出
        できること。
        ※根拠: 顧客クレームが発生した不良品の最小リーク量を
          分析した結果、0.1 mL/min 以上のリークで組立工程内に
          インク汚染が発生することが確認されている。

  P-02: 誤検知率（偽陰性）
        リーク量 0.1 mL/min 以上の不良品を合格と誤判定する確率が
        0.01%未満であること（= 10,000台に1台未満）。

  P-03: 過検知率（偽陽性）
        リーク量 0.05 mL/min 未満の良品を不合格と誤判定する確率が
        0.1%未満であること（= 1,000台に1台未満）。

  P-04: 測定精度
        圧力測定値の繰返し誤差が ±0.5 kPa 以内であること
        （同一ワーク・同一条件での連続10回測定で評価）。

  P-05: サイクルタイム
        ワークのセット完了から測定完了・判定出力までの時間が
        60秒以内であること（設備全体のタクトタイム 90秒に対して
        余裕をもって収まること）。
        内訳目標: 封止操作 15秒 + 加圧・安定待ち 30秒 + 測定 10秒
                  + 判定出力 5秒

  P-06: 繰返し位置決め精度
        ワークを繰返しセット（100回）した際の封止部位置ずれが
        ±0.3mm以内であること（封止圧力の安定性に影響するため）。

  P-07: 耐久寿命
```

## 6. 偽陽性率・偽陰性率の設計と検証

セクション5の具体例では、P-02（偽陰性率 < 0.01%）・P-03（偽陽性率 < 0.1%）として数値目標が定められ、受入試験 R-03（10/10 NG判定）・R-04（30/30 合格判定）が検収基準として設定されている。しかし工具要求仕様書は「要求値を記載する文書」であり、**その値を達成するための設計指針**と**達成を統計的に検証する方法**は別途対処が必要である。本セクションではその乖離と対処法を解説する。

---

### 6.1 受入試験と統計的保証の乖離

受入試験 R-03・R-04 は「この治工具が正しく動作する」ことの**機能確認試験**であり、「偽陰性率が0.01%未満である」ことの**統計的証明**ではない。

ゼロ欠陥試験（不良ゼロを観測した場合）の率の95%上限は Clopper-Pearson 法（または近似のルール・オブ・スリー）で求まる。

```
不良発見数 k = 0、サンプル数 n のとき：

  率の95%上限 = 1 − 0.05^(1/n)   （Clopper-Pearson 厳密法）

  または近似：率の95%上限 ≈ 3/n   （ルール・オブ・スリー）
```

| 受入試験 | n | 95%上限 | P-02/P-03 の目標値 | 乖離 |
|---------|---|---------|-------------------|------|
| R-03（FNR確認） | 10 | ≤ 25.9% | < 0.01% | ×（目標の2,590倍） |
| R-04（FPR確認） | 30 | ≤ 9.5% | < 0.1% | ×（目標の95倍） |
| FNR 統計証明に必要な n | 29,957 | ≤ 0.01% | < 0.01% | ○ |
| FPR 統計証明に必要な n | 2,995 | ≤ 0.1% | < 0.1% | ○ |

P-02/P-03 を単独の受入試験で直接証明するには数万台規模のサンプルが必要であり、現実的ではない。

> **ポイント**: R-03・R-04 は「動作確認（機能検証）」として必要かつ重要である。一方、工具要求仕様書に記載する偽陰性率・偽陽性率は**設計目標値**として設定し、その達成は生産技術課による設計段階の MSA と工程設計技術課による量産監視の組み合わせで検証するのが正しい進め方である。

---

### 6.2 生産技術課による設計アプローチ

工具要求仕様書を受け取った生産技術課は、P-02・P-03 を達成できる治工具を設計するために以下3ステップを踏む。

#### 6.2.1 センサー測定原理の選定とS/N比の確認

偽陰性率・偽陽性率はセンサーの**信号対雑音比（S/N比）**に直結する。良品と不良品の測定値分布が十分に分離していなければ、どのように閾値を設定してもどちらかの誤判定率が悪化する。

```
                良品（リーク無）            不良品（0.1 mL/min 以上）
                   分布                          分布
  頻度              ╭──╮                          ╭──╮
  （度数）          ╱    ╲                        ╱    ╲
                   ╱      ╲                      ╱      ╲
                  ╱        ╲                    ╱        ╲
  ───────────────────────────────────────────────────────── 測定値
                        ▲ 閾値
              FPR ◀────┤────▶ FNR
             （誤排除）       （見逃し）
```

**信号分離幅の目安**: 良品・不良品の平均値の差（Δμ）がそれぞれの標準偏差（σ）の5倍以上（Δμ/σ ≥ 5）であれば、閾値の最適化で P-02・P-03 の両立が可能になる。これが技術的に困難な場合は測定原理（物理量・センサー種別）を見直す。

生産技術課はまず**計量値データ**でリーク量の分布を測定し、良品・不良品の分布形状と分離幅を定量的に把握してから閾値設計に進む。

#### 6.2.2 ROC 曲線による閾値最適化

閾値を変えると偽陽性率と偽陰性率はトレードオフの関係で変化する。P-02（FNR ≤ 0.01%）と P-03（FPR ≤ 0.1%）を同時に満たす閾値帯が存在するかを ROC 曲線で確認する。

| 閾値設定 | 偽陰性率（FNR） | 偽陽性率（FPR） | P-02/P-03 満足 |
|---------|--------------|--------------|---------------|
| 高すぎ（保守的） | 0.001% | 5.0% | P-02 ○ / P-03 × |
| 最適帯（例） | 0.005% | 0.05% | **P-02 ○ / P-03 ○** |
| 低すぎ（寛容的） | 2.0% | 0.01% | P-02 × / P-03 ○ |

ROC 曲線分析は**試作品または類似設備の測定データ**（良品 50〜100台、境界品・不良品 10〜20台）を取得して実施する。P-02・P-03 を同時に満たす閾値帯が確認できた場合に設計承認とする。

#### 6.2.3 計量値ゲージR&R（Variable MSA）による測定システム確認

閾値設定前に測定システム自体の能力を確認する。計量値ゲージR&R（繰返し性・再現性解析）を実施し、以下の基準を満たすことを確認する。

| 評価指標 | 目標値 | 意味 |
|---------|-------|------|
| %GRR（測定誤差の比率） | < 10% | 測定誤差が製品ばらつきの10%未満 |
| Ndc（管理用カテゴリ数） | ≥ 5 | 良品〜不良品を5段階以上に識別できる |

```
Ndc = 1.41 × (σ_部品 / σ_測定システム)

  Ndc ≥ 5：良品と不良品の分布を識別できる測定システムとして合格
  Ndc < 5：測定システムの改良（センサー精度向上・ノイズ低減）が必要
```

%GRR が高い（測定誤差が大きい）場合は閾値を設定しても判定がばらつくため、まず測定システムを改善する。

| ステップ | 実施内容 | 実施者 | 合格基準 |
|---------|---------|-------|---------|
| 1. 測定原理確認 | 良品・不良品の分布測定、信号分離幅の確認 | 生産技術課 | Δμ/σ ≥ 5 |
| 2. 閾値最適化 | ROC 曲線分析、P-02/P-03 を同時満足する閾値帯の特定 | 生産技術課 | FNR ≤ 0.01% かつ FPR ≤ 0.1% |
| 3. Variable MSA | 計量値ゲージR&R 実施 | 生産技術課 | %GRR < 10%、Ndc ≥ 5 |

### 6.3 工程設計技術課による検証アプローチ

工程設計技術課は生産技術課が設計した検査システムを**3段階**で検証する。

#### 6.3.1 計数型ゲージR&R（Attribute Agreement Analysis）

AIAG MSA 4th Edition が規定する計数型（合否判定型）の測定システム解析。治工具が安定して同じ判定を下せるかを統計的に評価する。

**実施手順**:
1. 良品・不良品・境界品（判定が難しい品）を合計 **50件以上** 用意する（境界品：リーク量 0.05〜0.1 mL/min 付近を意図的に含める）
2. 同一アペレーター（または複数アペレーター）が同一サンプルを **2回以上** 測定する
3. **Cohen's κ（カッパ係数）** を算出し合否を判定する

| κ の値 | 評価 |
|--------|------|
| < 0.40 | 不合格（測定システムの信頼性が低い） |
| 0.40 〜 0.74 | 条件付き（改善を検討） |
| ≥ 0.75 | **合格**（測定システムの信頼性が確認できた） |

各アペレーター・アペレーター間ともに κ ≥ 0.75 を合格基準とする。κ が低い場合は測定手順の明確化・治工具の改良を行い、再測定する。

#### 6.3.2 ゼロ欠陥試験（R-03/R-04）の確率論的位置づけ

受入試験 R-03・R-04 は「機能が正しく動作すること」の確認であり、偽陰性率・偽陽性率の統計的証明ではない（6.1 参照）。両者の役割を以下の通り明確に区別する。

| 試験 | 目的 | 何が確認できるか | 何が確認できないか |
|-----|------|----------------|-----------------|
| R-03（10/10 NG判定） | 機能確認 | 設定閾値付近での動作 | FNR が 0.01% 未満であること |
| R-04（30/30 合格判定） | 機能確認 | 過検知なしでの安定動作 | FPR が 0.1% 未満であること |
| 計数型ゲージR&R（6.3.1） | システム能力確認 | 境界品を含む繰返し性・再現性 | 実生産での率の絶対値 |
| p 管理図（6.3.3） | 量産監視 | 率の経時変化・ドリフト検知 | 初期値の証明 |

> **重要**: R-03・R-04 は「量産開始に必要な最低限の動作確認」として必須だが、これが P-02・P-03 の「証明」に相当するとは運用上も文書上も主張しないこと。統計的な根拠は量産モニタリングで構築する。

#### 6.3.3 量産データによる継続的モニタリング（p 管理図）

偽陽性率・偽陰性率の実際の値は量産データの蓄積によって推定できる。**p 管理図**（属性管理図）を用いてリアルタイムに監視する。

**偽陽性率のモニタリング**（良品の誤排除）:
- 全数検査で「NG判定されたワーク」を後工程で再確認し、実際には良品だったものを記録する
- p̄ = 過去の偽陽性件数 / 検査台数 として管理図を構成する

```
p 管理図の管理限界（各サブグループ n 台）：

  中心線（CL） = p̄
  上限（UCL）  = p̄ + 3 × √( p̄ × (1 − p̄) / n )
  下限（LCL）  = max( 0, p̄ − 3 × √( p̄ × (1 − p̄) / n ) )

※ サブグループ推奨サイズ：n ≥ 500
  （FPR ≈ 0.1% のとき UCL ≈ 0.14%、変化を検出できる感度を確保）
```

**偽陰性率のモニタリング**（不良品の見逃し）:
- 偽陰性は「検査を通過した不良品」のため直接カウントが困難
- 代理指標として **下流工程の脱出不良率**（組立後検査・最終検査での検出数）と **市場クレーム数（リーク関連）** をトラッキングする
- 累積 **30,000 台でリーク関連不良ゼロ**なら、FNR < 0.01% を 95% 信頼度で間接的に支持できる（ルール・オブ・スリー：3/30,000 ≈ 0.01%）

| 検証手法 | 実施タイミング | 最小サンプル数 | 合格基準 | 目的 |
|---------|-------------|------------|---------|------|
| 計数型ゲージR&R | 量産トライ前（検収時） | 境界部品 50件以上 | κ ≥ 0.75 | 検査システム能力の確認 |
| R-03（ゼロ欠陥試験） | 検収時 | 10 回 | 10/10 NG | 動作確認（機能検証） |
| R-04（ゼロ欠陥試験） | 検収時 | 30 回 | 30/30 合格 | 動作確認（機能検証） |
| p 管理図（FPR） | 量産開始後・継続 | サブグループ n ≥ 500 | 管理限界内 | 偽陽性率の経時監視 |
| 脱出不良トラッキング（FNR代理） | 量産開始後・継続 | 累積 30,000 台以上 | リーク関連クレーム ゼロ | 偽陰性率の間接的支持 |

|---------|---------|-------|---------|
| 設計段階 | センサー測定原理の選定・良品/不良品分布の計測 | 生産技術課 | 信号分離幅（Δμ/σ ≥ 5） |
| 設計段階 | ROC 曲線分析による閾値決定 | 生産技術課 | TPR-FPR トレードオフ評価 |
| 設計段階 | 計量値ゲージR&R（Variable MSA） | 生産技術課 | %GRR < 10%、Ndc ≥ 5 |
| 検収段階 | 受入試験 R-03・R-04（機能確認） | 工程設計技術課・生産技術課 | ゼロ欠陥機能確認 |
| 検収段階 | 計数型ゲージR&R（Attribute MSA） | 工程設計技術課 | Cohen's κ ≥ 0.75 |
| 量産監視 | p 管理図による偽陽性率モニタリング | 生産技術課・品質保証課 | p 管理図 |
| 量産監視 | 脱出不良率による偽陰性率の間接推定 | 品質保証課 | 脱出不良トラッキング |

工具要求仕様書に記載する P-02・P-03 の数値は、PFMEA・顧客クレーム分析から設定した**設計目標値**である。この目標値を達成・維持するためには、生産技術課による **Variable MSA**（設計段階）、工程設計技術課による **Attribute MSA**（検収段階）、品質保証課・生産技術課による **p 管理図**（量産監視）の三段構成で検証チェーンを構築することが重要である。

## 7. 工具要求仕様書の記載水準チェックリスト

工具要求仕様書の発行前に、以下のチェックを行うことを推奨します。

### 7.1 必須確認項目

| チェック項目 | 確認内容 |
|------------|---------|
| ☐ PFMEA連動 | 工具要求の根拠となる故障モードを明記しているか |
| ☐ CC/SC対応 | 管理特性（CC/SC）に対応するポカヨケを機能要求に含めているか |
| ☐ 数値化 | 全性能要求が測定・検証可能な数値で記述されているか |
| ☐ 受入条件 | 検収の合否基準が客観的に判定できる形で記述されているか |
| ☐ 設計解の排除 | 材質・構造・部品名等の設計解が混入していないか |
| ☐ 制約の根拠 | 記載した制約条件に根拠があるか（不必要な制約は削除） |
| ☐ スケジュール | 量産トライ日程から逆算した治工具完成期限が明記されているか |

### 7.2 粒度の過不足確認

| 項目 | 記載過少の兆候 | 記載過多の兆候 |
|------|-------------|-------------|
| 機能要求 | 「適切に動作すること」等の曖昧表現がある | 特定の機構・部品名を指定している |
| 性能要求 | 数値がなく定性的な記述になっている | 製品図面の公差をそのまま転記している |
| 制約条件 | 設備インターフェース情報が不足している | 材質・板厚・加工方法まで指定している |
| 受入条件 | 「確認する」「問題がないこと」で終わっている | 測定箇所・測定機器・手順まで設計している |

## 8. よくある課題と対策

### 課題1: 要求仕様書と設計仕様書の境界が曖昧

**問題**: 工程設計技術課が設計解（材質・構造）を要求仕様書に記載してしまい、生産技術課の設計自由度が失われる。

**対策**:
- 「What（何を達成するか）」と「How（どう達成するか）」を意識的に分離する
- 性能要求には数値で書き、その数値の根拠（品質特性・コントロールプランとの対応）を明記する
- 設計解に見える記述は「制約」として分離し、その制約の理由を添える

### 課題2: 受入条件が検収後まで決まっていない

**問題**: 受入条件が明記されていないため、検収時に工程設計技術課と生産技術課で合否判定の基準が食い違う。

**対策**:
- 工具要求仕様書の発行時点で受入条件を記載し、生産技術課と事前合意する
- 「量産トライで問題が出なければOK」という曖昧な受入条件を排除する
- マスターワーク（リーク量既知のリファレンス品）の準備責任者を明記する

### 課題3: CC特性に対応する治工具要求が不十分

**問題**: 重要特性（◆CC）に関連する工程の治工具であるにもかかわらず、ポカヨケ要求・インターロック要求が記載されていない。

**対策**:
- PFMEA（故障モードの影響度S＝9/10のもの）を必ずインプットとして活用する
- CC/SC特性に対応する工程では、「全数検出」「自動判定」「データ記録」を機能要求の定型として含める
- 品質保証課を工具要求仕様書の承認者に加える

### 課題4: 仕様変更後に工具要求仕様書が更新されない

**問題**: 製品設計変更や工程変更が発生した際、治工具の要求仕様書だけが旧版のままになる。

**対策**:
- 製品設計変更・工程変更の影響評価プロセスに、工具要求仕様書の更新チェックを組み込む
- 治工具設計仕様書・治工具設計図の改訂が発生した場合、その上流の工具要求仕様書も同時に改訂する

## 9. ベストプラクティス

1. **PFMEAをインプットの起点にする**: 工具要求仕様書の機能要求は、PFMEAの故障モードと1対1で対応関係を持つよう整理する。「なぜその機能が必要か」をPFMEAのFM番号で説明できる状態にする

2. **受入条件を先に書く**: 「最終的にどう検証するか」を先に考えると、機能要求・性能要求が自然と測定可能な形で記述できる

3. **制約は最小限にする**: 治工具の設計最適化は生産技術課の専門知識を最大活用することで達成される。工程設計技術課が設計を先取りせず、真に必要な制約のみを記載する

4. **CC/SC工程では品質保証課を承認者に加える**: ポカヨケ要求の妥当性を品質保証の視点で確認することで、IATF 16949の要求事項への適合を確保する

5. **生産技術課との対話で要求を精緻化する**: 工具要求仕様書の初版はドラフトとして生産技術課に渡し、実現可能性の観点からフィードバックをもらって要求の精緻化を行う（ウォーターフォール的な一方通行にしない）

6. **マスターワークの準備計画を含める**: 検出感度確認・量産トライに必要なマスターワーク（良品・不良品リファレンス）の準備責任者・準備期限を受入条件に明記する

7. **量産開始の十分前に発行する**: 治工具の設計・製作・検収には通常8〜16週間を要する。量産トライ開始日から逆算し、十分な製作期間を確保した時点で工具要求仕様書を発行する
