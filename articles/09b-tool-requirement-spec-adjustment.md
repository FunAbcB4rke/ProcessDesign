# 調整工程・補正工程の工具要求仕様書

> **前提**: 本記事は [工具要求仕様書（09a）](./09a-tool-requirement-spec.md) の続編です。文書の基本構造（ヘッダー情報・機能要求の粒度・受入条件の書き方等）は 09a を参照してください。本記事では、検査工程とは性質の異なる**調整工程・補正工程**に特化した記載事項を解説します。

---

## 1. 概要

### 検査工程との本質的な違い

工具要求仕様書の多くは**検査工程**（リーク検査・寸法検査・電気検査）を対象としている。検査工程の機能要求は PFMEA とコントロールプランから導出でき、「何を合否判定するか」が出発点になる。

一方、**調整工程**と**補正工程**は性質が根本的に異なる。

```
検査工程：
  ワーク → 測定 → Go/No-Go 判定
  インプット: PFMEA・コントロールプランの管理特性

調整工程：
  ワーク → 測定 → 物理的操作（ねじ・スライダ・バルブ等）→ 再測定 → 目標値内か？ → 完了
  インプット: 「何を・どの目標値に・どの許容範囲で」合わせるか → 製品設計から取得必須

補正工程：
  ワーク → 測定 → 補正値の算出 → 補正データをワークに書き込み → 確認測定 → 完了
  インプット: 「何を測定するか・補正値の計算式・書き込む先と形式・確認規格」→ 製品設計から取得必須
```

**調整工程・補正工程において PFMEA とコントロールプランだけでは工具要求仕様書を書けない**。目標値・許容範囲・算出式の根拠は製品設計の文書にしか存在しない。この情報が口頭でやりとりされると、後から「なぜその数値か」のトレーサビリティが失われる。

> **重要**: 目標値・許容範囲・補正算出式を口頭またはメールのみで受け取った場合、根拠文書が発行されるまで工具要求仕様書の発行を保留する。口頭確認のみによる数値の記載は禁止する。

## 2. 調整工程と補正工程の定義

### 2.1 調整工程（Adjustment Process）

製品を**物理的に操作して**、指定の特性値を目標値に合わせ込む工程。

| 観点 | 内容 |
|------|------|
| 結果 | ワーク自体の物理状態が変化する（ねじが締まる、スライダが動く、など） |
| 完了条件 | 測定値が目標値の許容範囲に入ること |
| 典型例 | センサー感度調整、ヘッドギャップ調整、バルブ開度調整、テンション調整 |
| 治工具の役割 | 測定基準の提供・調整操作のアクセス確保・合否の自動判定 |

### 2.2 補正工程（Compensation / Correction Process）

ワーク個体ごとの**ばらつきを測定し**、そのばらつきを相殺する補正データを算出してワーク（またはワーク内メモリ）に書き込む工程。

| 観点 | 内容 |
|------|------|
| 結果 | ワーク自体の物理状態は変わらないが、補正データ（係数・テーブル等）が書き込まれる |
| 完了条件 | 補正データ書き込み後の確認測定が規格を満たすこと |
| 典型例 | ドット着弾位置補正、センサーゼロ点・スパン補正係数の書き込み、色補正プロファイルの書き込み |
| 治工具の役割 | 基準測定・補正値の算出・データ書き込み・書き込み確認・確認測定 |

## 3. 製品設計から取得すべきインプット

### 3.1 調整工程に必要なインプット

以下の情報はすべて**文書番号付きの正式な文書として**取得する。口頭・メール確認のみでの工具要求仕様書発行は禁止する。

| 必要な情報 | 内容 | 本来の出どころ文書 |
|-----------|------|-----------------|
| 調整対象パラメータ | 何を調整するか（物理量・単位） | 製品仕様書・機能設計書 |
| 調整目標値（Target） | 合わせ込む目標値 | 製品仕様書・品質規格書 |
| 調整後の許容範囲（Tolerance） | 目標値±いくらまで許容するか | 品質規格書・出荷検査規格書 |
| 調整の方向・操作のヒント | どの操作でどの方向にパラメータが変わるか | 製品設計書（製品設計者から取得） |
| 確認測定の方法 | 調整完了を何で・どう測定して確認するか | 試験規格書・製品仕様書 |
| 調整回数の上限 | 許容範囲に入らない場合の取り扱い | 品質規格書・工程設計規則 |
| 再調整トリガー | いつ再調整が必要か（ドリフト基準） | 製品設計書・保守仕様書 |

#### 製品設計への依頼時のポイント

```
依頼時に明確にすべき事項：
  ① 目標値と許容範囲は「工具要求仕様書に文書番号付きで引用する」ことを伝える
  ② 暫定値・口頭値・メール確認値は記載できないことを事前に合意する
  ③ 調整治具の設計開始に必要な期日を示し、文書の発行スケジュールを確認する
  ④ 文書が未発行の場合は工具要求仕様書の発行を保留する（ルール化）
```

### 3.2 補正工程に必要なインプット

| 必要な情報 | 内容 | 本来の出どころ文書 |
|-----------|------|-----------------|
| 補正対象特性 | 何のばらつきを補正するか（物理量・単位） | 製品仕様書・機能設計書 |
| 測定方法と基準器 | 何をどの基準器で測定するか | 試験規格書・製品仕様書 |
| 補正値の算出式 | 測定値から補正係数・補正データをどう算出するか | ソフトウェア設計書・機能設計書 |
| 書き込み先と形式 | 補正データをどこに（メモリアドレス・ファイル等）何の形式で書き込むか | ソフトウェア設計書・インターフェース仕様書 |
| 補正後の確認規格 | 書き込み後の確認測定の合否基準（補正残差） | 品質規格書・出荷検査規格書 |
| 補正限界値 | 補正データが限界値を超えた場合の取り扱い | 品質規格書・製品設計書 |
| データの保存・管理方法 | 工程内での補正データの取り扱いルール | トレーサビリティ規定・品質マニュアル |

## 4. 調整工程の工具要求仕様書の記載方法

### 4.1 要求背景（Section 3.2）への追記

調整工程では、「なぜその治工具が必要か」に加えて**「どの目標値に合わせるか」の根拠文書番号**を要求背景に必ず明記する。

```
記載例：
  【要求背景】
    本治工具は工程No.○○（○○調整）において使用する。

    調整の目標値・許容範囲は以下の文書に基づく：
    ・製品仕様書  DS-PX500-Rev.03  第4.2項「○○特性の規格値」
    ・品質規格書  QS-PX500-Rev.01  別紙A「工程内調整判定基準」

    ※上記文書が未発行の場合、本工具要求仕様書の発行を保留する。
      口頭・メール確認のみによる発行は行わない。
```

根拠文書の確認状況を仕様書末尾に記録しておくと、発行可否の判断根拠を残せる。

```
【根拠文書の確認状況（要求背景との対応）】
  ・DS-PX500-Rev.03  発行済（2025/11/01）  ✓
  ・QS-PX500-Rev.01  発行済（2025/12/15）  ✓
  → 上記確認により、本仕様書を発行する。
```

### 4.2 機能要求（Function Requirements）

調整工程の治工具には、検査工程にはない「**調整操作インターフェース**」機能と「**調整完了の自動判定**」機能が必要になる。

| 機能カテゴリ | 記載例 |
|------------|-------|
| ワーク保持機能 | ワークを固定保持しつつ、調整操作箇所（○○ねじ等）へのアクセスを妨げないこと |
| 測定・表示機能 | 調整対象パラメータをリアルタイムで測定し、現在値・目標値・上下限値を同一画面に表示できること |
| 調整完了自動判定機能 | 測定値が許容範囲に入った状態で規定時間維持された時点で「調整完了」を自動出力できること |
| 調整記録出力機能 | 調整完了時の測定値・調整日時・ワーク識別番号を上位システムに自動出力できること |
| ポカヨケ機能 | ワーク誤方向セットを防止すること、および「調整完了」出力前に取り出せない、または警告を発すること |
| 再調整検知機能 | 許容範囲内での完了後も規定回数を超える調整操作を検知・記録できること |

> **注意**: 「作業者が目視で測定値を確認して範囲に入ったら完了」という設計は避ける。個人差・読み取り誤差が品質ばらつきに直結するため、自動判定を機能要求として明記する。

### 4.3 性能要求（Performance Requirements）

| 性能項目 | 粒度・記載内容 | 根拠の考え方 |
|---------|-------------|------------|
| 測定確度 | 調整中の測定値誤差が ±○○ 以内であること | 許容範囲幅の 1/10 以下を目安とする |
| 表示更新速度 | 測定値の表示更新間隔が ○○ ms 以下であること | 作業者の操作に追従できる速度 |
| 調整完了判定の繰返し性 | 同一調整量での完了判定が ○○回中○○回一致すること | 治工具固有の判定ばらつきを数値化 |
| サイクルタイム | セットから調整完了出力まで ○○ 秒以内であること | タクトタイムからの逆算値 |

## 5. 補正工程の工具要求仕様書の記載方法

### 5.1 要求背景（Section 3.2）への追記

補正工程では、補正対象特性の規格に加えて**補正値算出式と書き込み仕様の出典文書番号**を必ず記載する。

```
記載例：
  【要求背景】
    本治工具は工程No.○○（○○補正データ書き込み）において使用する。
    PX-500は個体ごとに○○のばらつきを持つため、量産工程において
    個体別補正データを書き込み、出荷規格を満たすことが必要である。

    補正値の算出式・書き込み形式・確認規格は以下の文書に基づく：
    ・ソフトウェア設計書  SW-PX500-CAL-Rev.02  第3章「○○補正アルゴリズム」
    ・インターフェース仕様書  IF-PX500-Rev.01  第5項「補正データメモリマップ」
    ・品質規格書  QS-PX500-Rev.01  別紙B「補正後確認規格値」

    ※上記文書が未発行の場合、本工具要求仕様書の発行を保留する。
```

### 5.2 機能要求（Function Requirements）

補正工程は「測定→算出→書き込み→確認」の4ステップが治工具に求める機能の柱になる。

| 機能カテゴリ | 記載例 |
|------------|-------|
| ワーク保持機能 | ワークを測定基準面に対して固定保持できること |
| 基準測定機能 | 補正前の○○特性を規定の基準器・測定条件で自動測定できること |
| 補正値算出機能 | 測定値から補正データを自動算出できること（算出式は SW-○○-Rev.○○ 第○章に従う） |
| データ書き込み機能 | 算出した補正データをワークの指定メモリ領域に書き込めること（書き込み仕様は IF-○○-Rev.○○ 第○項に従う） |
| 書き込み確認機能 | 書き込んだデータを読み返し、書き込みデータとバイト単位で一致することを確認できること |
| 確認測定・合否判定機能 | 書き込み後に○○特性を再測定し、確認規格（QS-○○-Rev.○○ 別紙○）への適合を自動判定できること |
| 補正限界検知機能 | 算出した補正データが規定の補正限界値（SW-○○-Rev.○○ に規定）を超えた場合、異常品として検出・報告できること |
| 記録出力機能 | 測定値・補正値・書き込み結果・確認測定値・判定結果・実施日時・ワーク識別番号を上位システムに出力できること |

> **書き込み確認機能について**: 補正データの書き込みエラー（通信断・メモリ不良等）はワークが出荷後に想定外の動作をする原因になる。読み返し確認は省略せず、機能要求として必ず明記する。

  根拠文書の確認状況:
    ・SW-PX500-CAL-Rev.02  発行済（2026/01/20）  ✓
    ・IF-PX500-Rev.01      発行済（2025/12/01）  ✓
    ・QS-PX500-Rev.01      発行済（2025/12/15）  ✓
    → 上記確認により、本仕様書を発行する。

【1. 要求背景】
═══════════════════════════════════════════════════════════════════

  工程No.130（ドット着弾位置補正）では、PX-500の個体ごとにドットの
  着弾位置ずれを測定し、補正係数を算出してヘッドドライバーICの
  不揮発性メモリに書き込む。

  個体ごとのヘッドユニット取り付けばらつきにより着弾位置がずれ、
  補正なしでは出荷検査（着弾位置ずれ ±15 μm 以内）を満たせない。

  補正値算出式・書き込み仕様・確認規格は以下の文書に基づく：

    ・IF-PX500-Rev.01 第5項
      → 書き込み先メモリアドレス・データ形式（符号付き16bit整数）
    ・QS-PX500-Rev.01 別紙D
      → 補正後確認規格: 着弾位置ずれ残差 ±15 μm 以内

【2. 機能要求】
═══════════════════════════════════════════════════════════════════

  F-01: ワーク保持機能
        PX-500本体を測定基準面に対して固定保持できること。

  F-02: 基準測定機能
        補正用テストパターンを印刷させ、カメラ・センサーにより

  F-03: 補正値算出機能

  F-04: 補正データ書き込み機能

  F-06: 確認測定・合否判定機能

  F-07: 補正限界検知機能

  F-08: 記録出力機能

## 8. よくある課題と対策

### 課題1: 目標値・許容範囲が文書化されていない段階で仕様書を書こうとする

**問題**: 製品設計が完了していない時期（品質規格書が未発行）に治工具の発注を急ぎ、口頭でもらった暫定値をそのまま工具要求仕様書に記載してしまう。後から規格値が変わった際に、治工具の作り直しや設定変更が生じる。

**対策**: 要求背景に根拠文書番号を記載することをルール化し、「文書番号記載なし＝発行不可」とする。暫定値しか取得できない場合は「Rev.Draft（暫定）」として管理し、正式版の発行は文書確定後とする。

### 課題2: 補正値の算出式がソフトウェア設計書にしかなく、工程設計技術課に伝わっていない

**問題**: 補正アルゴリズムはソフトウェア開発部門が管理しており、工程設計技術課が把握していない。治工具に実装すべき算出式が不明なまま仕様書を書こうとする。

**対策**: 補正工程の工具要求仕様書を発行する前に、ソフトウェア設計書の「補正アルゴリズム」章をインプットとして受領し、文書番号・版数・受領日を要求仕様書に記載する。

### 課題3: 調整完了の判定を作業者の目視・感覚に委ねている

**問題**: 「作業者が目視で測定値を確認して、範囲に入ったと判断したら完了」という工程設計にしてしまう。個人差・読み取り誤差が品質ばらつきに直結する。

**対策**: 機能要求に「調整完了の自動判定機能」を明記し、人の目視判断に依存しない設計を治工具側に要求する。

### 課題4: 補正限界超えのワークの扱いが決まっていない

**問題**: 補正係数が限界値を超えたワーク（組み付け不良・部品不良の疑い）の扱い（廃棄・再組立・上位に判断委ね）が工程設計上決まっていない。

**対策**: 機能要求に「補正限界超え検知機能」を必ず含め、限界値の根拠文書番号（ソフトウェア設計書）も記載する。限界超え時の処置フローは工程指示書（[07-process-instructions.md](./07-process-instructions.md)）に別途定める。

### 課題5: 書き込み確認（読み返し）を省略する

**問題**: 「書き込みコマンドが成功ステータスを返したから確認不要」として、補正データの読み返し確認を治工具の機能要求から省いてしまう。通信エラーやメモリ不良による書き込みミスが見逃され、補正なしのワークが流出するリスクがある。

**対策**: 書き込み確認（読み返し一致確認）を独立した機能要求（F-05 相当）として必ず記載する。書き込みコマンドの成功ステータスとデータの正確な書き込みは別物であることを生産技術課と合意しておく。
