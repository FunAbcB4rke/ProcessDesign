# 特性マトリクス（Characteristics Matrix）

## 1. 概要

### 特性マトリクスとは

特性マトリクス（Characteristics Matrix）とは、**製品特性（Product Characteristics）と製造工程（Process Operations）の関係**を一覧化したマトリクス（行列）形式の文書です。どの工程でどの製品特性が「作られる」「影響を受ける」「基準として使われる」かを視覚的にマッピングします。

AIAG APQP マニュアル（Section 3.4）で、工程設計・開発フェーズ（Phase 3）のアウトプットの一つとして定義されています。

### 目的

| 目的 | 説明 |
|------|------|
| 工程と特性の関係可視化 | どの工程がどの製品特性に影響するかを一目で把握する |
| 重要工程の特定 | 多くの特性に影響する工程（＝品質上の重要工程）を識別する |
| 重要特性の特定 | 多くの工程に影響される特性（＝管理の複雑な特性）を識別する |
| PFMEAの補助 | 工程と特性の関係を明確にし、PFMEAの故障分析を支援する |
| コントロールプランへの展開 | 管理すべき工程・特性の組み合わせを漏れなく特定する |
| 部門間の共通認識 | 設計・生産技術・品質間で、工程と特性の関係を共有する |

### 工程設計における位置づけ

```
工程フローチャート ──→ マトリクスの行（工程）を定義
         +
製品図面・仕様書   ──→ マトリクスの列（特性）を定義
         │
         ▼
    特性マトリクス ──→ 工程と特性の関係をマッピング
         │
         ├──→ PFMEA（どの工程のどの特性にリスクがあるかの入力）
         └──→ コントロールプラン（管理すべき工程・特性の特定）
```

特性マトリクスは、工程フローチャートとPFMEAの**橋渡し**となる文書です。工程フローチャートが「何をするか」を示し、特性マトリクスが「それによって何が影響を受けるか」を示します。

## 2. マトリクスの構造

### 基本構造

特性マトリクスは、**行に製造工程（Operations）**、**列に製品特性（Characteristics）** を配置した行列です。

```
                     製品特性（列）
                  ┌──────────────────────────┐
                  │ 特性A  特性B  特性C  特性D │
                  │                          │
製造工程（行）     │                          │
  工程1           │   X              L       │
  工程2           │          X      C       │
  工程3           │   L      X              │
  工程4           │                 X    X   │
                  └──────────────────────────┘
```

### 記号の意味

セルには、その工程と特性の間にどのような関係があるかを記号で示します。

| 記号 | 英語 | 意味 | 説明 |
|:----:|------|------|------|
| X | Created / Modified | 生成・変更 | その工程でこの特性が作られる、または変更される |
| L | Locating | 位置決め | その特性がこの工程でワークの位置決め基準として使用される |
| C | Clamping | クランプ | その特性がこの工程でワークの固定基準として使用される |

> **補足**: 記号体系は企業ごとに異なる場合があります。「X」のみで関係の有無だけを示すシンプルな方式も広く使われています。重要なのは、関係の有無を明確にすることです。

### ヘッダー情報

| 項目 | 内容 |
|------|------|
| 品名 / 製品名 | 対象製品の名称 |
| 品番 / 図番 | 製品の部品番号や図面番号 |
| 作成日 / 改訂日 | 初版日と最新改訂日 |
| 作成者 / 承認者 | 作成・承認の責任者名 |

## 3. 作成手順

### Step 1: 製品特性の洗い出し（列の定義）

製品図面・仕様書から、管理対象となる製品特性を洗い出します。

**洗い出すべき特性**:
- 寸法特性（長さ、径、位置度、幾何公差など）
- 外観特性（表面粗さ、キズ、色差など）
- 材料特性（硬度、引張強度、膜厚など）
- 機能特性（嵌合、気密性、導通など）
- 特殊特性（CC / SC）

### Step 2: 製造工程の洗い出し（行の定義）

工程フローチャートから、全工程を転記します。

**含めるべき工程**:
- 受入工程
- すべての加工工程
- 検査工程
- 運搬・保管工程（特性に影響を与える場合）

### Step 3: 関係のマッピング

各工程と各特性の交点（セル）に対して、関係の有無と種別を記入します。

**判断の観点**:
- この工程で、この特性は**新たに作られるか**？ → X
- この工程で、この特性は**変更されるか**（追加加工、熱変形等）？ → X
- この工程で、この特性は**位置決め基準として使われるか**？ → L
- この工程で、この特性は**固定基準として使われるか**？ → C
- この工程と、この特性に**関係がないか**？ → 空欄

### Step 4: 分析と活用

完成したマトリクスを分析します。

| 分析の観点 | 方法 | 活用 |
|-----------|------|------|
| 列の「X」が多い特性 | 多くの工程で影響を受ける | 管理が複雑な特性 → CPで重点管理 |
| 行の「X」が多い工程 | 多くの特性に影響する | 重要工程 → PFMEAで重点分析 |
| 「L」「C」の関係 | 基準面として使用されている | 基準面の精度管理が重要 |
| 空白行 | どの特性にも影響しない工程 | 本当に必要な工程か再検討 |
| 空白列 | どの工程にも紐づかない特性 | 工程の抜け漏れの可能性 |

## 4. 具体例

### 金属プレス部品（ブラケット A）の特性マトリクス

品名: ブラケット A　品番: BK-1234

**製品特性（列）**:
- A: ブランク外形寸法（100±0.5mm）
- B: 絞り深さ（15.0±0.3mm）◆CC
- C: 穴位置度（φ0.2以内）◆CC
- D: 穴径（φ8.0 +0.05/-0.00）◇SC
- E: 外形寸法（120±0.3mm）◇SC
- F: 塗装膜厚（15±3μm）◇SC
- G: 外観（キズ・打痕なし）

```
                              製品特性
工程                    A     B     C     D     E     F     G
No.  工程名            ブラン 絞り  穴位  穴径  外形  膜厚  外観
                       ク外形 深さ  置度
                             ◆CC   ◆CC  ◇SC  ◇SC  ◇SC
────────────────────────────────────────────────────────────
 10  原材料受入・保管
 20  受入検査
 30  材料搬送
 40  ブランキング       X                                   X
 50  1次プレス成形      L     X                 X           X
 60  2次プレス成形      L           X     X                 X
 70  工程内検査
 80  バリ取り                                               X
 90  洗浄                                                   X
100  表面処理                                         X     X
110  外観検査
120  最終検査
130  梱包                                                   X
```

### マトリクスから読み取れること

| 分析結果 | 読み取り | アクション |
|---------|---------|-----------|
| 特性G「外観」は多くの工程（40,50,60,80,90,100,130）でXが付く | 外観は多くの工程で影響を受けやすい | 各工程でのキズ防止策を検討。搬送・保管含め広く管理 |
| 工程50「1次プレス成形」はA,B,E,Gの4特性に影響 | 最も多くの特性に影響する重要工程 | PFMEAで重点的にリスク分析。CPで厳格な管理を設定 |
| 特性B「絞り深さ◆CC」はXが工程50のみ | 単一工程で決まる特性 | 工程50の管理に集中すればよい |
| 特性A「ブランク外形」が工程50,60でLマーク | 後工程の位置決め基準として使用 | ブランキング工程での精度確保が後工程品質に直結 |

## 5. PFMEAおよびコントロールプランとの連携

### PFMEAへの活用

特性マトリクスで「X」が付いた工程と特性の組み合わせが、PFMEAの分析対象になります。

```
特性マトリクス                 PFMEA
─────────                 ─────

工程50 × 特性B（X）  ──→   工程50: 絞り深さ不足の故障分析
工程50 × 特性E（X）  ──→   工程50: 外形寸法不良の故障分析
工程60 × 特性C（X）  ──→   工程60: 穴位置度不良の故障分析
```

### コントロールプランへの活用

「X」が付いた組み合わせは、コントロールプランで管理すべき項目の候補です。

| 特性マトリクスの結果 | CPでの管理指針 |
|--------------------|--------------|
| X（特性の生成・変更） | その工程で製品特性を管理項目として設定 |
| L / C（基準として使用） | 基準面の精度を工程特性として管理 |
| 特殊特性（CC/SC）にXが付く工程 | 高頻度・高精度の管理を設定 |

## 6. 類似ツールとの違い

### 特性マトリクス vs PFMEA

| 観点 | 特性マトリクス | PFMEA |
|------|-------------|-------|
| 目的 | 工程と特性の「関係」を可視化 | 工程の「リスク」を分析 |
| 内容 | 関係の有無（X / L / C） | 故障モード・影響・原因・S/O/D/AP |
| 詳細度 | 俯瞰的（関係のマップ） | 詳細（各故障モードの分析） |
| 作成順序 | PFMEAの前（インプット） | 特性マトリクスの後 |

### 特性マトリクス vs QC工程表

| 観点 | 特性マトリクス | QC工程表 |
|------|-------------|---------|
| 目的 | 関係のマッピング | 管理方法の規定 |
| 内容 | 工程×特性の関係 | 管理基準・測定方法・頻度・異常時処置 |
| 形式 | マトリクス（行列） | 表形式（工程ごとの管理項目一覧） |
| 作成順序 | 先（関係の把握） | 後（管理方法の詳細化） |

## 7. よくある課題と対策

### 課題1: 特性の洗い出しが不十分

**問題**: 図面の寸法特性だけを列挙し、外観・材料特性・機能特性が漏れている。

**対策**:
- 図面だけでなく、仕様書・顧客要求・過去のクレーム情報も参照する
- 寸法・外観・材料・機能の4カテゴリで漏れがないかチェックする
- 特殊特性リストと照合し、CC/SCが全て含まれていることを確認する

### 課題2: 関係の判断が曖昧

**問題**: その工程が本当にその特性に影響するのか、チーム内で意見が分かれる。

**対策**:
- 「この工程を省略したら、この特性はどうなるか？」と考える
- 影響が「直接的」なものをXとし、「間接的」なものは注記で補足する
- 不明な場合はXを付けておき（保守的に判断）、PFMEAで詳細を分析する

### 課題3: 作成されるが活用されない

**問題**: APQP書類として作成するが、PFMEAやCPの作成時に参照されていない。

**対策**:
- PFMEAのStep 2（構造分析）で特性マトリクスを明示的にインプットとして使用する
- マトリクスの分析結果（重要工程・重要特性の特定）をPFMEA・CPのレビューで確認する
- 「特性マトリクスでXが付いている組み合わせが、CPで管理されているか」をクロスチェックする

### 課題4: メンテナンスが追いつかない

**問題**: 設計変更や工程変更があっても、特性マトリクスが更新されない。

**対策**:
- 工程フローチャートの改訂時に連動して更新するルールを定める
- 設計変更による特性追加・削除時にも更新する
- 文書管理システムで他の工程設計文書と関連付ける

---

## 8. ベストプラクティス

1. **工程フローチャートと製品図面を並べて作成する**: 行と列のインプットを正確に揃える
2. **特殊特性を明確にマーキングする**: CC/SC記号を列ヘッダーに明記する
3. **PFMEAの前に完成させる**: 特性マトリクスの結果がPFMEAの分析範囲を決定する
4. **シンプルに保つ**: 記号は最小限（X / L / C または X のみ）で十分
5. **分析結果を活用する**: 作成して終わりではなく、重要工程・重要特性の特定に活用する
6. **Excelで管理する**: マトリクス形式はスプレッドシートとの相性が良い
